# 开发任务：按本设计文档完整实现游戏（使用包内资产）

## 角色与目标

你是一名资深游戏开发 agent。你的任务是：**仅依据本文件夹（brotato_original_devkit/）内的设计文档与资产**，从零完整实现这款波次生存类动作 Roguelite 游戏——机制、数值、操控手感、界面流程、画面、音频全部严格按文档落地。本文件夹是唯一规格来源，不要引入文档之外的设定或你对同类游戏的记忆。

## 输入材料

| 路径 | 作用 |
|---|---|
| `game_mechanics_docs/00_总览与目录.md` | 入口：全貌、核心公式速查、各章关系 |
| `game_mechanics_docs/01~12` | 逻辑层完整规格（属性/角色/武器/商店/道具/敌人/波次/效果/进度联机/手感/UI/表现层底层） |
| `asset_pack/assets/` | 游戏全套资产文件（按类别组织） |
| `asset_pack/资产对应_01~07.md` | 每个游戏元素 → 资产文件 + 使用参数（缩放/锚点/挂点/动画时序/九宫格/音量变调） |

## 硬性规则

1. **任何数值、公式、概率必须出自文档**，禁止凭记忆补全、禁止"合理估计"。文档冲突或缺失时不要猜：记录到 `OPEN_QUESTIONS.md`（条目格式：问题、涉及文档位置、你暂采用的保守处理），继续开发不阻塞。
2. **逻辑帧率 60 ticks/s 固定步**。文档中"帧"= 1/60 秒。百分数存储约定（角色属性整数百分比 vs 武器 0~1 小数）见 00 章与各章标注。
3. **资产使用参数照搬对应表**：精灵缩放/锚点/偏移、武器挂点、动画各阶段时长、九宫格边距、音量 dB 与变调随机档位。禁止"看起来差不多"。
4. **程序化表现必须按文档实现，不要想当然**（这些是最容易做错的点，文档均有明确结论）：
   - 角色待机/移动是位移+缩放的变换动画（squash & stretch），不是帧动画；朝向翻转用水平缩放取负；
   - 玩家移动瞬时达速、无加速度插值；武器瞬时指向索敌目标、无旋转插值；
   - 品质颜色、地面材料绿色全部是代码染色，资产本体不含品质色；
   - 武器各品质共用同一贴图，差异仅数值与 UI 边框色；
   - 地面为 12 子砖加权随机填充（约 82% 素面），六套主题每局随机；
   - 音乐为 11 首单列表洗牌轮播，场景区分仅靠音量分档（无场景切歌）。
5. **引擎**：推荐 Godot 4.x（文档参数语义与其最接近）；可用其他引擎，但必须保留 60Hz 固定逻辑步、12 层物理碰撞分层语义（12 章分层表）与全部参数语义。
6. 开发语言、代码结构自定，但数据应做成数据驱动（角色/武器/道具/敌人/波次表独立于代码），便于对照文档校对。

## 开发里程碑（按此顺序，每个里程碑出一份自测报告）

- **M1 数据层**：01 章四层属性聚合管线（永久/临时/联动/增益 + 上下限截断）+ 08 章效果系统（效果字典 234 键、效果施加/撤销、事件钩子）。验收：属性聚合单元测试通过。
- **M2 战斗核心**：03 章武器与伤害公式（伤害构建/冷却/暴击/燃烧）、06 章敌人（含 HP/伤害缩放公式、移动与攻击行为）、07 章波次生成（20 波刷怪表、生成位置与上限）、10 章操控（移动/索敌/拾取吸附/击退）。验收：能完整打通危险 0 的 20 波。
- **M3 经济与成长**：04 章商店（品质概率/价格/重随/锁定/合成）、02/05 章数据全量灌入（49 角色、209 道具、61 武器×4 品质）、升级选项。验收：商店概率统计测试 + 任选 5 角色数值抽查。
- **M4 元游戏**：09 章解锁/113 挑战/存档；无尽模式（07 章无尽公式）。
- **M5 界面与表现**：11 章 UI 流程/HUD/伤害数字（颜色与动画参数）、12 章粒子/屏震/闪白/音频播放规则，按对应表挂全部资产。
- **M6 收尾**：本地合作模式（09 章规则差异）、约 50 项设置、危险 1–5。

## 最终验收清单

1. **数值**：把 00 章速查公式 + 各章关键公式做成 ≥30 条自动化测试（伤害、玩家/敌人两套护甲、攻速冷却、经验曲线 (3+L)²、商店品质概率、重随价格、敌人 HP 缩放、收获结算、再生间隔、吸血概率与节流）。
2. **手感**：移动无插值、索敌"射程+200px 最近敌人"、拾取 150px 吸附圈 + 500px/s+1200px/s² 飞行、受击无敌帧 0.2–0.4s、出生预警 1s。
3. **画面音频**：抽 10 个元素对照资产对应表核对挂载参数；音效变调三档随机与限流；伤害数字颜色规则。
4. `OPEN_QUESTIONS.md` 内每条都有保守处理且不影响主流程。
