# 11. 界面流程与 HUD

> 本章覆盖全部界面流程、局内 HUD、伤害数字系统、升级选择界面、商店界面布局与交互（经济公式见 04 章）、结算界面、设置选项全清单、属性面板格式化规则。
> 基准分辨率：1920×1080，2D 等比拉伸、保持宽高比。UI 坐标均以此为基准。

---

## 1. 完整界面流程（场景级状态机）

游戏通过整场景切换在以下界面间流转，没有全局状态机，转换逻辑分散在各界面内：

| 界面 | 说明 |
|---|---|
| 标题画面 | 主菜单 |
| 局内战斗 | 波次主场景 |
| 商店 | 单人商店（合作模式有专用合作商店变体） |
| 角色选择 | 局前选择 |
| 武器选择 | 局前选择 |
| 难度选择 | 局前选择 |

合作模式专用：合作商店、合作续档、合作结算三个界面，按"是否合作局"自动切换。

### 1.1 流程图

```
启动（引导场景：启动 Splash 状态机）
 │ 等待进度存档系统加载完毕 → 播放启动动画
 │ （若存档全损坏[Steam/Epic]则停在引导页）
 ▼
标题画面
 │ ├─ Start ───────────────► 角色选择
 │ ├─ Continue(有存档运行)──► 单人:商店 / 合作:合作续档
 │ ├─ Options / Codex / Credits / Profile ─ 同场景内页面切换
 │ └─ Quit ─ 退出游戏
 ▼
角色选择 ──确认(全部玩家选定+0.8s 延迟[仅合作])──►
 │  若任一玩家角色有武器槽 → 武器选择
 │  否则直接发放初始武器与道具 → 难度选择
 │  返回键 → 标题画面
 ▼
武器选择 ──确认──► 难度选择
 │  返回 → 撤销选择并回角色选择，光标恢复到上次所选角色
 ▼
难度选择 ──点选难度──► 应用难度效果、初始化精英/Boss 波、保存 → 局内战斗
 │  返回 → 回武器选择(若有武器槽)或角色选择
 ▼
局内波次
 │ ├─ 波次计时归零 → 清场收尾
 │ ├─ 全员死亡 → 清场收尾（判定"本局失败/本波失败"）
 │ └─ 清场结束计时器超时：
 │      ├─ 波失败 + retry_wave 设置开 + wave>1 → 显示"重试本波"界面
 │      ├─ 输/赢 → 结算界面（单人/合作）
 │      └─ 普通过波 → 逐个处理升级与宝箱（升级界面）→ 等挑战弹窗播完 → 商店
 ▼
商店（单人/合作）
 │  GO 按钮(全部玩家按下) → 当前波 += 1 - 沙漏数 → 局内战斗
 │  第 19 波商店额外显示"无尽模式"按钮
 ▼
结算（单人/合作）
 ├─ Restart: 同配置重开 → 局内战斗
 ├─ New run: 回角色选择（光标定位到上次角色）
 └─ Exit: 回标题画面
```

### 1.2 关键条件

- **Continue 按钮显隐**：存在进行中对局存档、且 DLC 校验通过。继续时还会结算"沙漏挑战"（在波内退出后继续）。
- **菜单页面切换框架**：统一的页面切换器：显示并初始化目标页、隐藏来源页，并广播"页面已切换"事件。标题画面在非主菜单页时显示半透明遮罩。
- **结算后的难度解锁**：胜利时若 `当前危险+1` 高于角色当前可选上限且 ≤ 最大危险，则记录新解锁难度 = 当前危险 + 1，在结算界面进度栏展示（见 §9）。
- **暂停菜单内的退出**：退出确认 → 回标题画面，若波内退出且剩余时间 < 沙漏挑战阈值则记录 `chal_hourglass_quit_wave` 标记。重开确认 → 清空运行存档重开。结束本局确认 → 按当前进度判定胜负后进结算。

## 2. 启动与标题画面

### 2.1 引导场景

状态机 `READY → BOOT_SPLASH → HIDE_SCENE → CHANGE_SCENE`：首帧绘制后进入 BOOT_SPLASH；收到进度存档系统就绪事件后播放启动动画，应用音量与设置，然后切到标题画面。若存档为"全部损坏"（Steam/Epic）则不放行。

### 2.2 标题画面

- **动态背景（keyart）**：按设置 `main_screen_keyart` 选择：0=最新更新的 keyart、1=随机、≥2=指定第 n-2 张；同时换 Logo。
- **复位运行状态**：进入标题画面即把当前区域归 0 并重置对局状态，清空合作连接。
- **音乐**：若标记了"重载音乐"则重新起播。从标题画面进角色选择时音乐音量渐变 -5dB。

### 2.3 主菜单按钮

左列：Continue（条件显示）、Start、Options、Codex、Profile（文本 `MENU_PROFILE` 带当前档位号 +1）、Quit；右列：More games/DLC、Newsletter、Community、Credits。要点：

- 主机平台隐藏 Newsletter/Community 等网络入口，并把 Credits/Codex 移到左列；Quit 仅 PC 桌面平台显示。
- 拥有 DLC 时 More games 显示普通文本，否则显示金色 `MENU_DLC_AVAILABLE_STANDARD`（按钮文本色 `#FFD700`）。
- 版本号 Label（主机版去掉后缀），Xbox 平台显示 Gamertag。
- **错误提示 Label**：存档加载失败按加载状态枚举显示对应翻译键（`CORRUPTED_SAVE` 等 7 种）。
- 焦点规则：有可继续运行时 Continue 抓焦点，否则 Start。

### 2.4 网络公告（announcement）

- 公告系统：启动时向公告服务器拉取当前平台的已发布公告（带起止时间过滤、已读 id 过滤），下载多语言文案与图片后广播"公告就绪"事件。
- 公告控件位于标题画面角落。首次显示从左侧滑入 0.5s（指数缓出）；点击关闭按钮标记已读并滑出 0.4s；鼠标悬停整体放大到 1.05 倍（0.1s）；点击气泡打开公告链接。

### 2.5 其他标题页

- **Credits**：富文本 + 返回按钮，取消键可退出。
- **Profile 存档档案页**：3 个存档位（软盘动画），LT/RT 或按钮切换浏览位；显示每档：获胜次数、解锁角色数、完成挑战数、是否有进行中运行（软盘插入动画 `run_on/run_off/run_nosave`）、全解锁标记。操作：Select（切换当前档）、Reset（确认弹窗）、Unlock all（确认弹窗，标记 `is_unlock_all_save`）、Copy ×2（从另一档复制，确认弹窗）。
- **Codex 图鉴**（弹出页）：4 个 Tab（挑战/道具/武器/敌人，LT/RT 切换）。道具与武器按 Tier 排序展示，未解锁显示锁图标，点击显示 `???`；武器详情页提供该武器全升级链的 4 个 Tier 按钮；敌人页显示 HP/伤害/护甲随波数增长曲线图、速度条、击退抗性条、掉落材料数。顶部进度条显示各 Tab 收集百分比。每个图鉴清单支持排序下拉（按 Tier/名称/购买次数/击杀数/强度/HP/类别/获得顺序/武器伤害/装填/射程/实体类型 12 种，排序选择持久化到 `settings.sort_inventory_presset`）。

## 3. 局前选择界面（角色 / 武器 / 难度）

### 3.1 通用框架（选择界面基类）

- 结构：最多 4 套网格清单 + 4 个信息面板（物品面板/角色面板）+ 背景图（当前 Zone 的界面背景）。
- 子类钩子：取全部候选元素 / 取已解锁元素 / 是否展示锁定项 / 全员选定回调。
- 元素列表生成：每个元素复制后标记 `is_locked = 未解锁`；若不展示锁定项（武器选择）直接剔除锁定项。列表头部可插入"随机"特殊元素（启用随机且可选项 >1 时）。
- 焦点/悬停：聚焦元素时在对应玩家面板显示详情；随机元素显示 RANDOM 面板；锁定元素隐藏数据面板。进入界面时光标恢复到上次所选元素。
- **确认规则**：玩家按下元素即记录选择，面板打勾（仅合作）；当所有在场玩家都已选择：单人立即完成，合作启动 0.8s 延迟计时器（期间任何玩家按取消可撤回自己的选择并中止计时）。
- 返回：取消键仅单人模式可整页返回。

### 3.2 角色选择

- 元素=全部角色；角色 tier 用于边框颜色 = 该角色在当前 Zone 已通关的最高危险等级（0..5）。底部图例显示危险 0-5 的颜色（若玩家自定义过 tier 颜色才显示）。
- 信息面板显示该角色：最高通关难度（含 enemy scaling 缩写文本）与无尽最高波记录。enemy scaling 文本格式：` - {(h*d*s)^(1/3)%} - C - R{重试} - B{ban数}`，全 100% 时省略百分比。
- 锁定角色聚焦时显示锁定面板：对应解锁挑战的描述（由挑战系统按奖励反查）。
- **运行选项面板**：Zone 选择下拉（仅多 Zone 时显示）、无尽模式开关（持久化 `settings.endless_mode_toggled`）、合作模式开关（写入对局玩法模式）、Ban 模式开关（需完成对应挑战才显示，持久化 `settings.ban_mode_toggled`）。切 Zone 即时换背景/元素配色/信息面板。
- **合作加入流程**：开启合作后清空玩家，显示 4 个加入面板（按住接受键加入，进度环按连接进度填充并用玩家色）；每新连入一名玩家把其光标放到清单第一个元素。随机选择从全部已解锁角色中抽取。PS 平台为手柄灯条设置 4 色。
- 全解锁存档显示全解锁角标。

### 3.3 武器选择

- 元素=角色的起始武器 + 起始道具；**未解锁的不显示**。
- 网格列数：单人用预设列数；合作时总列数 16 按人数均分。
- 左侧面板显示当前角色（单人）；无武器槽且无初始道具的角色自动视为已选并把面板换成角色展示。
- 合作模式按住"详情"键可临时把自己的面板切回角色查看，松开恢复；底部提示条按在场玩家的输入设备种类显示对应按键图标，多设备用 `/` 分隔。
- 确认后：随机项从未锁定项抽取；写入所选武器/道具，再发放初始武器与道具 → 难度选择。

### 3.4 难度选择

- 元素=全部难度档；解锁条件：`难度值 ≤ 角色在当前 Zone 的 max_selectable_difficulty` 或默认解锁。该界面无"随机"项。
- 进入时光标定位到该角色上次选择的难度；左右面板显示角色与所选武器（仅单人）。
- 点选难度后依次执行：记录 `difficulty_selected_value`（单人）→ 写入当前危险、重置并生成精英波表、记录启用 DLC、应用难度效果、为每名玩家发满 Ban 令牌、生成头目波表、快照 enemy scaling 设置 → 进入局内战斗并累计 `run_started+1`。

## 4. 局内 HUD（四边边距 24px）

HUD 元素按玩家索引组织。**布局位置**：P1 左上、P2 右上、P3 左下、P4 右下（索引 0/2 靠左，0/1 靠上；下方玩家的金币行移到血条上方）。

| 元素 | 显示内容与数据来源 | 更新时机 | 备注 |
|---|---|---|---|
| 血条 | `当前/最大生命`，中央 Label `"cur / max"`（cur 取 max(hp,0)） | 生命变更事件 | 进度色暗红 `(0.721,0,0)`；受击瞬间整条变白并由计时器复原；有 `hit_protection` 效果时变紫 `(0.678,0.353,1)`（每物理帧刷新效果色） |
| 经验条 + 等级 Label `"LV.{level}"` | 当前经验 / 升级所需经验；显示值 `int(xp) % ceil(max_xp)` | 经验增加事件；升级时刷新等级 Label | |
| 材料(金币)计数 | 图标+数字，玩家当前材料数 | 金币变更事件 | 合作模式数字与图标染玩家色，血条/经验条透明度 0.75 |
| 奖励金袋（仅 P1 容器内） | 全队共享奖励金 | 奖励金变更事件；值为 0 隐藏 | 波结束清场时显示，地上材料飞向其屏幕位置对应的世界坐标"金袋"；清场后悬停显示 `INFO_BONUS_GOLD` 浮窗 |
| 波次标题 | `WAVE n` 大写 | 进场一次 | 顶部居中 |
| 波次倒计时 | `ceil(剩余秒数)` | 每帧 | 剩 <6s 起每秒播 tick 音并在首次 tick 时变为 `color_negative`；存在"半波转换属性"效果时半波刻变 `deepskyblue`；波结束隐藏 |
| 角色头顶血条（跟随角色） | 同血条 | 同血条 | 仅当设置 `hp_bar_on_character` 且 HP 不满且未死亡时显示；玩家死亡时主血条置 100、变白并闪烁淡出 |
| 待处理升级队列 | 每次升级追加一个升级图标（带等级值） | 升级事件 | 同图标 >4 个时折叠成 1 个带计数（同类图标上限 4）；悬停显示 `INFO_LEVEL_UP` 浮窗；单人时容器右对齐且右边距为 0，合作居中分四列 |
| 待处理宝箱队列 | 拾取的待处理消耗品（宝箱）图标 | 拾取宝箱事件 | 合作开启 `share_coop_loot` 时分给当前队列最短的玩家；悬停 `INFO_ITEM_BOX`；合作模式图标缩放 0.77 |
| 受伤暗角 | 全屏暗角，强度 `multiplier = clamp(hp/max,0.3,0.8)` | HP 更新（仅单人 P1） | 多人禁用；设置 `darken_screen=false` 时固定 0.8 |
| 屏幕变暗层 | 波结束渐暗：alpha 0→0.5，1s 线性 | 清场时 | |
| 波次完成大字 | `WAVE_COMPLETED` / `WAVE_FAILED` / `RUN_WON` / `RUN_LOST` | 清场尾部 | 打字机效果：每 0.1s 显示 1 个字符并播随机音(-10dB)；屏宽居中，高 400；输/赢换专用结算字体 |
| 准星 | 手柄手动瞄准时显示在角色前方 200px | 每帧 | 鼠标手动瞄准时改为把鼠标光标换成准星贴图（热点 35,35）；非手动瞄准时隐藏鼠标 |
| 信息浮窗 | 清场阶段悬停 HUD 队列/金袋的说明文本 | 鼠标进出事件 | |

**波次结束时序**（详见 07 章数值）：波计时归零（结算波末属性/收获等）→ 清场：渐暗屏幕、停表、启动清场结束计时器（普通 1s；输 2.5s / 赢 4s / 失败且可重试 1.5s）、播放结束音、吸取地面材料与消耗品、（赢时等玩家胜利动画播完后播胜利音）→ 波次完成大字 → 计时器超时决定去向（§1.1）。失败分支音乐渐弱至 -20dB，商店分支 -8dB。

**重试本波界面**：垂直按钮组，显示 `RETRY_NUMBER`（含已重试次数）；确认 → 恢复波开始状态、重试数+1 → 重打本波；取消 → 结算界面。显示期间禁用暂停。

## 5. 伤害数字 / 浮动文字系统

浮动文字由专门的浮动文字管理器（局内版与商店版两套）驱动。注意：全局确认弹窗管理器并非伤害数字系统（见 §13）。

### 5.1 单条浮动文字（Label）

动画两段，参数由管理器传入（默认 `duration=0.5s`、方向 `(0,-80)`、扇形散布 `spread=PI/4`）：

1. **漂移段**：位置沿 `direction.rotated(rand(-spread/2, spread/2))` 位移，时长 `duration`，弹性缓出。
2. **消失段**：再用 `duration` 同时把缩放 → (0,0)（弹性缓入缓出）与透明度 → 0（线性）。
3. 完成后隐藏并发"可复用"事件回对象池。

文本一律大写；颜色写在自身调制色；可附带图标（显示在文本右侧 +8px，默认缩放 0.5）。

### 5.2 上限与合批

- **同屏上限 100 条**：超出后非 `always_display` 的文字直接丢弃。玩家受伤/承伤、ONE_SHOT 提示标记 `always_display=true` 不受限。
- **伤害数字本身不合批**（每次命中一条）；**合批发生在掉落端**：材料超过 50 个时新材料并入已有材料增值（见 07 章），拾取增值材料时显示 `x{boosted}`（§5.3）。
- 属性增减图标类浮字按"玩家当前未结束的浮字数"向上偏移堆叠：位置 = 玩家位置 - (0,50) - n*(0,30)，浮字结束后计数回收。

### 5.3 颜色与内容规则

需开启设置 `damage_display`，否则单位伤害数字全部不显示（玩家文字走同一开关，亦被关闭）。

| 事件 | 文本 | 颜色 | 备注 |
|---|---|---|---|
| 敌人普通受伤 | 伤害值 | 白 | duration 0.5s |
| 敌人暴击 | 伤害值 | 黄 | |
| 护甲减伤起效 | 伤害值 | 灰 | |
| 一击必杀效果 | `ONE_SHOT`（翻译键） | 白 | always_display |
| 玩家受到伤害 | `-值` | 设置色 `color_negative`（默认红） | always_display |
| 玩家闪避 | `DODGE` | 白 | always_display，走翻译 |
| 玩家伤害被无效化 | `NULLIFIED` | 白 | always_display |
| 玩家治疗 | `+值` | 设置色 `color_positive`（默认绿） | 治疗事件触发 |
| 拾取增值材料 | `x{倍数}` | color_positive | 增值倍数 >1 时 |
| 升级 | `LEVEL_UP` | 白 | 玩家位置 |
| 敌人变异（curse 等状态变化） | `MUTATION` | 白 | |
| 特殊命中图标 | 伤害值+图标 | 按上规则 | 命中类型"暴击得金 / 诅咒击杀得金"附对应图标 |

### 5.4 属性/收获浮字（带图标）

- 属性增加：`+值` + 属性图标，**金色 `#76ff76`**，时长 `duration*2 = 1s`，并播随机正面音效（-3dB+修正）。
- 属性减少：`值`（自带负号）+ 图标，`color_negative`，播负面音效（-8dB）。
- 波末收获（harvesting）：同上，使用收获图标，方向改为向下 `(0,50)`。
- 炮台获得属性：显示在炮台位置 -50y。
- 忽略列表：管理器可配置不显示的属性清单。

### 5.5 商店浮字

- 重随获得属性（骰子类道具）：在重随按钮位置显示带图标浮字。
- 偷窃被抓：显示精英图标（无文本，图标原尺寸），单人方向 `(25,-100)`、合作 `(0,-30)`，播"被抓"音效。

## 6. 升级选择界面（波末）

单人与合作各挂一套升级界面（同一套逻辑，合作版标记为合作 UI）。

### 6.1 出现时机与处理顺序

- 波次正常结束（非输赢）时由清场结束计时器调起，传入待处理宝箱与升级队列。
- **每名玩家先逐个处理宝箱（consumables），再逐个处理升级**（宝箱队列非空优先）；处理一个 → 回调应用效果 → 刷新联动属性与属性栏 → 取下一个；全部处理完发"处理完毕"事件，界面隐藏，进商店。
- 顶部/角落仍显示"剩余待处理"图标队列，每处理一个移除一个。
- 合作模式四名玩家**并行**各自处理自己的队列（每人一个容器 + 独立焦点模拟）。

### 6.2 升级四选一

- 从升级池按等级抽 4 个升级（数量可能更少；选项面板按需隐藏）。每个选项是一个面板：描述 + CHOOSE 按钮，面板边框按 Tier 染色。
- **重随按钮**：首次取价用与商店相同的重随价格公式（见 04 章）；只有候选 >1 时显示；点击扣材料、重随计数+1、重新抽 4 个（排除上一组）。钱不够直接忽略（无动画）。
- 默认焦点：有第 2 个选项时聚焦第 2 个，否则第 1 个。
- 防连点：每次按钮按下后启动延迟计时器锁定。

### 6.3 宝箱（item box）处理

- 开箱得出道具；界面显示道具详情面板（边框按 Tier 染色）、三个按钮：
  - **Take**：加入道具；若该道具与已有装备可"重复合成"则按钮上叠加提示图标。
  - **Recycle(+回收值)**：按钮文本带回收金额；"详情"键可快捷触发（单人）；回收入账。
  - **Ban (x/y) (+回收值)**：需完成 ban 挑战且本局启用 ban；图标与进度条染 `color_negative`；`holding_button` 设置开时需长按（进度 0.025/物理帧 ≈0.67s）并播放收缩消失动画（0.2s 缩到 0.5/旋转/淡出），随后扣 ban 令牌、加入 banned 列表、给回收值。
- **宝箱额外道具效果**（如藏宝图）：开宝箱时按概率把额外道具压入"附加待展示队列"，依次弹出展示（随机或指定 id、value=1），并校验数量上限。
- 单人布局：屏幕中央选项 + 右侧常驻属性面板与属性浮窗；合作：每人一块面板，轮播第二页才是属性页。每次选择后属性面板实时刷新。
- 暂停后恢复时重新聚焦。

## 7. 暂停菜单（局内）

- 触发：局内任意玩家按下各自的暂停键；商店内同理。置全局暂停标志，记录暂停发起者（其手柄拥有菜单控制权）。失焦自动暂停由设置 `pause_on_focus_lost` 控制。
- 主页按钮：Resume / Codex / Restart / End run / Options / Quit。Restart、Quit、End run 均先切到二次确认页。
- 信息区：当前玩家的**属性面板**、武器清单 `WEAPONS (n/槽数)`、道具清单（可切换查看 banned 道具，banned 显示为半透明）、**波次时间线**、难度行 `难度名 - ENDLESS? - scaling 缩写 - WAVE n`。合作模式顶部有玩家切换轮播。
- **波次时间线**：显示 1..20 波刻度（无尽 ≥10 波后或波 >20 时改为 `当前±10` 窗口）。每 5 波或有事件的刻度放大并显示数字；精英/族群波显示精英/Horde 图标，第 20 波显示 Boss 图标（危险 5 显示双 Boss 图标）；已经过的刻度黄色 `(1,1,0)`，未到白色；精英波与第 20 波未过为 `color_negative`、已过为与基色 50% 插值；当前角色头像图标停在当前波刻度上方。
- 返回逻辑：取消键/暂停键在主页直接恢复游戏，在子页回主页。打开 Codex 期间屏蔽返回。商店内暂停会隐藏商店内容并立即保存运行快照。

## 8. 商店界面（布局与交互；经济公式见 04 章）

商店分单人版与合作版，共享同一套基础逻辑。

### 8.1 单人布局

- 标题 `MENU_SHOP (WAVE n)`。
- 左侧：4 个商品卡 + Reroll 按钮 + 金币 Label。
- 右侧：属性面板（主/副属性 Tab）+ 属性说明浮窗。
- 底部：玩家装备区（武器清单 `WEAPONS (n/槽)` + 道具清单，武器列数随武器数在 3..6 列自适应、道具列补满 12 列）、GO 按钮文本 `GO (WAVE n+1)`（拥有沙漏时 n+1-沙漏数）、下一精英波提示。
- 第 19 波且非无尽：显示"无尽模式"按钮，点击置无尽标记后等同 GO。
- 精英提示：文本 `ELITE_APPEARING/HORDE_APPEARING (wave)`，若下一波就是精英波则面板边框变灰；精英列表逐个列出未来精英/Horde 波图标+波数，下一波的条目底色用危险红样式。

### 8.2 商品卡

- 内容：道具详情（名称按 Tier 染色、类目行、效果行、武器属性富文本）、价格购买按钮（材料图标+价格；买不起时文字 `color_negative`）、锁定按钮、Ban 按钮、偷窃按钮（仅拥有偷窃次数时显示，文本 `MENU_STEAL xx%` 被抓概率，<1% 精确到 0.1）。
- 面板边框按 Tier 染色；锁定时边框变白。HP 商店效果（如 Bull 角色）价格除以 20 向上取整并把价格图标换成 HP 图标。可合成重复武器/可克隆道具时在价格旁叠加提示小图标。
- **购买**：先校验货币与武器可购规则（tier 上下限、近战/远程禁购、槽位满但可升级则允许直接合成、禁重复武器族），0.05s 防连点。买后商品卡失活（透明且禁用）。钱不够触发金币 Label 抖动（§13）。
- **锁定**：Lock 按钮或手柄"次要确认"键；写入该玩家的商店锁定状态；锁定项重随时保留并回填前几格。`keep_lock` 设置决定进店时是否恢复上次锁定。"禁止锁定"类效果隐藏锁按钮。
- **Ban**：仅道具（非武器）可 ban；按钮文本 `BAN_SHOP (剩余令牌)`；`holding_button` 开时长按充能（0.025/物理帧）→ 播 ban 动画 → 移除该商品、扣令牌（不给钱，与宝箱 ban 不同）、商品卡失活并刷新空店免费重随。渔夫角色的鱼饵道具不可 ban。
- **偷窃**：按偷窃按钮或"次要确认"键；被抓概率命中时在按钮处弹精英图标浮字并把随机精英加入下一波；无论是否被抓，商品价格按 0 结算入库。
- **悬停类目行**：武器显示协同浮窗（武器 Set 协同：每档 `(n) 效果…`，当前生效档全亮、未生效灰色 `#555555`）；道具显示标签说明（PET/STRUCTURE）。浮窗位置在商品卡右侧 +5px。

### 8.3 重随按钮

文本 `REROLL - 价格`（大写）；免费重随（道具或空店奖励）时价格显示 0；快捷键为"详情"键（单人），`holding_button` 开时需按住 0.5s 充能。重随逻辑（锁定保留、骰子属性、金鱼彩蛋音效、升 Tier 道具消耗等）随商店逻辑执行；钱不够时金币行抖动；重随后若买不起任何商品则焦点移到 GO。重随价旁可叠加"升 Tier 来源道具"小图标。

### 8.4 装备区交互与属性实时预览

- 焦点/悬停装备元素 → 弹出物品浮窗：道具详情 + 协同 + 标签 + **上波伤害行**（武器显示 `DAMAGE_DEALT_LAST_WAVE`，数字千分位格式化、绿色；工程炮台道具显示其追踪伤害）。
- 点击**武器**元素 → 物品浮窗进入按钮态（Combine 合成 / Recycle 回收 / Cancel），背景弹出阻挡层；Combine 仅当可合成时显示；"锁定当前武器"类效果隐藏合成与回收。合成逻辑（含诅咒武器合并取最大诅咒系数、DLC 诅咒道具回调）与回收逻辑均即时结算，播放对应音效并刷新商品颜色/属性面板。
- **属性面板实时刷新**：任何购买/合成/回收/重随后刷新属性面板；浏览商店元素时属性面板焦点被禁用避免抢焦点。
- 焦点丢失兜底：优先回到"可聚焦商品"（从上次聚焦的商品向两边搜索，先找买得起的），否则免费重随时回 Reroll、再否则 GO。

### 8.5 GO 与离店

- 单人按 GO 直接走；**合作需全员按 GO**：按下的玩家显示对勾，再按取消/扳机可撤回。全员就绪后：保存运行快照、结算锁定使用挑战、消耗沙漏（每个 -1 波并移除道具）、额外精英概率注入下一波 → `当前波 += 1 - 沙漏数` → 局内战斗。
- 离开商店界面时结算"锁定商品被诅咒"概率与保底 pity。
- 进店时机效果：`destroy_weapons`、`upgrade_random_weapon`（铁砧）等在"商店效果未结算"标记下执行一次。
- 从存档继续进店：恢复商品列表/重随计数/偷窃次数等。每次进店和暂停时保存运行快照。

### 8.6 合作商店

- 每名玩家一个面板：轮播第 0 页=自己的 4 商品 + 装备 + Reroll/GO，第 1 页=属性页（LT/RT 翻页）；金币图标/数字/重随图标染玩家色，面板边框玩家色。
- 2 人局加宽边距 75px；商品锁定按钮在合作中隐藏（用"次要确认"键），进店时商品视觉上对所有人清锁显示。
- 合作物品浮窗：聚焦商品时自动弹详情，含操作提示行（锁定/解锁/偷窃含概率/Ban 含剩余令牌/查看背包），按"详情"键可整体隐藏/恢复弹窗（提示文本 `COOP_POPUP_ENABLE/DISABLE_HINT`）。协同面板仅 ≤3 人时常显。
- 无尽按钮仅 P1 面板显示。
- **合作续档界面**：逐个让玩家按住加入（显示该玩家的装备、金币与彩色面板，进度条按住充能），全部加入后进合作商店。取消键退回标题画面。

## 12. 属性面板与文本格式化

### 12.1 属性面板

- 结构：标题 + Primary/Secondary 两个 Tab 按钮（手柄 LT/RT 切换）+ 通用区（等级行）+ 主属性列 + 副属性列；DLC 属性动态插入。默认最小高 780。
- **每行属性条目**：小图标 + 属性名 + 数值。颜色规则：值 >0 → `color_positive`；<0 → `color_negative`；=0 → 白；DLC 属性可有颜色覆写。**上限显示**：闪避显示 `值 | 上限`（闪避上限 < 值或 < 60 时）；HP/速度/暴击率有上限时同样 `值 | cap`。
- 副属性行支持"反向"着色（如敌速：越低越绿）；结构攻速行用武器数值系统实时计算。
- 聚焦/悬停属性行 → 属性浮窗显示图标+标题+动态说明（含玩家当前数值），位置自动避让；合作模式聚焦边框染玩家色。

### 12.2 文案格式化系统

- 统一入口 `text(key, args, arg_signs)`：翻译 key 后替换 `{i}` 占位。**符号染色**：正面 → `color_positive`，负面 → `color_negative`，覆写 → 诅咒紫 `#ca61ff`，中性不染。
- **正号规则**：维护一张"需要运算符的键表"（约 90 个 key），列出哪些 key 的哪个参数需要加运算符——值 ≥0 加 `+`，负值不加（自带 `-`）。若 key 文案没有 `{0}` 占位但参数 0 需要运算符，则自动把 `{0} ` 接在文案前（日语接在后）。
- **百分号规则**：另一张"需要百分号的键表"列出需要追加 `%` 的参数位。
- 千分位格式化：数字按千位逗号分隔（用于上波伤害显示）。
- 物品名称统一按 Tier 染色；类目行规则：武器=武器 Set 名；角色/升级/难度；道具按 PET/STRUCTURE/UNIQUE/`LIMITED (持有/上限)`/ITEM 组合；诅咒物品图标面板显示诅咒角标。

## 13. 通用 UI 组件与服务

- **弹窗管理器**：集中管理每名玩家的物品浮窗/属性浮窗；连接清单/属性/商店容器的悬停/聚焦/按下事件；元素被"按下"后弹窗进入按钮态并锁定（其间悬停不再切换），重置焦点解锁。
- **浮窗定位**：以附着控件为锚，依据其在父矩形上/下半与左/右半决定弹出方向，距离 10px，钳制在父矩形内，重叠时横向避让；父矩形未指定时取全屏 1920×1080。
- **全局确认弹窗**：常驻全局服务（如平台特定提示），挂在最顶层、暂停时仍处理；以（描述、确认文本、确认回调）三个参数唤起，确认后恢复之前的焦点。
- **界面特效服务**：
  - 输入设备侦测：根据输入事件判定键鼠/Xbox/PS/Switch 手柄并广播"设备切换"事件，所有按键图标控件据此换贴图。
  - 买不起抖动：播错误音、放大 1.1 倍、染 `color_negative`、5 次随机角度回摆（每次 0.03s）后复原。
  - 入场动画：清单元素入场 0→1 缩放 0.1s。
  - ban 动画：播 ban 音、染负色、0.2s 同时缩到 0.5/旋到 0.5rad/淡出到透明。
- **网格清单组件**：清单基础设施。同 id 非诅咒道具自动堆叠计数；锁定元素显示半透明锁图标；支持 banned 半透明显示；元素尺寸基准 96×96，≤80px 自动换小字体；焦点邻居自动布线（行末跨行滚动 / 行内循环两种模式）。
- **轮播组件**：合作面板翻页，箭头按钮或 LT/RT（手柄时箭头换成扳机图标且禁点击）。
- **焦点模拟体系**：为每名合作玩家模拟独立 UI 焦点（共享控件上可识别操作者是哪个玩家）。
- **文本省略按钮**：文本超宽自动省略号。另有 Tooltip、滑杆选项（值显示为百分比）、带音效滑杆（聚焦/改值音效）。
