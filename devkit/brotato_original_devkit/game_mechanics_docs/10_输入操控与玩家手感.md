# 10. 输入操控与玩家手感

> 本章覆盖输入映射、玩家移动、朝向与动画、武器索敌/手动瞄准、击退、拾取吸附、暂停、本地多人设备分配等全部"手感"相关设计。
>
> 全局前置约定（影响手感）：
> - 逻辑帧率固定 **60 物理帧/秒**；文中大量计时按"60 × 每帧时长"折算，即"以 60fps 帧为单位的增量"。
> - 俯视角玩法：无重力、无线性/角阻尼，所有移动完全由游戏逻辑驱动。
> - 基准分辨率 1920×1080，2D 等比拉伸、保持宽高比。
> - 默认鼠标光标为自定义贴图，热点 (3,3)。

## 2. 输入系统（全局服务）

输入系统为常驻全局服务，**暂停时仍持续运行**。

### 2.1 键鼠/手柄状态与鼠标显隐
每个输入事件：
- **本地合作且玩家数 >1 时鼠标永久隐藏**。
- 收到手柄按钮/有效摇杆事件或键盘事件 → 隐藏鼠标；收到鼠标移动/按键 → 显示鼠标。
- "正在使用手柄"标志：手柄事件 → true；键盘或鼠标事件 → false。"有效摇杆事件"判定为 `abs(轴值) > 摇杆死区(0.5)`。

### 2.2 摇杆死区
全局软件死区 `0.5`，与各动作绑定中的 0.5 一致。移动取值采用**圆形死区 + 重缩放**：向量长度 ≤0.5 输出零向量；>0.5 时按 `(len-0.5)/(1-0.5)` 重映射后保持方向。

### 2.3 手柄菜单导航回显（echo）
十字键与左摇杆在菜单中模拟方向导航：
- 首次触发延迟 **0.3s**，随后每 **0.09s** 重复一次。摇杆首次按下立即触发一次。
- 通过注入虚拟导航动作实现；若设备已被合作模式占用，动作名追加 `_设备号` 后缀。

### 2.4 动作按设备复制（本地多人核心）
启动时把输入映射中每个动作复制为 `动作名_0 … 动作名_7`（共 8 套）：手柄事件按设备号生成；物理设备 0 的手柄映射到虚拟设备 **6**，键盘映射到虚拟设备 **7**。键盘复制时死区取 0.5。

### 2.5 "用手柄移动"选项
设置 `movement_with_gamepad`（默认 true）。关闭时重载输入映射后，从 `button_move_*`、`analog_move_*`、`rjoy_*` 共 12 个动作中**删除所有手柄事件**，即手柄只能用于菜单，移动/瞄准只认键鼠。主机平台强制开启。

### 2.6 焦点丢失
窗口失焦发出"失去焦点"事件（配合设置 `pause_on_focus_lost`，默认 true 自动暂停）；`mute_on_focus_lost` 默认 false。手柄断开同样触发"失去焦点"事件。

> 本作**不提供手柄震动/rumble 功能**。

## 3. 玩家移动的逐帧设计

### 3.1 输入向量读取
玩家移动由专属移动行为组件处理。每物理帧读取：

```
analog = 左摇杆向量（analog_move_*）
button = 按键向量（button_move_*，WASD/方向键/十字键）
movement = analog if analog.length() > button.length() else button
```
（合作模式下动作名带 `_设备号` 后缀。）

- 向量读取自动处理对角归一（按键同时按下时输出长度 1 的对角向量）与摇杆圆形死区。
- **鼠标移动模式**（设置 `mouse_only`，默认 false，仅单人有效）：`movement = 鼠标全局坐标 - 玩家坐标`；当 |dx|<20 且 |dy|<20（最小移动距离 20），**或按住鼠标左键**时输出零向量（即左键=驻足）。

### 3.2 速度应用——瞬时达速、无加速度插值

每物理帧：

```
当前移动 = （可移动 ? 读取输入 : 零向量）
速度向量 = 移动输入分量 + 当帧击退分量
移动输入分量 = 当前移动.normalized() × 移动速度
移动速度 = max(0, 当前速度属性 + 常驻加成 + 衰减型加成)
```

结论（实现关键）：
- **没有加速度/惯性/插值**：输入向量被归一化，无论摇杆推多浅，移动方向上的速度恒为"移动速度"（像素/秒）；松开输入当帧速度归零。摇杆模拟量只影响"摇杆 vs 按键谁赢"的比较，不影响速度大小。
- 玩家基础速度 **450 px/s**。换算最终速度：`450 × (1 + capped(Speed%)/100)`（速度属性有上限封顶）。
- 常驻加成：波内效果用；衰减型加成：临时增/减速，正值以 **60 px/s²** 衰减回 0，负值（减速）以 **300 px/s²**（60×5）恢复回 0。施加减速有保护：当前已减速超过 `0.45×speed` 时新值除以 20，低于 `-0.7×speed` 直接忽略，且单次封底 `-0.9×speed`。

### 3.3 物理体与碰撞滑动
玩家是运动学刚体（位置由逻辑驱动而非物理模拟），物理材质摩擦 0：
- 每物理帧以容差 0.08 对 `速度×帧时长` 做碰撞试探，把可行位移直接加到全局坐标；若碰撞则取速度沿碰撞法线的滑动分量再试探一次并叠加——即**手写的单次斜面滑动**（行为等价于一次迭代的滑动移动）。
- 碰撞层/掩码：玩家层=2（player），掩码=130（层 2 player + 层 8 obstacles）→ 玩家之间互相阻挡、被地图边界/障碍阻挡，但不会被敌人挤位（敌人走刚体推挤逻辑）。
- 本体碰撞形状：圆 r=24，偏移 (0,-12)；受击判定区：圆 r=21。

敌人（对照）：保持动力学刚体，由物理回调设置线性速度，附带：出界位置钳制到当前区域矩形、即将越过移动目标点时直接吸附到目标并清零速度（防抖）。

### 3.4 移动锁定与波结束
- "移动锁定"状态下移动读取冻结返回上一帧值——仅敌人冲锋/宠物用，玩家不会被设置。
- 波结束清场：禁止移动、移动向量清零、停掉全部计时器、关闭物理处理与受击判定区——清场阶段玩家完全不可操作。
- 死亡：同样禁止移动并禁用受击判定区。

## 4. 朝向、翻转与移动动画

玩家动画每物理帧随移动更新：

1. **贴图翻转**：`movement.x > 0` → 所有动画子精灵与影子横向缩放取正；`movement.x < 0` → 取负。**`movement.x == 0` 时保持原朝向**（纯竖直移动不转身）。注意翻转的是角色动画精灵组（含外观道具贴图），不含武器（武器自行处理，§6.4）。
2. **动画播放速度**：
   - idle 动画：播放速率 = 1。
   - move 动画：播放速率 = `当前实速 ÷ 450`——移速越快腿越快，受减速/加速实时影响。
3. **状态切换**：idle 且 movement≠0 → 播 `move` 并启动跑步烟尘粒子；move 且 movement=0 → 播 `idle` 并停烟尘。
4. `move` 动画长 0.6s 循环，其中 t=0.1 与 t=0.4 各触发一次脚步音 → 随机脚步音，-6dB，音调随机 ±0.1。
5. **区域 1（深渊/水域）**：动画名替换为 `idle_swim/move_swim/landing_swim`，脚步音换为游泳音，烟尘换为游泳气泡。
6. 波开始空降：播放 `landing` 落地动画（播放速率重置 1）并实例化降落伞，结束后回 idle。
7. "原地不动/移动中"临时属性（如石像/激流道具）：movement 是否为零向量直接驱动对应临时属性的加卸与计时器。

敌人朝向：基类逻辑仅按"随移动镜像"开关翻转精灵横向缩放。

## 5. 击退/位移对操控的影响

### 5.1 通用击退管线
- 被击中时 `击退向量 = 击退方向 × 最终击退量`。方向混合：若攻击者是玩家，`0.2×命中盒方向 + 0.8×(玩家→受击者)方向`。
- 每物理帧击退速度叠加进速度向量：`速度 = 移动输入 + 击退向量 × (100 - 抗性×100)`。击退量量纲很小（如 15），×100 后才是 px/s 级别。
- **衰减**：每物理帧击退向量向零做 0.1 系数的线性插值，即指数衰减、每帧保留 90%（60fps 下半衰期约 6.6 帧）。击退不抢夺移动控制——玩家全程可输入，二者矢量相加。
- 死亡击退保底击退量 15。
- 击退穿透（knockback_piercing）对有抗性的目标补偿：`amount + amount×min(res,pierce)/(1-res)`。
- 负值击退=拉拽（如鞭类）：按与攻击者距离、双方体型半径、双方移动方向点积等计算最大可拉距离并削减；预期位移按 `|amount|×100/6` px 估算，拉拽同时给目标施加等量衰减型减速。

### 5.2 玩家
玩家受伤**完全不写入击退向量**，受伤事件中方向恒为零向量——**玩家受击没有击退/硬直，移动输入永不中断**。手感上玩家只有受击闪白（0.1s）+ 无敌帧（§8）。

## 6. 武器索敌与瞄准

### 6.1 目标感知（事件驱动，无周期扫描）
- 每把武器有一个索敌触发区域，半径 = `武器射程 + 200`（感知扩展量 200）——**索敌范围比射程大 200px**，用于提前转向。
- 敌人进入/离开通过区域进出事件维护"范围内目标"列表；目标死亡即时移除并清当前目标。

### 6.2 每物理帧的指向逻辑
非手动瞄准时：
- **未在攻击动作中**：在范围内目标中找**距离最近且大于 `min_range` 的敌人**（距离平方比较），把武器**瞬时**指向它（无旋转插值/角速度限制）。无目标时保持当前待机角。
- **攻击动作中**：锁定当前目标实时跟踪角度 + 本次散布角；目标失效则回退重新索敌。

### 6.3 开火条件
同时满足：
1. 冷却归零（冷却以 60fps 帧为单位，每帧递减 `60×帧时长`，仅非攻击中递减）；
2. `玩家移动向量 == 零` **或** 拥有"移动中可攻击"效果——该效果**默认值为 1**，即默认移动可攻击，仅个别角色（猎人等）置 0 后必须驻足；
3. 当前目标有效且距离 ∈ [min_range, max_range+50]，**或**处于手动瞄准（无视目标）。

满足则延迟一帧进入攻击行为（近战缓动突刺/横扫、远程生成弹丸+后坐缓动）。射击散布：`spread = rand(-(1-accuracy), +(1-accuracy))` 弧度，直接加在朝向角上；击退方向取武器当前朝向。

### 6.4 武器外观朝向
- 纵向翻转：朝向角在 (-90°, 90°) 视为朝右否则翻转。
- 待机角：玩家面向右时为待机角（挂点传入恒为 0），面向左时为 `PI - 待机角`。武器挂点坐标由武器数量决定；>6 把时按半径 `60+(n-6)×5` 圆周均布。

### 6.5 手动瞄准
手动瞄准判定每帧计算（带帧内缓存、帧末清空）：
- 设置 `manual_aim = true`（默认 false）：单人恒手动；合作模式下仅手柄玩家手动。
- 设置 `manual_aim_on_mouse_press = true`（默认 false）：**按住鼠标左键期间**手动（仅单人），或手柄玩家右摇杆长度平方 > 0.05（≈偏转 22.4%）期间手动。
- 两开关可叠加（主机上 `manual_aim` 即等同按压模式）。

手动瞄准时每物理帧：
- 手柄：右摇杆非零 → 武器朝向 = 手柄瞄准向量的角度。手柄瞄准向量每物理帧更新为右摇杆归一化向量，**松开摇杆保持最后方向**（初值 (1,0)）。
- 键鼠：武器朝向 = (鼠标全局位置 - 武器位置) 的角度。
- 开火不再要求有目标（§6.3 条件 3），射击距离参数取满射程 `max_range`（近战因此总是满距离横扫）。
- 与自动索敌切换是**逐帧无缝**的：手动瞄准标志每帧重算，按压式松开左键当帧即回自动索敌。

### 6.6 准星与光标
- 单人非清场阶段的手动瞄准视觉处理：
  - 手动瞄准 + 手柄：隐藏鼠标，显示世界内准星精灵，位置 = 玩家位置 + (0,-32) + `200 × 手柄瞄准向量`（准星距玩家 200）。
  - 手动瞄准 + 键鼠：显示鼠标并把光标贴图换成准星（热点 (35,35)）。
  - 仅 `manual_aim_on_mouse_press`（未按下）：保持鼠标可见以便随时按压；`mouse_only` 模式鼠标恒可见。
- 光标管理为常驻全局服务（暂停仍运行）：战斗内准星由局内逻辑处理；光标管理只负责在普通光标（热点 3,3）与手动瞄准光标（热点 35,35）之间切换并恢复普通光标。鼠标隐藏时跳过处理。

## 7. 拾取系统（材料/消耗品吸附）

### 7.1 两层触发区域

| 区域 | 形状 | 碰撞掩码 | 作用 |
|---|---|---|---|
| 吸附区 | 圆 r=150（基础） | 96（层 6 items + 层 7 gold） | 触发吸附 |
| 拾取区 | 圆 r=32，偏移 (0,-24) | 96 | 实际拾取 |

- 拾取范围属性：`r = max(30, 150 × (1 + PickupRange%/100))`（进场时应用）。拾取判定圈 r=32 固定不变。
- 某些效果会关闭金币吸附：清除两区域掩码中的 gold 层位。

### 7.2 吸附触发规则
进入吸附区时：
- 治疗类消耗品：仅当 `当前HP < 最大HP` 才吸附；材料（金币）总是吸附。
- 金币竞争：若本玩家比当前吸附者更近，则**抢走**吸附权（距离平方比较）。
- 进入拾取区时：吸附者为空或为自己才拾取（防止吃掉飞向队友的物品）。
- 玩家**受到伤害时**（dmg>0）主动吸附范围内全部消耗品（毒果除外）。

### 7.3 飞行动画参数
掉落物每物理帧三阶段：
1. **弹出**：向弹出目的地做指数趋近插值（速度系数 5/秒），直到距目标平方距离 ≤400（即 20px 内）；
2. **待机**：待机计数 10，按每帧 `60×帧时长` 递减（≈0.167s），期间恢复可被探测；
3. **吸附**：向玩家位置直线逼近，初速 **500 px/s**，每物理帧 +20（即 **+1200 px/s²** 加速）。吸附者死亡则速度重置 500 并停住。

拾取后发出拾取事件、重置回对象池。消耗品拾取叠加音效与各种"拾取消耗品时"效果。

## 8. 受击、无敌帧与输入

- 无敌帧时长：`iframes = clamp(伤害占最大HP比例 × MAX / 0.15, MIN, MAX)`，`MIN=0.2s`、`MAX=0.4s`，无尽模式按无尽强度系数同除。即受击 ≥15% 最大生命 → 满额 0.4s。
- 仅可闪避（dodgeable）伤害触发：禁用受击判定区 + 启动一次性无敌计时器，超时若未在清场则重新启用。`bypass_invincibility` 类伤害（如每秒扣血诅咒）无视无敌帧。
- **无敌帧/受击对输入零影响**：无硬直、无移动惩罚、无输入屏蔽；仅闪白（0.1s）+ 屏幕震动（设置 `screenshake` 默认 true）。
- 闪避（dodge）走同一管线，伤害归零仍触发无敌帧与音效。

## 9. 暂停与时间处理

- 触发：单人检测暂停键松开；合作模式遍历每个玩家的 `ui_pause_设备号`（主循环内轮询；暂停菜单自身处理暂停态下的解除）。
- 暂停时：置全局场景暂停标志、显示菜单、恢复默认光标、记录暂停发起者（菜单焦点归该玩家）。**不修改全局时间缩放**——暂停是纯场景级暂停。
- 暂停期间：输入系统与光标管理因"暂停时仍处理"标记继续运行（菜单导航 echo 保持可用；暂停时还会强制开启手柄 echo）。
- 解除：暂停态下该玩家的取消键或暂停键松开 → 子菜单先回退 → 解除暂停。解除后跳过主循环当帧的暂停轮询，防止同一次按键立刻重新暂停。
- 失焦自动暂停：设置 `pause_on_focus_lost` 默认 true；Switch 上手柄数不符也会强制弹暂停（解除前校验手柄数）。
- 合作模式中发起暂停的手柄掉线 → 暂停归属转移给 P0。

## 10. 本地多人输入设备分配

- 加入方式：大厅监听期间，任意设备**按住该设备的确认键连续 0.7s**即占用该设备加入；进度条按 `已按住时长/0.7` 上报。
- 设备号体系：物理设备 0 的手柄 → 虚拟设备 **6**；键盘 → 虚拟设备 **7**；其余手柄用原始设备号 1..5。一台设备只能绑一名玩家。
- 玩家类型识别：按手柄名称关键词分 Xbox/PS/Switch/键鼠，用于按键图标显示。
- 人数上限：4（Switch 为 2）。
- 进入战斗时把设备号写进各玩家的移动行为组件，移动/瞄准/UI 动作全部带 `_设备号` 后缀路由。
- 出生点：单人在地图中心；多人按 `中心±100px` 的 2×2 网格摆放。
- 设备断开：监听期自动移除该玩家；局内触发暂停（§9）。
- 多人时鼠标隐藏、单人手动瞄准 UI（准星）整体停用。
- 菜单焦点：合作 UI 用焦点模拟器按 `玩家索引 → 设备` 各自模拟焦点导航，暂停菜单焦点属于暂停发起者。

## 11. 输入相关设置项汇总

| 设置 | 默认 | 作用 |
|---|---|---|
| `mouse_only` | false | 鼠标走位模式（§3.1），鼠标恒显示 |
| `manual_aim` | false | 常驻手动瞄准（§6.5） |
| `manual_aim_on_mouse_press` | false | 按住左键/右摇杆时手动瞄准 |
| `movement_with_gamepad` | true | 关闭后手柄不能移动/瞄准（§2.5） |
| `pause_on_focus_lost` | true | 失焦自动暂停 |
| `mute_on_focus_lost` | false | 失焦静音 |
| `screenshake` | true | 受击/爆炸屏震开关 |
| `holding_button` | true | 商店等界面长按确认 |
| `keep_lock` | true | 商店锁定保留 |
| `character_highlighting` 等 | false | 角色/武器/弹幕描边高亮（辅助功能） |

无鼠标确认专属选项（菜单原生支持鼠标点击 + 键盘/手柄焦点导航并存）；无手柄震动选项（功能不存在）。

## 12. 实现自检清单（手感关键点速查）

1. 移动**瞬时达速/瞬时停止**，速度 = `450×(1+Speed%) + 常驻加成 + 衰减型加成`，方向恒归一化——摇杆没有模拟速度。
2. 对角移动不超速（输入向量限长 1）；键鼠/摇杆按"谁的向量更长谁生效"竞争。
3. 玩家是运动学刚体：碰撞试探 + 一次斜面滑动，容差 0.08；只与障碍层、其他玩家相碰。
4. 击退与移动**矢量相加**、每帧 ×0.9 衰减；玩家自身免疫受击击退。
5. move 动画 0.6s 循环，播放速率 = 实速/450；x 速度符号驱动整组贴图镜像，竖直移动不转身。
6. 武器索敌：感知半径 = 射程+200，事件式维护列表，每物理帧选"≥min_range 的最近敌"，朝向瞬时对准（无平滑）；开火还需距离 ≤ 射程+50。
7. 冷却、物品待机时间等大量逻辑以 `60×帧时长` 帧单位递减——务必按 60 物理帧实现。
8. 手动瞄准是逐帧开关（按压式松手即回自动），手柄准星距玩家 200px、锚点在玩家头顶 (0,-32)。
9. 吸附动画：弹出指数趋近（×5/s）→ 待机 0.167s → 500px/s 起步、+1200px/s² 直线追踪。
10. 暂停 = 纯场景级暂停，不动时间缩放；解除后跳过一帧暂停检测。
