# 08 效果系统（Effect System）

> 本篇规定效果系统的完整设计：效果条目的数据结构与存储方式、54 种效果类型逐条说明、效果键全清单、燃烧机制完整公式、触发型效果的事件钩子，以及持续型效果调度器。

本游戏中**一切角色/物品/武器/消耗品的数值与特殊能力都通过"效果"实现**。物品被购买时，其全部效果条目逐个"施加"，把数值写入每个玩家独立的**效果表**（一个 key→值 的字典）；出售/移除时反向"撤销"。游戏运行时各系统（武器、玩家、敌人、波次管理）按 key 读取该表并执行对应逻辑。

---

## 1. 系统架构

### 1.1 效果条目的基础数据结构

| 字段 | 类型 | 说明 |
|---|---|---|
| `key` | String | 效果主键（写入效果表的目标 key） |
| `text_key` | String | 描述文本翻译键（缺省用 key 大写） |
| `value` | int | 效果主数值 |
| `custom_key` | String | KEY_VALUE 类存储时的"容器键"（真正写入的字典 key） |
| `storage_method` | 枚举 | 存储方式，见 1.2 |
| `effect_sign` | 枚举 | UI 着色：POSITIVE/NEGATIVE/NEUTRAL/FROM_VALUE/FROM_ARG/OVERRIDE |
| `custom_args` | 参数数组 | 描述文本参数定制（取值/格式/正负性三类枚举） |
| `curse_factor` | float | DLC 诅咒强化系数，随效果一起存档 |
| `base_value` | int | REPLACE 存储方式下记录被覆盖前的旧值，供撤销时还原 |

### 1.2 存储方式（StorageMethod）—— 施加/撤销的五种写法

```text
SUM:              效果表[key] += value            （撤销: -= value）
KEY_VALUE:        效果表[custom_key] 中查找 [key, v]，找到则 v += value，否则 push [key, value]
                  （撤销: v -= value，减到 0 时移除该条目）
REPLACE:          base_value = 效果表[key]; 效果表[key] = value （撤销时还原 base_value）
APPEND_KEY:       效果表[custom_key].append(key)，已存在则不重复（撤销: 移除）
APPEND_KEY_VALUE: 效果表[custom_key].append([key, value])（撤销: 删除第一个匹配项）
```
每次施加/撤销后，该玩家的属性缓存全部失效，下次读取时重算。

### 1.3 存档规则

- 每个效果条目存档时输出 `{效果类型 id, key, custom_key, text_key, storage_method, value, effect_sign, base_value, curse_factor, custom_args}`；每种效果类型追加自有字段并声明唯一类型 id。
- 读档时按"效果类型 id"在全部效果类型注册表中匹配并还原合并；结构物效果同理。
- 读档恢复时为避免重复实例，武器效果先按"序列化数据哈希"缓存复用。
- 需要**整对象完整存档**的 key 共 14 个：`burn_chance, structures, explode_on_hit, explode_when_below_hp, explode_on_death, explode_on_consumable, convert_stats_end_of_wave, convert_stats_half_wave, modify_every_x_projectile, explode_on_overkill, explode_on_consumable_burning, convert_bonus_gold, scale_materials_with_distance, pets`；内含武器面板数据的 key 共 2 个：`projectiles_on_death, alien_eyes`。

### 1.4 应用入口

- **施加物品效果**：遍历物品的效果列表逐个施加；若是普通物品（非升级）且效果是属性 key，记录前后差值，正增量时触发 `gain_stat_for_equipped_item_with_stat` 联动（如 Snowball）。
- 对应的撤销入口与批量施加/撤销入口成对存在。
- 添加/移除物品或武器后重算 4 类"条件武器加成"（先减旧、再加新）：
  - `unique_weapon_effects`：每种**不同**武器一份
  - `additional_weapon_effects`：每把武器一份
  - `tier_iv_weapon_effects`：每把 ≥传说 武器一份
  - `tier_i_weapon_effects`：每把 ≤普通 武器一份
- 武器套装：先撤销全部旧套装效果，再统计同套装武器数（`all_weapons_count_for_sets` 生效时每个套装按武器总数计），≥2 时施加该套装第 `min(n-2, 最高档)` 档的效果。
- 难度效果：开局对所选难度的效果列表逐一施加。
- 消耗品：拾取时按物品效果通道施加。

### 1.5 属性读取管线

- `属性(key) = 效果表[key] × 属性增益(key)`，其中 `属性增益 = 1 + 效果表["gain_"+属性]/100`——即 `gain_stat_*` 系列 key 是**属性增益百分比修正**。
- 最终属性 = 永久属性（上式） + 临时属性层 + 联动属性层（结果带缓存）。
- 受上限属性约束的读取：max_hp/speed/dodge/crit_chance 分别受 `hp_cap / speed_cap / dodge_cap / crit_chance_cap` 上限约束。
- 直接增减属性的接口会修改效果表并广播"属性增加/移除"事件。

## 2. 全部效果类型清单

效果类型共 **54 种**（基类 1 + 物品类 36 + 武器类 15 + 消耗品类 2），另有数据结构 2 种（燃烧数据、燃烧类型）、文本参数枚举 4 种（取值/格式/正负性/存储方式），以及"行为效果"的数据包装结构。

### 2.1 通用/容器类

| 效果类型 | 类型 id | 字段 | 机制 |
|---|---|---|---|
| 基础效果 | `effect` | 见 1.1 | 按存储方式写入效果表 |
| 武器空效果 | `weapon_null` | — | 施加/撤销均为空操作；武器自带效果不进玩家效果表，由武器结算时自行读取 |
| 武器双数值空效果 | `null_double_value` | `value2: float` | 仅承载双数值供武器逻辑/文本使用 |
| 双数值效果 | `double_value` | `value2`, `sum_strategy{SUM_BOTH, SUM_VALUE_1, SUM_VALUE_2}` | KEY_VALUE：向 `效果表[custom_key]` push `[key, value, value2]`；SUM：在 `效果表[key]` 数组中按策略合并 `[value, value2]` 对（SUM_VALUE_1 = value2 相同则 value 累加，依此类推），减到 0 移除 |
| 双键值效果 | `double_key_value` | `key2`, `value2` | 向 `效果表[custom_key]` 合并 `[key, value, key2, value2]`，相同 (key,key2,value2) 的条目 value 叠加 |
| 子效果容器 | `effect_with_sub_effects` | `sub_effects: Array` | 把**自身对象** append 到 `效果表[key]`；消费方遍历子效果（如 `convert_bonus_gold`：每攒满 value 奖励金币就施加一次全部子效果） |
| 武器版子效果容器 | `weapon_effect_with_sub_effects` | 同上 | 施加为空操作（武器版，由武器数值规则读取，如 `modify_every_x_projectile`） |
| 线性缩放效果 | `linear_scaling_effect` | `min_value, max_range, buffer, invert_scaling` | `效果表[key] = [self]`（REPLACE 式独占）。取值公式：`x-=buffer; slope=(|min|+|value|)/max_range; pct=clamp(min_value+slope*x, min_value, value)`，invert 时为 `value-slope*x`。用于按输入量线性插值的百分比效果 |

### 2.2 属性转换/操控类

| 效果类型 | 类型 id | 机制 |
|---|---|---|
| 属性转换 | `convert_stat` | 把**自身** push 进 `效果表[custom_key]`（`convert_stats_end_of_wave` 或 `convert_stats_half_wave`）。结算公式：`chunks = floor(源值×pct_converted/100 / value)`，扣 `chunks×value`（按属性增益反折算永久点数），加 `chunks×to_value`；源可为 `materials`（金币）。波末版为永久属性，半波版写入临时属性层 |
| 最高最低属性互换 | `swap_max_min_stat` | 找出 16 个主属性中数值最高/最低（>0）的两个，互换其**永久部分**：`新永久值 = (对方总值 - 本方临时 - 本方联动) / 本方增益`；撤销为空操作（一次性，不可逆） |
| 属性上限 | `stat_cap` | `效果表[key] = value`，或"取当前属性值为上限"模式下设为该属性当前值；撤销时恢复为极大数。配合 hp_cap/speed_cap/dodge_cap/crit_chance_cap |
| 属性增益修正 | `stat_gains_modifications` | 对所列每个属性执行 `效果表["gain_"+属性] += value`，即修改属性增益倍率（见 1.5） |
| 按属性联动加属性 | `gain_stat_for_every_stat` | 向 `效果表[stat_links]` push `[key, value, 缩放源, 每N点, 仅永久]`。联动属性层每次重算时 `bonus = int(value × (被缩放量 / 每N点))` 加到 key 上；被缩放量支持：材料、结构数、宠物数、存活/燃烧敌人数、不同/普通/传说/重复物品数、特定物品数、存活树、缺失生命%、空闲武器槽、任意属性（"仅永久"决定是否含临时属性）。UI 展示用同一公式 |
| 武器版属性联动 | `weapon_gain_stat_for_every_stat` | 武器版，施加为空操作；`increased_stat_name` 指明加到武器哪个属性上，在武器面板初始化时按同公式直接加到武器面板 |
| 武器职业加成 | `class_bonus` | 向 `效果表[weapon_class_bonus]` push `[套装 id, 属性, value]`：给某武器职业（套装）全部武器加属性 |
| 武器类型加成 | `weapon_type_bonus` | 向 `效果表[weapon_type_bonus]` push `[近战/远程, 属性, value]`：按近战/远程类型加武器属性 |
| 武器堆叠 | `weapon_stack` | 武器自带（空效果派生）：持有 N 把同堆叠 id 武器时每把额外 +`value` 的指定属性（文本显示 `(N-1)×value`，结算在武器初始化中） |
| 武器冷却界限 | `weapon_cooldown` | `效果表[key]` 追加 value 并排序；`minimum_weapon_cooldowns` 取**最大**值为冷却下限，`maximum_weapon_cooldowns` 取**最小**值为上限；结算时：总冷却（帧）低于下限则补足差额，高于上限则削减 |

### 2.3 触发型（数值对容器）类

| 效果类型 | 类型 id | 机制 |
|---|---|---|
| 概率属性伤害 | `chance_stat_damage` | 向 `效果表[custom_key]` push `[属性 key, value(%), chance(%), 追踪 key]`。custom_key 决定触发点：`dmg_when_pickup_gold / dmg_when_death / dmg_when_heal`，统一结算：对**随机一名敌人**造成 `round(floor(max(1, value%×该属性)) × (1+%伤害))` 伤害，逐条按 chance 独立判定 |
| 百分比易伤 | `percent_damage` | 向 `效果表[enemy_percent_damage_taken]` push `[来源 id, key(关联属性), value(%易伤), 持续秒数, 最大层数, 最大触发次数, 描边颜色]`；由敌人侧"易伤标记"行为消费（见 §5） |
| 武器版百分比易伤 | `weapon_percent_damage_effect` | 同上但施加为空操作：挂在武器攻击判定区的效果列表上，命中时由行为读取 |
| 间隔临时属性 | `temp_stats_per_interval` | 向 `效果表[custom_key]` 合并 `[key, value, 间隔, 受击重置]`。玩家每秒计数，`计数 % 间隔 == 0` 时向临时属性层加 `[key, value]`；"受击重置"累积的部分在受击时一次性移除 |
| 武器每 N 杀加属性 | `weapon_gain_stat_every_killed_enemies` | 武器自带：该武器本波击杀数 `% value == 0` 时获得 `stat_nb` 点指定属性 |
| 燃烧击杀计数 | `gain_stat_for_kill_enemies_while_burning` | 向 `效果表[custom_key]` push `[key, value(每N个), value2(获得量), value3(每波上限), 计数, 已得次数]`。敌人**处于燃烧状态时被击杀**则计数+1，每满 value 个且未达 value3 上限时获得 value2 点 key 属性；计数在波开始清零 |
| 魅惑 | `charm` | 双数值效果派生，KEY_VALUE 存入 `charm_on_hit`：`[key(缩放属性), value(%), value2]`，魅惑持续 8 秒（全局常量）；敌人侧的魅惑状态与来源记录由 DLC 注入的行为处理 |
| 武器版魅惑 | `null_charm` | 魅惑效果的武器版，施加为空操作 |
| 斩杀命中 | `weapon_one_shot_on_hit` | 武器自带：命中时 `value%` 概率直接斩杀（伤害提到目标当前生命），带专属音效/粒子 |
| 缺失生命加伤 | `weapon_player_health_stat` | 武器自带（key=stat_damage 时）：命中伤害 += `round(value × 缺失生命% / for_every_health_percent)` |
| 命中衍生弹 | `weapon_projectiles_on_hit` | 武器自带：命中时从受击者位置发射 `value` 枚按所配武器面板结算的投射物，方向随机或自动索敌 |
| 死亡投射物/眼球 | `projectile` | 物品版：`效果表[key]`（`projectiles_on_death` 或 `alien_eyes`）首次写 `[数量, 武器面板, 自动索敌, 冷却]`；再次施加时数量累加，且缩放属性按投射物数量**加权平均合并**：`new = (old×n_old + add×n_new)/(n_old+n_new)`。`projectiles_on_death` 在敌人死亡时发射，`alien_eyes` 按冷却周期发射 |
| 武器命中减速 | `weapon_slow_on_hit` | 武器自带：减速% = `-max(缩放属性值,0) × value`；物品版数据存 `slow_on_hit` 数组 `[属性, value]`，武器缩放含该属性即把命中减速量减去 value；燃烧 tick 也触发元素减速 |
| 区域减速标记 | `weapon_slow_in_zone` | 标记类（无参数）：武器生成减速区域 |
| 额外敌群 | `extra_enemy` | 向 `效果表[key]`（如 `extra_enemies_next_wave`）按敌群配置标识合并 `[敌群, value]`，下一波多刷该敌群 |

### 2.4 爆炸类

| 效果类型 | 类型 id | 机制 |
|---|---|---|
| 武器爆炸 | `weapon_exploding` | 武器自带：字段 `chance(0-1), 爆炸表现, 缩放, 基础烟雾量, 音量修正`。武器命中时按 chance 在命中点爆炸，伤害=攻击判定区伤害。爆炸实例从对象池取用 |
| 物品爆炸 | `item_exploding` | 把**自身** push 进 `效果表[key]`（`explode_on_hit / explode_on_death / explode_on_consumable / explode_on_overkill` 等）。自带 `武器面板数据`、`追踪 key`、`随缺失生命缩放`（true 时伤害改为 `value × 缺失生命点数`） |
| 物品爆燃 | `item_exploding_and_burn` | 上者 + 燃烧数据：结算时把缩放后的燃烧数据写入爆炸面板，爆炸可点燃 |
| 低血爆炸 | `item_exploding_when_below_threshold` | + `hp_threshold`(%)：受击后生命 ≤ 上限×阈值% 时爆炸，**每波只触发一次** |

**通用爆炸结算**：同 key 下多个效果 → `总概率 = Σchance`（一次判定），`总伤害 = Σ各效果爆炸伤害`，其余参数（暴击、精度、燃烧）取第一个效果；爆炸伤害公式 `max(1, round((damage + Σ缩放属性) × (1 + %伤害 + 爆炸伤害%)))`。玩家受击爆炸（explode_on_hit）则每个效果独立判定 chance 并在自身周围随机偏移爆炸。

## 3. 燃烧（Burning）机制完整公式

### 3.1 数据合成（武器/结构/宠物面板初始化时）

把**武器自带燃烧**与**全局 burn_chance**（效果表中的燃烧数据对象）合成：

```text
若 全局与武器均无燃烧 → 返回空燃烧数据
若 base.is_global_burn（物品爆燃等已标记全局）:
    damage = 伤害加成(damage + Σ(缩放属性))                     # 即 ×(1+%伤害)
    返回（不再叠加全局）
若 武器无自带燃烧:  使用全局 burn_chance 的 chance/duration/damage/spread/缩放，is_global_burn=true
否则（武器有燃烧）:  damage += 全局.damage（chance/duration/spread 用武器自身）
最终:
    damage = max(1, damage + Σ(属性[i] × 系数[i]))              # 缩放属性折入伤害
    damage = round(damage × (1 + %伤害/100))                    # 结构(非宠物)用 structure_percent_damage
    spread += 效果表[burning_spread]
```
宠物另过"驯兽师"缩放。默认缩放：元素燃烧 `[[stat_elemental_damage, 1.0]]`；工程燃烧（火焰炮塔）`[[stat_engineering, 0.33]]`。

### 3.2 点燃判定

命中时：攻击带燃烧数据 且 `randf() <= chance`（**不吃幸运**）且非治疗型攻击 且 `效果表[can_burn_enemies] > 0` → 施加燃烧。

### 3.3 叠加/刷新规则

```text
已在燃烧:  若 新damage ≥ 旧damage → 接管来源(攻击来源/玩家编号)与缩放属性
           chance/damage/duration/spread 全部取 max(旧, 新)     # 不叠加，只取强
未燃烧:    记录燃烧数据副本; 启动燃烧计时; 全局燃烧敌人数 += 1
```

### 3.4 伤害 tick

- **基础 tick 间隔 0.5 秒**，生成时修正：`间隔 = max(0.1, 0.5 × (1 + burning_cooldown_increase% − burning_cooldown_reduction%))`（全玩家求和）。
- 每 tick：造成燃烧伤害（不可闪避、无视护甲）；`duration -= 1`，≤0 时停止燃烧。即**总伤害 = damage × duration**，默认持续时间 = duration × 0.5 秒。
- 附加结算：触发敌人侧"被点燃"行为钩子（易伤上 buff）；`slow_on_hit` 含元素伤条目时每 tick 减速；希腊火 `burning_enemy_hp_percent_damage`：燃烧伤害额外 + `当前生命 × Σ(value/100) / 无尽衰减系数`；燃烧来源武器收到"命中"回调（吸血等）。
- 关联效果：`burning_enemy_speed`（燃烧中敌人移速%）；`crit_on_hitting_burning_target`（武器对燃烧目标必暴击）；`bonus_non_elemental_damage_against_burning_targets`（对燃烧目标的非元素武器加伤）；`landmines_on_death_chance` 元素分支（死于燃烧时掉地雷）。

### 3.5 蔓延（Spread）

燃烧敌人身上的蔓延判定区（`spread > 0` 时启用）：每物理帧检查接触的**未燃烧活体**，找到第一个则把 `spread - 1` 后的燃烧数据传给它，随后本体 spread 清 0 并关闭蔓延区——即**一个火源总共可链式传染 spread 次，每次接触传 1 个**。被传染者继承相同 damage/duration 与剩余 spread。

## 5. 持续型效果调度器（行为效果）

**结构**：单位行为效果基类（钩子：`出生时是否挂载 / 金币价值修正 / 受击 / 受到伤害 / 死亡`）→ 敌人行为效果（+`被点燃 / 附加伤害 / 目标更新` 钩子）和玩家行为效果（+`移动` 钩子）；另有场景级行为效果，持有实体生成调度与波次管理的引用。

**注册与更新**：
- 调度器以配置数组注册 `场景行为 / 敌人行为 / 玩家行为`（本体只注册了"易伤标记"一个敌人行为，DLC 可追加）。
- 每波开始刷新激活列表：对每个行为实例化一次询问"出生时是否挂载"，通过的留在激活列表，之后**每个新刷的敌人**在初始化时挂载这些行为；玩家出生时同理挂载玩家行为。
- 单位在受击/受伤/死亡/燃烧/移动时遍历自身挂载的行为调用对应钩子。

**唯一内置行为"易伤标记"**（消费 `enemy_percent_damage_taken`，如蓝图浆果类物品）：
- 挂载条件：任一玩家 `enemy_percent_damage_taken` 非空，或任一武器带该效果。
- 受击/被点燃钩子：收集玩家效果表与攻击判定区上的易伤数据，**要求攻击的缩放属性包含效果关联属性**（或"全属性"）才生效。
- 叠层规则：按来源 id 维护激活条目——已存在则取 `max(层数上限/持续时间/易伤%)`、刷新计时，未达最大层数才 +1 层并累计易伤；最大触发次数 ≠ -1 时该来源在此敌人身上的总触发次数封顶；超时一次性移除该来源全部叠层。
- 结算：附加伤害钩子返回累计易伤百分比，加入敌人受伤计算。

## 6. 概率型效果与幸运（Luck）

- 通用概率判定：`chance==0 → false`，否则 `randf() <= chance`——**该判定本身不含幸运修正**。
- 因此**效果类概率（燃烧 chance、爆炸 chance、heal_on_kill、dmg_on_dodge、gold_on_crit_kill、charm、extra_item_in_crate 等）一律不受幸运影响**，只受效果数值本身（多件叠加提升 chance）影响。
- 幸运的作用点在**掉落与商店**：
  - 消耗品掉落：`drop_chance = min(1, base_drop_chance × (1 + Σ全玩家 luck/100))`，无尽波再除以 `(1+无尽系数)`；箱子掉率与心愿标签出现率同样受幸运/相关效果（`crate_chance`、`stat_boosted_wanted_item_tag`）影响。
  - 商店物品品质权重按幸运调整。
- 暴击是独立通道：`crit_chance = 武器基础 + 受上限约束的暴击属性/100`，判定在单位受击结算中。

## 7. 效果叠加规则汇总

| 场景 | 规则 |
|---|---|
| 同 key 标量效果（SUM） | 直接累加（多件同物品 = 多次施加线性叠加） |
| KEY_VALUE 容器 | 相同 key 的条目 value 合并累加；减到 0 删除 |
| 全局燃烧 `burn_chance` | 多件叠加：chance、damage、spread **相加**，duration **取最大** |
| 敌人身上的燃烧 | 不叠加：四项各取 max（新伤害≥旧时换源） |
| 爆炸容器（explode_on_*） | 概率 Σchance 一次判定，伤害 Σdamage，其余取首个效果 |
| 死亡投射物/眼球 | 数量累加；缩放属性按数量加权平均 |
| 百分比易伤 | 同来源 id 叠层至最大层数，参数取 max，受最大触发次数限制 |
| 武器冷却上下限 | 下限取所有 `minimum_weapon_cooldowns` 的最大值，上限取 `maximum_weapon_cooldowns` 的最小值 |
| 条件武器加成（unique/additional/tier） | 每次武器变动整组撤销重算，份数 = 满足条件武器数 × 效果值 |
| 套装加成 | 永远只激活当前档位（n-2 索引），换档先全撤销 |
| `gain_stat_*` 增益修正 | 多件相加后作为统一乘数 `(1+Σ/100)` 作用于该属性全部永久点数 |
| 属性联动（stat_links） | 每条独立计算 `int(value × 量/每N点)` 后累加进联动属性层 |
| 临时属性层 | 波内累计，波末一次性全清 |

## 8. 速查：三个最复杂机制

1. **燃烧**：全局/武器双源合成（§3.1）→ 命中概率点燃（不吃运）→ 敌身单实例 max 合并 → 0.5s tick（受 burning_cooldown_* 修正）× duration 次 → spread 链式传染。
2. **百分比易伤标记**：物品/武器效果 → 玩家效果表数组 → 敌人侧行为按来源 id 叠层/限触发/超时清除 → 受伤时加成。
3. **属性联动 / 转换 / 互换**：永久(×增益) + 临时 + 联动三层取值；转换与互换均需按属性增益反折算永久点数，避免增益修正循环放大。
