## 1. 系统总览

武器系统由以下子模块组成：

| 模块 | 职责 |
|---|---|
| 武器实体 | 战场中的武器对象：索敌、冷却、开火、命中回调 |
| 武器类型 | 两大类型：近战（MELEE）/ 远程（RANGED） |
| 武器数值面板 | 武器面板属性数据（冷却、伤害、暴击等全部战斗数值） |
| 武器元数据 | 武器作为"商店物品"的包装：价格、等级、套装、升级链 |
| 攻击行为规则 | 近战挥击/突刺动画与判定、远程弹道发射；攻击总时长计算 |
| 武器结算规则 | 把"基础数值 + 玩家属性 + 物品效果"折算成实战数值；弹道生成；爆炸生成 |
| 弹道规则 | 弹道移动、穿透、反弹、爆炸、特殊弹道行为 |
| 命中判定区 | 携带伤害/暴击/击退/燃烧等参数的触发区域 |
| 套装数据 | 武器类别与套装加成数据（共 15 套） |
| 合成/价格规则 | 同名同级合成、商店定价、回收价 |

**武器数据组织**：每把武器按"武器 × 等级"拆分为独立数据档，每个等级各有一份武器元数据（含价格/套装/特效/升级目标）和一份武器数值面板（含全部战斗数值），部分武器另有燃烧数据或附属弹道数据。元数据的 `upgrades_into` 指向下一等级，构成升级链。

## 2. 武器数据结构

### 2.1 武器元数据

| 字段 | 类型/示例 | 说明 |
|---|---|---|
| `my_id` | `"weapon_knife_1"` | 唯一 ID（带等级后缀） |
| `weapon_id` | `"weapon_knife"` | 武器族 ID（各等级共享，用于合成判定/堆叠判定） |
| `name` | `"WEAPON_KNIFE"` | 本地化键；展示名 = 名称 + 等级罗马数字（I 级不显示数字） |
| `tier` | 0~3 | 内部 0 起；对应玩家可见 Tier I~IV |
| `value` | int | 基础价格（商店价/回收价的基数，见 §8.4） |
| `type` | 近战 / 远程 | 武器类型 |
| `sets` | 套装数据列表 | 所属武器类别（1~2 个） |
| `scene` | 表现资源 | 武器的外观与动画表现配置 |
| `stats` | 武器数值面板 | 该等级的战斗数值 |
| `upgrades_into` | 武器元数据引用 | 合成后的下一等级；IV 级为空 |
| `effects` | 效果列表 | 武器自带特效（燃烧、爆炸、结构、击杀成长等，见 §11） |
| `is_cursed` / `curse_factor` | bool / float | 诅咒武器标记（DLC 诅咒系统入口） |
| `tracking_text` / `tracked_value` | String / int | 面板上的累计统计（如"已获得 X 金币"），合成时累加继承 |
| `add_to_chars_as_starting` | Array | 作为某角色初始可选武器 |

### 2.2 武器数值面板（公共字段）

| 字段 | 默认值 | 说明 |
|---|---|---|
| `cooldown` | 60 | 冷却，单位帧（60 帧 = 1 秒） |
| `damage` | 1 | 基础伤害 |
| `accuracy` | 1.0 | 精准度 0~1；开火时附加随机偏角 `随机(-1+accuracy, 1-accuracy)` 弧度 |
| `crit_chance` | 0.03 | 暴击率（0~1） |
| `crit_damage` | 1.5 | 暴击倍率 |
| `min_range` / `max_range` | 0 / 150 | 最小/最大射程（像素） |
| `knockback` | 0 | 击退力度 |
| `knockback_piercing` | 0.0 | 击退穿透（0~1，无视敌人 `knockback_resistance` 的比例） |
| `can_have_positive_knockback` / `can_have_negative_knockback` | true / false | 击退方向许可（负击退=吸引） |
| `effect_scale` | 1.0 | 命中特效缩放（也传给命中判定区） |
| `scaling_stats` | `[["stat_melee_damage",1.0]]` | **伤害成长数组**：`[属性, 系数]` 列表，伤害=基础+Σ(属性值×系数)，见 §3.5 |
| `lifesteal` | 0.0 | 吸血概率（每次命中按概率回 1 HP，见 §7.4） |
| `shooting_sounds` / `sound_db_mod` | — / -5 | 攻击音效 |
| `is_healing` | false | 治疗型命中（命中玩家时回血而非伤害且不会点燃） |
| `custom_on_cooldown_sprite` | null | 长换弹期间替换贴图（左轮空弹巢、弩空弦等） |
| `recoil` / `recoil_duration` | 25 / 0.1 | 后坐位移（像素）/时长（秒），同时决定远程攻击动画时长 |
| `additional_cooldown_every_x_shots` | -1 | 每 X 发触发一次"大换弹"；-1 = 无 |
| `additional_cooldown_multiplier` | -1.0 | 大换弹时冷却倍率 |
| `speed_percent_modifier` | 0 | 命中减速百分比（负值减速，命中时对目标施加衰减型减速） |
| `burning_data` | 燃烧参数 | 含点燃概率/每跳伤害/跳数/蔓延/成长系数 |
| `attack_speed_mod` | 0.0 | 由类别加成物品写入的武器级攻速修正 |

### 2.3 近战数值面板（附加字段）

| 字段 | 说明 |
|---|---|
| `attack_type` | 攻击方式：THRUST(0) 突刺 / SWEEP(1) 挥击 |
| `alternate_attack_type` | true 时每次攻击在突刺/挥击间交替 |
| `deal_dmg_on_return` | true 时收回阶段命中判定区不关闭（回程也能打人） |

### 2.4 远程数值面板（附加字段）

| 字段 | 默认值 | 说明 |
|---|---|---|
| `nb_projectiles` | 1 | 每次开火的弹道数 |
| `projectile_spread` | 0.0 | 散射角（弧度，单侧；实际每发取 `随机(朝向-spread, 朝向+spread)`） |
| `piercing` | 0 | 穿透次数 |
| `piercing_dmg_reduction` | 0.5 | 每次穿透后的伤害衰减比例 |
| `bounce` | 0 | 反弹次数 |
| `bounce_dmg_reduction` | 0.5 | 每次反弹后的伤害衰减比例 |
| `can_bounce` | true | false 则任何来源的反弹都被清零（粒子加速器） |
| `projectile_speed` | 3000 | 弹速（像素/秒） |
| `increase_projectile_speed_with_range` | false | true 时弹速随射程属性提升（火焰喷射器） |
| `projectile_scene` | 空 | 弹道表现资源；为空时使用默认子弹弹道 |

## 3. 武器属性与玩家属性的联动公式（数值重算全流程）

每波开始（及属性变动重算）时，武器结算规则从基础数值面板生成实战用的当前数值面板（近战/远程各有入口），按以下顺序执行（顺序影响结果）：

> 下文 `stat(X)` 表示玩家属性 X 的当前总值 = 永久属性 + 临时属性 + 联动属性之和。

### 3.1 复制基础值 + 类型/类别加成

1. 复制全部基础字段。
2. **武器类型加成** `weapon_type_bonus`（物品效果"全部近战/远程武器 +X 某字段"）：直接对同类型武器的对应字段做加法。
3. **武器类别加成** `weapon_class_bonus`（物品效果"X 类武器 +Y"）：对持有该套装的武器叠加；其中 `stat_percent_damage` 记入 `set_bonus_dmg`（百分比，最后乘算），`lifesteal`、`crit_damage` 的数值要 `/100`。

### 3.2 武器自带效果对数值的修改

- **燃烧效果**：覆写 `burning_data`；
- **爆炸效果**：标记 `is_exploding`（伤害将吃爆炸伤害加成）；
- **命中减速效果**：叠加 `speed_percent_modifier`（命中减速）；
- **同名堆叠效果**（木棍）：`字段 += value × max(0, 同名武器数-1)`；
- **属性联动效果**：按"每 N 点某属性"给武器字段加值。

### 3.3 Scaling 注入与命中减速联动

- 物品效果 `weapon_scaling_stats`（"所有武器伤害额外随 X 属性成长 +N%"）把 `[属性, N/100]` 并入 `scaling_stats`（已有同属性则系数相加）。结构（炮塔等）与"特殊生成"弹道（命中附属弹道）**不**享受此注入。
- 物品效果 `slow_on_hit`：若武器 scaling 含指定属性，则 `speed_percent_modifier -= 值`。

### 3.4 攻速 → 冷却

```
atk_spd = (stat(攻速%) + attack_speed_mod) / 100      # 结构/宠物武器固定为 0
cooldown = max(2, cooldown / (1 + atk_spd))           # 攻速为正
cooldown = max(2, cooldown * (1 + |atk_spd|))         # 攻速为负
```
（冷却下限为最小冷却 = 2 帧）。攻速为正时 `recoil` 与 `recoil_duration` 也除以 `(1+atk_spd)`（缩短攻击动画）。
随后应用物品的**冷却下限**效果（若"攻击总时长×60 < 下限"，把差额加回 `cooldown`，并按比例压缩大换弹倍率）与**冷却上限**效果（超出部分从 `cooldown` 扣除）。

### 3.5 伤害公式（核心）

```
# ① scaling 加成（加算）
damage = max(1, base_damage + Σ scaling_stats[i].系数 × stat(scaling_stats[i].属性))
#    其中属性为 stat_levels 时取"玩家等级"
# ② 百分比加成（乘算）
percent = 1 + stat(%伤害)/100                          # 结构改用 stat(结构%伤害)
explosion_bonus = stat(爆炸伤害%)/100                   # 仅带爆炸效果的武器
damage = max(1, round(damage × (set_bonus_dmg + percent + explosion_bonus)))
```
即最终伤害 = `(基础 + Σ属性×系数) × (1 + %伤害/100 + 套装%伤害/100 + 爆炸%/100)`，向最近整数取整，下限 1。

### 3.6 暴击 / 精准 / 吸血 / 击退

```
crit_chance += stat_capped(暴击率)/100        # 结构默认不可暴击，除非 structures_can_crit
accuracy    += effect(accuracy)/100           # 物品"精准度"效果
lifesteal   += stat(吸血%)/100                # 结构无吸血
knockback    = clamp(knockback + effect(击退), 允许的正负区间)
#  仅允许负击退的武器，玩家击退加成取负向；
#  negative_knockback 效果（诅咒）会把正击退整体反转为吸引
```

### 3.7 射程（近战减半）

- 近战：`max_range = max(25, max_range + stat(范围)/2)`（**范围属性只吃一半**）；
- 远程：`max_range = max(25, max_range + stat(范围))`；
- 结构：`max_range = max(25, max_range + stat(结构范围))`（结构走独立的结构数值结算）；
- 若 `min_range > 0`，下限改为 `min_range + 25`（最小射程 = 25）。

### 3.8 远程公共字段联动

```
nb_projectiles += effect(projectiles)                  # 物品"+N 弹道"（基础弹道数>0 才生效）
projectile_spread += effect(projectiles) × 0.1         # 每多 1 弹道散射角 +0.1 rad
piercing += effect(piercing)                           # 物品"+N 穿透"
piercing_dmg_reduction = clamp(基础值 - stat(穿透伤害%)/100, 0, 1)
bounce = can_bounce ? 基础值 + effect(bounce) : 0
bounce_dmg_reduction = clamp(基础值 - stat(反弹伤害%)/100, 0, 1)
projectile_speed = increase_projectile_speed_with_range
                 ? clamp(基础 + 基础/300 × stat(范围), 50, 6000) : 基础
```

### 3.9 燃烧数据初始化

- 武器无自带燃烧、玩家无全局燃烧（`burn_chance` 效果）→ 空燃烧；
- 武器无自带燃烧但有全局燃烧 → 直接采用全局燃烧参数；
- 武器自带燃烧 → `damage += 全局燃烧 damage`；
- 然后：`damage = (damage + Σ scaling) × (1 + %伤害/100)`（结构用结构%伤害）；`spread += effect(burning_spread)`。
- 命中时点燃判定见 §7.3。

### 3.10 爆炸伤害（物品/地雷用）

`max(1, round((基础+Σscaling) × (1 + %伤害/100 + 爆炸伤害%/100)))`。带爆炸效果武器的爆炸直接沿用命中判定区伤害（已含爆炸加成，见 §3.5）。

### 3.11 每 X 发强化弹

物品效果 `modify_every_x_projectile` / 武器自带的"子效果组"特效：先临时套用子效果（如 +N 弹道），重算一份远程数值面板按 X 值缓存；附加散射 `+0.15 × 弹道增量`。开火时若 `发数 % X == 0` 则当次使用强化面板。

### 3.12 攻击总时长（面板"冷却"显示值）

- 近战：`总时长 = atk_duration/2 + back_duration + recoil_duration + cooldown×倍率/60` 秒，其中：
```
atk_spd      = (stat(攻速) + attack_speed_mod)/100
range_factor = max(0, 距离 / clamp(70×(1+atk_spd/3), 70, 120))
atk_duration = max(0.01, 0.2 - atk_spd/10) + range_factor×0.15
back_duration= atk_spd>0 ? 0.2/(1+atk_spd×3) : 0.2
```
- 远程：`总时长 = recoil_duration×2 + cooldown×倍率/60` 秒。

## 4. 攻击循环：索敌、冷却与开火条件

### 4.1 索敌

- 索敌触发区域半径 = `max_range + 探测附加距离(200)`。敌人进入/离开/死亡都会实时维护"射程内目标列表"。
- 目标 = 范围内**距离大于 `min_range` 的最近敌人**。
- 非手动瞄准时武器朝向目标；攻击中保持朝向并附加本次散射角。手动瞄准（按住鼠标/右摇杆）时武器直接朝向光标/摇杆方向。

### 4.2 开火条件

同时满足：① 冷却 ≤ 0；② 玩家静止（或拥有 `can_attack_while_moving` 效果，即"移动射击"）；③ 目标距离在 `[min_range, max_range+50]` 内（有效射程额外 +50 容差），或处于手动瞄准。冷却只在**非攻击动画期间**按 60/秒递减。

### 4.3 冷却随机抖动

```
max_rand = min(武器数(≤6) × cooldown / 5, 武器数 × 5)
下次冷却 = 随机(max(1, cooldown - max_rand), cooldown + max_rand)
```
波次开始时冷却基数被压到最多 180 帧（3 秒），避免重武器开场过久不攻击。

### 4.4 大换弹（`additional_cooldown_every_x_shots`）

`发数 % X == 0` 时本次冷却 = `cooldown × additional_cooldown_multiplier`，并在换弹期间显示 `custom_on_cooldown_sprite`（冷却 ≤10 帧时换回）。使用者：左轮（每 6 发 ×8）、链枪（每 100 发 ×60）、弩/狙击枪贴图（无大换弹但有空弦贴图）。

### 4.5 开火流程

1. 累计开火计数，命中 §3.11 的强化弹则临时换面板；
2. `reload_turrets_on_shoot` 效果会重置炮塔冷却；
3. 计算散射：`spread = 随机(-1+accuracy, 1-accuracy)` 弧度并加到自身旋转；
4. 设定命中判定区击退向量 = 武器朝向 × `knockback`；
5. 调用射击行为（近战/远程，见 §5/§6）；
6. 取下次冷却。

### 4.6 其它武器级机制

- **攻击 ID 统计**：每次攻击分配自增的攻击编号，按编号统计单次攻击命中数/击杀数（保留最近 20 次），用于"单次攻击击杀 N 个敌人获得属性"效果（`gain_stat_when_attack_killed_enemies`，矮人角色工程加成等）与挑战成就。
- **每击杀 N 个敌人获得属性**（击杀成长效果）：按本波击杀计数取模触发，幽灵系武器用，见 §11.2。
- **break_on_hit**（诅咒效果）：命中时按 `value%` 概率自毁并掉金币。
- **武器外框高亮**：按 Tier 颜色描边。

## 5. 近战武器机制（突刺 / 挥击）

近战没有独立弹道，**伤害判定 = 武器贴图上的命中判定区在缓动动画期间启用**。命中判定区在数值重算时写入伤害/暴击/燃烧等参数，每次攻击开始清空忽略列表（每次攻击每个敌人只判一次：命中后把目标加入忽略列表）。

### 5.1 突刺 THRUST

1. 后撤 `recoil` 像素，历时 `recoil_duration`（此阶段**无判定**）；
2. 启用命中判定区，向前推进到 `x + max_range`，历时 `atk_duration/2`；
3. 到达后关闭命中判定区（若 `deal_dmg_on_return=true` 则保持开启）；
4. 用 `back_duration` 收回；`deal_dmg_on_return` 武器在收回结束才关闭命中判定区。

### 5.2 挥击 SWEEP

- 实际挥击距离 `atk_distance = min(max_range, max(250, 目标距离))`（最小挥击距离 = 250），并据此重算 `atk_duration`；
- 阶段：后撤并旋转到一侧 `±0.9π`（无判定）→ 启用命中判定区，扫到正前方 `x + atk_distance×0.75`、侧移 `atk_distance/2`（历时 `atk_duration/4`）→ 继续扫到另一侧 `∓0.9π`（再 `atk_duration/4`）→ 关命中判定区（或回程继续判定）→ 收回。
- 即挥击覆盖约 ±162° 的扇形，半径约为 `atk_distance`。挥击模式的有效射程直接用 `max_range`（不加 50 容差）。

### 5.3 交替攻击

`alternate_attack_type=true` 的武器（剑、斩首之剑、王者之剑、战斗棍）每次开火后切换突刺/挥击。手动瞄准时近战在攻击动画中不再跟随光标旋转。

### 5.4 近战命中后的武器侧逻辑

命中 → 吸血判定（§7.4）→ 把目标加入忽略列表 → 对每个爆炸效果按 `chance` 再掷一次爆炸（以命中点为中心、伤害=命中判定区伤害、忽略被命中者；注意单位侧还有一次独立判定，见 §7.2）→ 伤害统计/挑战/break_on_hit。

## 6. 远程武器机制（弹道生成 / 穿透 / 反弹）

### 6.1 弹道生成

开火一次生成 `nb_projectiles` 枚弹道，每枚方向独立取 `随机(朝向 - projectile_spread, 朝向 + projectile_spread)`，全部共享同一攻击编号。生成由武器结算规则统一处理：

- 弹道从**枪口**位置射出，速度向量 = 方向 × `projectile_speed`；
- 弹道实例从对象池取（按弹道表现资源分池；无自定义弹道用默认子弹）；
- 弹道效果列表会注入玩家的 `pierce_on_crit` / `bounce_on_crit` 效果（与武器自带的同名效果数值合并）；
- 随后武器播放后坐动画（位移 `recoil`，往返各 `recoil_duration`）。

### 6.2 弹道飞行与射程

- 每帧 `位置 += 速度向量 × 帧时长`；
- **射程以存活时间实现**：`存活时间 = (max_range + 100) / projectile_speed`（弹道附加飞行距离 = 100，手动瞄准时减半为 +50），超时即回收；离开镜头范围也回收；
- `rotation_speed != 0` 的弹道每帧自转 25°（手里剑）。

### 6.3 命中：反弹优先于穿透

```
命中 → 吸血判定 → 把目标加入忽略列表
if bounce > 0:   反弹
elif piercing <= 0:  停止
else:            piercing -= 1
                 damage = max(1, damage - damage × piercing_dmg_reduction)
```
- **穿透衰减**：每穿 1 个敌人，伤害乘 `(1 - piercing_dmg_reduction)`，下限 1（被玩家"穿透伤害"属性减免，§3.8）。
- **反弹**：`bounce -= 1`；从场上随机取一个敌人作为新目标（取不到则随机方向）；速度大小不变换向；**射程变为无限（10000）**重新计时；**击退清零**；伤害乘 `(1 - bounce_dmg_reduction)`，下限 1。
- **暴击转化**：弹道暴击时，每层 `bounce_on_crit` 效果 +1 反弹、每层 `pierce_on_crit` 效果 +1 穿透，效果层数逐次消耗（手里剑、弩）。

### 6.4 特殊弹道类型

|---|---|---|

## 7. 命中结算：暴击、爆炸、燃烧、吸血

命中流程：命中判定区（武器/弹道/爆炸共用同一套规则）进入敌人受击判定区 → 进入单位受击结算。

### 7.1 暴击

受击结算时掷骰：`是否暴击 = 随机数0~1 <= crit_chance`；暴击则 `伤害 = round(伤害 × crit_damage)`。补充：拥有 `crit_on_hitting_burning_target` 效果时命中燃烧目标必暴击；斩杀效果（斩首之剑）按 `value%` 概率直接造成等于目标当前生命的伤害。暴击会触发武器闪光动画与 `bounce_on_crit/pierce_on_crit` 转化。

### 7.2 爆炸效果的双判定路径

- **单位侧**：命中时对每个爆炸效果按 `chance` 判定，成功则**取消直接伤害**，在目标位置生成爆炸（伤害/暴击/燃烧参数复制自命中判定区；爆炸命中/击杀回连武器统计）；
- **武器侧**（仅近战武器自身命中判定区）：命中事件后再按 `chance` 独立判定一次额外爆炸（忽略被命中者）。
- 爆炸范围 = `effect.scale × (1 + stat(爆炸范围%)/100)`；带爆炸效果武器的伤害已计入 `stat(爆炸伤害%)`（§3.5）。

### 7.3 燃烧

未触发爆炸的命中：若 `随机数0~1 <= burning_data.chance` 且玩家有 `can_burn_enemies`（默认有）且非治疗命中 → 点燃目标。燃烧叠加规则：同一敌人取 chance/damage/duration/spread 的最大值；`spread>0` 时燃烧蔓延给附近敌人。

### 7.4 吸血

每次命中做吸血判定：`随机数0~1 < lifesteal` 则回 1 HP（拥有双倍吸血效果时回 2）。近战与弹道命中均会触发。

### 7.5 其它命中联动

- `speed_percent_modifier != 0`：按敌人基础移速百分比施加衰减减速；
- 伤害统计：按武器栏位累计到武器 DPS 面板；伤害追踪键用于物品统计；
- 敌人死亡时的 `landmines_on_death_chance` 效果：若武器 scaling 含指定属性（或燃烧致死对应元素），按概率掉地雷。

## 8. 武器等级（Tier）与合成机制

### 8.1 等级

武器分 Tier I~IV（数据内 `tier = 0..3`）。同一武器每个等级是独立的武器元数据（独立价格与数值），用 `upgrades_into` 串成链。并非所有武器都有完整 I~IV：如圆锯/锤/剑/火球等从 II 级起，迷你机枪/狙击枪/核弹等从 III 级起，传奇类（王者之剑、镰刀、电钻、毁灭者、链枪、加特林激光）仅 IV 级（见 §10 表）。

### 8.2 合成条件

```
可合成 ⇔ 持有 ≥2 把 my_id 相同的武器（同名且同 Tier）
        且 upgrades_into != null
        且 tier < max_weapon_tier 效果值
```
诅咒与非诅咒的同名同级武器**可以**互相合成（合成时优先选择诅咒状态不同的那把作为材料）。

### 8.3 合成执行

- 移除 2 把材料武器，添加 `upgrades_into` 武器（高一级）；材料的 `tracked_value`（武器面板累计统计）相加继承给新武器；
- 若商店购买同名武器时**武器栏已满**：自动触发"购买即合成"（只移除 1 把旧武器 + 新买的 1 把）；
- 任一材料为诅咒武器 → 产物也为诅咒，诅咒强度取材料（及其效果）`curse_factor` 的最大值；
- 拥有 `lock_current_weapons` 效果（角色限制）时禁止合成/弃置。
- **合成不退钱、不花钱**：产物价值即其自身 `value`（用于此后的回收计价）。

### 8.4 价格计算

商店标价（武器同样适用）：
```
v = value × (1 + weapons_price%/100)                 # 武器价格效果（仅武器）
标价 = max(1, (v + wave + v × wave × 0.1)
            × (1 + (items_price% + 特定物品价格%)/100)
            × (1 + endless_factor/5))
```
即基础价 + 波数加项 + 每波 10% 的乘项。各等级 `value` 大致按 ×2 递增（如小刀 15/31/61/122）。

### 8.5 回收（弃置）价格

```
回收价 = min(当前标价, max(1, 当前标价 × clamp(0.25 + recycling_gains%/100, 0.01, 1)))
```
默认即标价的 25%；`value=1` 的物品固定回 1。武器回收还吃 `specific_items_price` 折价因子。

## 9. 武器类别（Class/Set）与套装加成

- 拥有 `all_weapons_count_for_sets` 效果（角色"百艺"类）时，每个出现过的类别计数 = 武器总数；

每击杀 N 个敌人（N 随等级 20/18/16/12 递减）永久 +1 属性，由武器本波击杀计数触发：幽灵斧→%伤害；幽灵燧石→攻击速度；幽灵权杖→最大生命。

### 11.3 引爆 / 爆炸类

木板（25/30/35/40% 概率）、等离子大锤（25/50%）、动力拳套（25/50%）、毁灭者（50%）为近战概率爆炸；火球、火箭筒、核弹发射器为 100% 命中爆炸（核弹爆炸范围 ×2.0/×2.25）；粉碎者 Shredder 为概率爆炸（50/65/80/100%）且自带 3 穿透 0 衰减。爆炸伤害 = 武器命中判定区伤害（已含爆炸%加成），双判定路径见 §7.2。

### 11.4 治疗 / 吸血类

医疗枪（吸血 40~60%）、圆锯（45/50/60%）、剪刀（40~60%）、镰刀（100% 吸血 + 每秒扣 3 HP + 玩家每次受击临时 +3% 伤害，`temp_stats_on_hit`）；剁肉刀：拾取食物额外 +1/+1/+1/+2 HP；利齿：吸血成长 + "每损失 25/20/15/10% 生命 +1% 吸血"。

### 11.5 其它特殊机制

- **木棍 Stick**：同名堆叠效果 —— 每多持有 1 把任意等级木棍，伤害 +4/6/8/10（§3.2）。
- **战斗棍**：伤害随玩家等级成长（`stat_levels` scaling）+ 全局经验 +2/5/9/15%，突刺/挥击交替。
- **马上长枪**：伤害随移速成长，+2~5% 移速，但"静止时 -10/-15/-20/-25% 伤害"（`temp_stats_while_not_moving`）。
- **斩首之剑 Vorpal**：命中 1/2/3% 概率"斩杀"（斩杀效果：造成目标当前生命值的伤害）。
- **短剑 Dagger / 电钻**：暴击击杀 50~80%（电钻 100%）概率掉 1 金币。
- **电钻 Drill**：冷却 2 帧 + 周期性临时 +10 攻速（`temp_stats_per_interval`）。
- **王者之剑 Excalibur**：伤害 = 200 + 近战×2 + 最大生命×2，但持有 -3 护甲；交替攻击。
- **链枪 Chain Gun**：冷却 2 帧 ×3 弹道，每 100 发强制换弹（冷却 ×60 = 2 秒），换弹期间换贴图。
- **左轮 Revolver**：每 6 发换弹（冷却 ×8）。
- **歼灭者 Obliterator**：99 穿透 0 衰减 + 远程×10/×15 成长，170 帧冷却的全屏狙击。
- **粒子加速器**：弹道是拉伸至射程的整条光束（§6.4），99 穿透 0 衰减、不可反弹、命中点燃+减速。
- **火焰喷射器**：99 穿透但衰减 100%（穿透后伤害保持下限 1），靠点燃输出；弹速随射程提升。
- **电击枪 Taser**：弹道带减速区（-200 衰减速度）+ 高等级多弹道散射。
- **土豆发射器**：伤害随最大生命成长（远程×0.25 + 最大生命×0.2）。
- **手 Hand**：伤害固定 1，提供 +3/6/9/18% 收获。
- **石头 Rock**：II 级起附带 +1/+1/+2 护甲、III 级起 +2 最大生命。
- **手里剑 Shuriken**：暴击时 +1~4 反弹且反弹**无衰减**（`bounce_dmg_reduction=0`）；弹道自转。
- **弩 Crossbow**：暴击时 +1~4 穿透且穿透**无衰减**（`piercing_dmg_reduction=0`）。
- **冰锥 Icicle**：元素成长的高暴击穿透弹。
- **狙击枪**：伤害随射程属性成长（+0.2/0.3），命中后从目标弹出 5/8 枚延迟激光。
- **诅咒武器**：`is_cursed` 武器枪口挂诅咒粒子，由 DLC `curse_item` 强化数值。

## 附：关键公式速查

| 项目 | 公式 |
|---|---|
| 最终伤害 | `max(1, round((基础 + Σ属性×系数) × (1 + %伤害/100 + 套装%/100 + 爆炸%/100)))` |
| 冷却(攻速) | `max(2, 基础冷却 / (1 + 攻速/100))` |
| 暴击率 | `武器暴击率 + capped(玩家暴击率)/100`，伤害 ×`crit_damage` |
| 近战射程 | `max(25, 基础 + 范围/2)`；远程 `max(25, 基础 + 范围)` |
| 散射角 | `随机(-1+accuracy, 1-accuracy)` rad/次 + 每发 `±projectile_spread` |
| 穿透衰减 | `dmg = max(1, dmg × (1 - piercing_dmg_reduction))` |
| 反弹 | 随机新目标、射程无限、击退清零、`dmg × (1 - bounce_dmg_reduction)` |
| 弹道射程 | 存活时间 `(max_range+100)/弹速` 秒 |
| 商店价 | `(v + wave + v×wave×0.1) × 价格因子`，`v = value×(1+武器价格%)` |
| 回收价 | `标价 × clamp(0.25 + 回收加成%, 0.01, 1)` |
| 套装档位 | 计数≥2，取 `set_bonuses[min(n-2, 4)]` |
