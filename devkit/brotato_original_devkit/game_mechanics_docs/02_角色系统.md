# 02 角色系统

> 本章覆盖：角色数据结构、全部 49 个角色的完整数值、角色专属机制的底层规则、角色对商店的影响、解锁条件。以下均为本游戏的设计规格值。

> **角色数量说明**：本作共提供 **49 个可用角色**。另预留了 `character_hiker / druid / builder / dwarf / gangster / ogre / creature / curious` 等扩展角色 ID，未配备角色数据，不可用。

---

## 1. 角色数据结构

### 1.1 数据层级

```
物品基础数据
  └─ 物品数据
       └─ 角色数据
```

角色本质上是一种"物品"：开局时把角色数据当作普通物品加入玩家物品列表，其 `effects` 数组随之被逐条施加，从此进入该玩家的全局效果字典。

### 1.2 字段定义

**角色数据专有字段**：

| 字段 | 类型 | 含义 |
|---|---|---|
| `wanted_tags` | Array[String] | "偏好物品标签"。商店刷物品时有 5% 概率（见 §4.2）只保留带这些标签的物品 |
| `banned_item_groups` | Array[String] | 禁用物品组（组定义见 §4.4），这些物品不会出现在该角色的商店中 |
| `banned_items` | Array[String] | 按物品 ID 逐个禁用 |
| `starting_weapons` | Array | 开局武器候选列表。开局界面让玩家从中**选择 1 把** |
| `starting_items` | Array | 开局直接拥有的物品（仅 beast_master 使用，4 只宠物） |

**继承自物品数据**：`max_nb`（所有角色均为 -1，无数量上限）、`item_appearances`（外观贴图）、`tags`（角色自身标签，仅 beast_master 为 `["pet"]`）、`replaced_by`。

**继承自物品基础数据**：

| 字段 | 说明 |
|---|---|
| `my_id` | 唯一 ID，全部形如 `character_xxx` |
| `unlocked_by_default` | 是否默认解锁（仅 brawler / crazy / mage / ranger / well_rounded 为 true） |
| `name` | 本地化键，形如 `CHARACTER_XXX` |
| `tier` / `value` | 所有角色均为 tier=0、value=1 |
| `effects` | **效果列表**，角色全部机制都由效果条目数组承载 |
| `tracking_text` | 角色面板上的追踪统计文本（bull / glutton / lich / lucky 为 `DAMAGE_DEALT`，统计角色专属伤害） |
| `is_cursed` / `curse_factor` | 诅咒（所有角色均为 false / 0.0） |

### 1.3 效果条目结构

每条效果包含：

| 字段 | 说明 |
|---|---|
| `key` | 写入效果字典的键（如 `stat_max_hp`、`weapon_slot`、`pacifist`） |
| `value` | 数值 |
| `custom_key` | 复合存储时的容器键（如 `stats_on_level_up`、`temp_stats_on_dodge`） |
| `storage_method` | 存储方式：`0 SUM`（累加）、`1 KEY_VALUE`（向 custom_key 容器追加 `[key, value]`）、`2 REPLACE`（直接替换，记录 base_value 以便撤销）、`3 APPEND_KEY`、`4 APPEND_KEY_VALUE` |
| `effect_sign` | UI 颜色：0 正面 / 1 负面 / 2 中性 / 3 由 value 正负决定 / 4 由参数决定 / 5 覆盖 |
| `text_key` / `custom_args` | 仅影响描述文本格式化，不影响数值 |

角色用到的**效果类型**：

| 效果类型 | 行为 |
|---|---|
| 属性获取效率效果 | "属性获取效率"修正：对 `stats_modified` 中每个属性写入 `gain_<stat>` 键（累加百分比）。之后所有该属性的增减都会按属性增益系数比例缩放（如 +33% 伤害类属性获取） |
| 属性联动效果 | 联动属性：向 `stat_links` 数组写入 `[目标属性, value, 被缩放属性, nb_stat_scaled, perm_stats_only]`，即"每 `nb_stat_scaled` 点 X 获得 `value` 点 Y"。被缩放对象除普通属性外还支持特殊计数：`materials`（当前材料）、`structure`、`pet`、`living_tree`、`common_item`/`legendary_item`/`different_item`（拥有的不同 I 级/IV 级/任意物品数）、`percent_player_missing_health`（已损失生命百分比）等，详见 01 章联动属性层与一次性存量换算 |
| 武器类别加成效果 | 武器类别加成：向 `weapon_class_bonus` 写入 `[set_id, stat, value]`，武器初始化属性时若属于该 set 则套用 |
| 属性转换效果 | 属性转换：注册到 `convert_stats_end_of_wave`（波次结束，永久转换）或 `convert_stats_half_wave`（波次过半，临时转换），由属性转换规则按"每 `value` 点源属性 → `to_value` 点目标属性、转换 `pct_converted`% "执行 |
| 概率属性伤害效果 | 概率属性伤害：向 `dmg_when_heal` / `dmg_when_pickup_gold` 等容器写 `[属性, 百分比, 概率, 追踪键]`，触发时对一名随机敌人造成 `属性值 × 百分比/100 × (1+%伤害)` 的伤害 |
| 爆炸注册效果 | 注册爆炸：写入 `explode_on_hit` / `explode_on_consumable` 容器，附带独立的爆炸武器参数（伤害、缩放、范围等） |
| 双值效果 | 双值参数（streamer 的"静止回金"用，`[value, value2]` = [百分比, 每跳上限]） |

### 1.4 角色机制的事件挂钩点（触发规则总表）

角色效果只是把数据写进效果字典；真正的触发逻辑按以下规则结算：

| 效果键（custom_key 或 key） | 触发时机与规则 |
|---|---|
| `stats_on_level_up` | 每次升级时把容器里的属性永久加给玩家 |
| `temp_stats_on_dodge` | 闪避成功时获得**当波临时属性**（临时属性层） |
| `temp_stats_on_hit` | 受到伤害时获得当波临时属性 |
| `explode_on_hit` | 受击时在自身位置产生爆炸 |
| `explode_on_consumable` | 拾取消耗品时在其位置产生爆炸 |
| `dmg_when_heal` | 每次治疗触发概率属性伤害 |
| `dmg_when_pickup_gold` | 每次拾取材料触发概率属性伤害 |
| `stats_below_half_health` | 生命低于 50% 时获得临时属性，回到 50% 以上移除 |
| `temp_stats_while_moving` / `temp_stats_while_not_moving` | 移动/静止时获得临时属性；其中 `percent_materials` 子项由计时器按跳结算材料 |
| `consumable_stats_while_max` | 满血时拾取消耗品 → 永久属性 |
| `convert_stats_end_of_wave` | 波次结束执行永久转换 |
| `convert_stats_half_wave` | 波次进行到一半时执行临时转换 |
| `stats_end_of_wave` | 波次结束时永久增减属性 |
| `unique_weapon_effects` | 每持有 1 种**不同**武器套用一次容器内属性 |
| `additional_weapon_effects` | 每持有 1 把武器（含重复）套用一次 |
| `tier_iv_weapon_effects` / `tier_i_weapon_effects` | 每持有 1 把 IV 级 / I 级武器套用一次 |
| `pacifist` | 波次结束：材料&经验 +=`round(存活敌人数 × value/100)`，群浪(Horde)波减半 |
| `cryptid` | 波次结束：材料&经验 += `存活树木数 × value` |
| `trees` | 每波生成树木数量修正（所有玩家求和） |
| `trees_start_wave` | 树木在开波 5 秒内立刻生成 |
| `beast_master_effect` | 宠物攻击的属性缩放扩展：将 melee/ranged/elemental/engineering 中缺失的缩放项以首个缩放系数补全 |
| `hp_shop` | 商店货币改为**最大生命值**（购买扣最大生命） |
| `weapon_slot` | 武器栏位数（默认 6，REPLACE 0=禁武器，SUM -5=1 格，REPLACE 12=12 格） |
| `weapon_slot_upgrades` | 只要当前栏位 < 该值，升级奖励**强制变为"+1 武器栏"** |
| `min_weapon_tier` / `max_weapon_tier` | 商店武器档位钳制 |
| `minimum_weapons_in_shop` | 商店每次刷新保底武器数 |
| `destroy_weapons` | 每次进商店时销毁玩家全部武器 |
| `guaranteed_shop_items` | 商店必出指定物品（fisherman 的鱼饵） |
| `specific_items_price` | 指定物品价格百分比修正 |
| `items_price` / `weapons_price` | 物品/武器价格百分比修正（公式见 §4.3） |
| `upgraded_baits` | 持有的鱼饵生成额外敌人波组：每 10 个 +1 组，每 15 个组内敌人 +1 |
| `harvesting_growth` | 每波结束收获属性自增长百分比（默认 5） |
| `recycling_gains` | 回收物品返还比例 = clamp(25% + value%, 1%, 100%) |
| `gain_pct_gold_start_wave` | 开波时获得/失去当前材料的 value%（-100 即清空材料） |
| `next_level_xp_needed` | 升级所需经验 ×(1+value/100) |
| `lose_hp_per_second` | 每秒受到 value 点伤害 |
| `can_attack_while_moving` | REPLACE 0 = 移动时武器不开火 |
| `no_melee_weapons` / `no_ranged_weapons` | 商店不再出现近战/远程武器 |
| `no_duplicate_weapons` | 商店不出现已持有武器的同名（或更低档）武器 |
| `all_weapons_count_for_sets` | 套装计数：任一武器的套装按"武器总数"计数（所有武器都算进每个套装） |
| `dodge_cap` | 闪避上限 REPLACE（默认 60） |
| `pickup_range` | 拾取范围加成 |
| `gold_drops` / `enemy_gold_drops` | 掉落材料的价值百分比修正（敌人掉落 = 两者之和） |
| `number_of_enemies` / `enemy_health` / `enemy_damage` / `enemy_speed` / `map_size` | 全局波次参数修正（敌人生成与初始化时读取） |
| `structure_attack_speed` | 结构攻速百分比 |
| `knockback` | 全武器击退加成 |
| `projectiles` / `piercing` / `accuracy` | 全武器投射物数 / 穿透 / 精准修正 |
| `consumable_heal` | 消耗品治疗量修正（vampire -100 → 吃水果不回血） |
| `no_heal` | 完全无法恢复生命 |
| `starting_item` / `starting_weapon`（custom_key） | 开局自动获得指定物品/武器（在选择武器之外**额外**获得） |
| `boosted_wanted_item_tag` | wanted_tags 触发概率 5%→10% |

## 2. 解锁机制（底层）

- 解锁由**挑战系统**驱动：挑战数据的奖励类型为"角色"且奖励指向某个角色时，完成该挑战即把该角色记入跨局进度存档的已解锁角色列表。
- 属性型挑战（`stat` 字段非空）在每次属性变化后检查：正值要求 `属性 ≥ value`，负值要求 `属性 ≤ value`。
- 计数型挑战（击杀/采集/伐树）使用**跨局累计**的击杀数 / 材料采集数 / 伐树数，每波结束检查。
- 胜局难度挑战：赢下一局后由胜局结算自动完成 `chal_difficulty_ + 当前危险等级` 挑战。
- 每个角色还有一个同名挑战（如 `chal_brawler`）= "用该角色赢一局"，奖励为物品/武器（不是角色解锁条件）。

## 3. 全部 49 个角色详解

> 通用说明：
> - 以下为各角色的设计数值。
> - "获取效率"= 属性获取效率效果，影响后续所有该属性的增减幅度。
> - "8 项伤害修正" = `stat_percent_damage, stat_ranged_damage, stat_melee_damage, stat_elemental_damage, explosion_damage, piercing_damage, bounce_damage, damage_against_bosses`。
> - "额外起始武器"= `custom_key=starting_weapon` 效果，开局自动获得，**不占用**玩家在起始武器列表中的自选名额。
> - 起始武器后的罗马数字为武器档位。

### 3.1 Well Rounded（character_well_rounded，均衡者）— 默认解锁

| 效果 | 数值 |
|---|---|
| 最大生命 | +5 |
| 速度 | +5 |
| 收获 | +8 |

- wanted_tags：无；无禁用。
- 起始武器（13 选 1）：Knife I, Stick I, Fist I, Cactus Mace I, Ghost Flint I, Wrench I; Pistol I, SMG I, Double Barrel Shotgun I, Slingshot I, Taser I, Wand I, Icicle I。

## 4. 角色对商店的影响

### 4.1 商店刷新流程

每个商店槽位先决定类型（武器/物品）：第 1~2 波保底 2 把武器，第 3~5 波保底 1 把；`minimum_weapons_in_shop`（arms_dealer / baby = 1）抬高保底值；若 `weapon_slot = 0`（bull / beast_master）则全部改为物品。`guaranteed_shop_items`（fisherman 的鱼饵）优先占用物品槽。

### 4.2 wanted_tags（偏好标签）

每个**物品**槽位有 5% 的概率触发"偏好过滤"——把不带任何 `wanted_tags` 标签的物品从候选池移除（即该槽位必出带偏好标签的物品）。Beast Master 的 `boosted_wanted_item_tag` 把这一概率提升为 10%。武器槽不受 wanted_tags 影响（武器走"同武器 20%~25% / 同套装 35%~55%"权重）。

### 4.3 价格修正

```
最终价格 = max(1, (V + wave + V×wave×0.1) × (1 + (items_price + specific_item_price)/100) × (1 + endless))
其中武器的 V = 基础价 × (1 + weapons_price/100)
```
- `weapons_price`：arms_dealer -95。
- `items_price`：baby -20、entrepreneur -25、glutton +25、mutant +50、saver +50。
- `specific_items_price`：fisherman 对 item_bait -100（落到下限 1）。
- 回收价 = 现价 × clamp(0.25 + recycling_gains/100, 0.01, 1.0)（entrepreneur +25 → 50%）。

### 4.4 banned_item_groups 物品组定义

| 组 | 包含物品 |
|---|---|
| harvesting | blood_donation, crown, fertilizer, tractor, wheelbarrow |
| melee_damage | mastery, goat_skull, riposte |
| ranged_damage | lens |
| melee_and_ranged_damage | big_arms, hedgehog |
| lifesteal | butterfly, bat, whetstone, decomposing_flesh, bloody_hand, fresh_meat |
| lifesteal_and_hp_regeneration | blood_leech |
| hp_regeneration | mushroom, plant, sad_tomato, medikit, fairy, potion, fried_rice, baby_squid, coral, penguin |
| consumable_heal | jerky, weird_food, lemonade, fruit_basket |
| speed | terrified_onion, beanie, power_generator |
| engineering | lighthouse, cog, nail, toolbox, book, pencil |
| elemental_damage | strange_book, frozen_heart, boiling_water, snowball, toxic_sludge, fuel_tank, will_o_the_wisp |
| armor | stone_skin, helmet, metal_plate |
| dodge | adrenaline, chameleon, gambling_token, ghost_outfit |

角色的 `banned_item_groups` / `banned_items` 在为该波随机抽取物品时从物品池剔除；**仅在最终波之前生效**，进入无尽模式后只应用无尽禁用表（piggy_bank, crown）。

### 4.5 档位（Tier）出现概率

每个槽位独立掷骰，从高档到低档判定：

| 档位 | 起始波（MIN_WAVE） | 每波增加概率 | 概率上限 |
|---|---|---|---|
| IV 级（红） | 6 | +0.23%/波 | 8% |
| III 级（紫） | 2 | +2%/波 | 25% |
| II 级（蓝） | 0 | +6%/波 | 60% |
| I 级（白） | — | 基础 100%（兜底） |

公式：`chance = (wave-1-MIN_WAVE) × 每波增量 × (1+幸运/100)`（幸运为负则除以 `1+|幸运|`），再与上限取小。**角色通过幸运属性间接强烈影响高档位概率**（如 lucky +100 幸运直接让 II/III/IV 档概率翻倍）。
角色直接修正：`min_weapon_tier`（knight：武器档位 ≥ II）、`max_weapon_tier`（wildling：武器档位 ≤ II）只对武器槽钳制；`weapon_slot_upgrades`（baby）改写升级奖励而非商店。

## 5. 附录

### 5.1 默认效果基线
- `weapon_slot` 默认 6；`dodge_cap` 默认 60；`harvesting_growth` 默认 5；`hp_cap` 默认无穷大。
- 角色效果在开局即施加，与物品效果共用同一套效果字典与施加/撤销机制。

### 5.2 起始武器选择流程
开局界面列出角色的 `starting_weapons` 候选（经统一排序）供玩家选 1 把；`custom_key = "starting_weapon"` 的效果武器（brawler 的拳、crazy 的刀、ranger 的手枪、engineer 的扳手、wildling 的木棍、cyborg 的 III 级机枪）额外自动发放。

### 5.3 角色专属追踪（tracking_text）
bull / glutton / lich / lucky 的 `tracking_text = "DAMAGE_DEALT"`，其爆炸/属性伤害通过 `tracking_key`（如 `character_bull`、`character_lucky`）写入追踪统计表，在角色面板显示累计伤害。
