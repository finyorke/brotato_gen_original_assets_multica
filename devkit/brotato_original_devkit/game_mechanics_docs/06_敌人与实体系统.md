# 06 敌人与实体系统

> 本篇规定游戏内全部实体的设计：实体分类、单位通用规则、敌人、Boss/精英、玩家建造物、出生预警、玩家实体、中立单位与宠物。
> 注：本设计中所有"帧"均按 60 帧/秒计，即 60 帧 = 1 秒。

---

## 1. 实体类型与层级体系

实体类型共六类：`玩家、敌人、中立、结构物、Boss、宠物`。

设计层级：

```
实体（基础物理体）
├── 单位                                  # 有 HP/移动/受击的活动单位
│   ├── 玩家
│   ├── 敌人
│   │   ├── 巡逻型敌人                    # 侦测到玩家后加速（保留设计，当前未配置给任何敌人）
│   │   └── Boss                          # Boss 与精英共用同一框架
│   ├── 中立单位                          # 树等中立物
│   └── 宠物
├── 结构物                                # 玩家建造物（不走单位的 HP 体系）
│   ├── 炮塔系列
│   └── 地雷
└── 果冻护盾                              # 直接挂在实体层级下的环绕护盾宠物
```

每个单位的标准组成模块：`动画贴图`、`物理碰撞体`、`受击判定区`、`接触伤害判定区`、`动画控制`、`移动行为`、`攻击行为`、`持续效果挂件`、`受击闪白计时（0.1s）`、`燃烧计时（0.5s）`。

## 2. 实体基础规则

| 机制 | 说明 |
|---|---|
| 死亡 | 死亡时禁用碰撞、标记死亡、播放死亡动画并广播死亡事件；死亡动画结束后回收进对象池 |
| 死亡参数 | 死亡携带的参数：`击退向量 / 是否清场 / 是否被玩家击杀 / 击杀玩家编号 / 致命一击伤害值 / 是否处于燃烧 / 伤害来源` |
| 对象池 | 实体复用：重生时重置显示/碰撞/休眠状态 |
| 增益（Boost） | 标记为"可被增益"的实体可被增益怪/道具强化，增益参数含 `HP/伤害/速度/范围/攻速` 五项百分比加成与"是否显示描边" |
| 描边/调色 | 最多 4 层描边（增益金色、诅咒色、高亮色），叠加颜色调制栈 |
| 活动范围 | 初始化时由地图边界减去贴图半宽/半高得到活动范围的最小/最大坐标 |

## 3. 单位通用规则

### 3.1 属性结构

- **静态属性表**（每种敌人一份配置）：
  `health（生命）, health_increase_each_wave（每波生命成长）, speed（速度）, speed_randomization（速度随机量）, damage（伤害）, damage_increase_each_wave（每波伤害成长）, attack_cd（攻击冷却）, value（掉落价值）, knockback_resistance（击退抗性 0~1）, gold_spread（金币散布）, can_drop_consumables（可掉消耗品）, always_drop_consumables（必掉消耗品）, base_drop_chance（基础掉率）, item_drop_chance（物品掉率）, min/max_consumable_tier（消耗品层级范围）, armor（护甲）, armor_increase_each_wave（每波护甲成长）, generate_blood（是否产生血迹）`。
- **运行时属性**：`生命, 速度, 伤害, 护甲, 闪避`，分"当前值"与"最大值"两份。

### 3.2 波数基础成长公式

```
基础伤害(wave) = damage + damage_increase_each_wave × (wave - 1)
基础HP(wave)   = health + health_increase_each_wave × (wave - 1)
```

护甲成长：`armor = round((基础护甲 + armor_increase_each_wave × (wave-1)) × (1 + 百分比修正/100))`。

速度初始化：
```
speed_rand = rand(-speed_randomization, +speed_randomization) × 辅助功能速度系数
最终速度 = 敌人最终速度公式(基础速度 + speed_rand, ∏(1 + 玩家enemy_speed效果/100), 百分比修正)
```
燃烧时额外叠加每个玩家的 `burning_enemy_speed` 效果；宝藏怪叠加 `loot_alien_speed` 效果。

### 3.3 移动

- 每物理帧 `velocity = 移动输入(归一化×移动速度) + 击退速度`。
- `移动速度 = max(0, 当前速度属性 + 临时加速 + 衰减型加速)`。
- `衰减型加速` 每秒回复 60（为负时按 5 倍速即每秒 300 回复）。
- 减速叠加保护：若当前衰减值已低于 `-70% speed` 则不再叠加；低于 `-45% speed` 时新减速值 ÷20；单次最低钳制为 `-90% speed`。
- 位置被钳制在地图矩形内；受控移动模式下做运动预测试并沿障碍滑动；接近移动目标点时直接吸附停止。

### 3.4 受击与伤害结算

流程：暴击判定（按 `crit_chance`；命中燃烧目标时若攻击带"对燃烧目标必暴击"特性则强制暴击）→ 斩杀类命中效果判定（按概率把伤害提到不低于目标当前 HP）→ 计算伤害加成与护甲 → 扣 HP → 闪白（0.1s）→ 击退 → 死亡判定。

敌方护甲减伤为**平减**：`实际伤害 = max(1, dmg - 当前护甲)`（在护甲生效的伤害类型下）。
伤害加成项：对高于/低于 X% HP 目标加伤、对燃烧目标的非元素加伤、持续效果挂件提供的附加伤害；汇总后 `dmg = round(dmg × (1 + 总加成/100))`。

敌人侧额外伤害通道：`bonus_current_health_damage`（当前 HP 百分比伤害）、巨人暴击 HP 伤害、燃烧目标 HP 百分比伤害（希腊火），均除以 `max(1, 无尽系数×0.2)` 在无尽模式衰减；HP 百分比的分母普通敌人为 100、Boss 为 1000（即对 Boss 效果为 1/10）。

死亡时：死亡击退 `击退方向 × max(击退量, 15)`（死亡击退量下限 15）；触发击杀回血（heal_on_kill）、暴击杀获得金币（gold_on_crit_kill）、暴击杀回血等效果。

### 3.5 击退（knockback）公式

- 命中时击退方向混合：`方向 = 0.2 × 攻击判定区击退方向 + 0.8 × (玩家→敌人方向)`。
- 击退向量赋值：`knockback_vector = 方向 × 最终击退量`。
- 每帧衰减：`knockback_vector = lerp(knockback_vector, 0, 0.1)`。
- 实际速度贡献：
  ```
  击退速度 = knockback_vector × (100 - 击退抗性 × 100)
  ```
  即抗性 1.0 完全免疫。
- **穿透击退**（击退穿透 > 0 时）：
  `amount' = amount + amount × min(抗性, 穿透) / (1 - 抗性)`（抗性=1 时不变），部分无视击退抗性。
- **拉拽（负击退）**：按与攻击者的距离、双方移动方向点积、双方受击区半径计算缓冲 `buffer = 双方半径 + 自身速度因子 + 攻击者速度因子 + min(距离/3, 150)`，最大可移动距离 `max(0, 距离 - buffer)`；预计位移 `|amount|×100/6` 超出时按比例缩小；若拉拽距离 > 200 还会临时把该敌人接触伤害削减为 `接触伤害 - min(伤害-1, 90%×伤害)`，1 秒后恢复。

### 3.6 燃烧

- 燃烧计时基础 0.5s/跳，受玩家 `burning_cooldown_reduction/increase` 修正，下限 0.1s。
- 叠加规则：已燃烧时取 `chance/damage/duration/spread` 各自较大值；伤害来源在新伤害 ≥ 旧伤害时刷新。
- 每跳伤害**无视护甲、不可闪避**；`duration` 每跳 -1，归零停止。
- 燃烧状态会重算速度（`burning_enemy_speed` 减速）并计入全局燃烧敌人数。

### 3.7 接触伤害与受击区处理

受击判定区被攻击判定区命中时的结算顺序：爆炸效果（替代直接伤害）、按燃烧数据概率点燃、命中后衍生弹（projectiles_on_hit）、命中减速（按基准速度百分比加入衰减型速度）。

## 4. 敌人通用规则

- 标识字段：`enemy_id`、`is_loot`（宝藏怪）、`can_be_cursed`（可被诅咒）、`can_be_charmed`（可被魅惑）、`to_be_removed_in_priority`（超量时优先被清除）。
- **目标选取**：在所有存活玩家与可被攻击宠物中选最近者；多目标时每 0.25s 刷新一次。
- **接触伤害**：接触伤害 = 当前伤害属性；命中后自身施加 `-max(200, 80%速度)` 的衰减减速；**被击退强于自身移动时接触伤害自动失效**（不会贴脸判定）。
- 属性表中的 `attack_cd`（30）仅作为基础攻击计时素材，主要攻击节奏由攻击行为自带的冷却控制。
- **远程伤害成长**：`弹体伤害 = 敌人最终伤害公式(行为伤害 + 行为每波伤害成长 × (wave-1))`，每个射击攻击行为独立配置。
- 宝藏怪强化：玩家有 `stronger_loot_aliens_on_kill` 效果时，宝藏怪 HP 按本局已击杀宝藏怪数量百分比提升。
- 魅惑（charm）：通过持续效果挂件实现，魅惑来源玩家编号被记录在敌人身上。
- 被增益怪强化时重置 HP/伤害/速度并按百分比放大。
- 命中时玩家 `remove_speed` 效果：敌人速度高于 `最大速度×(1-上限%)` 时扣 `最大速度×value%`。

### 4.1 敌人数值最终公式

全局乘区带缓存（每波清缓存）：

**伤害**：
```
damage = round( 波数基础伤害
  × (1 + Σ玩家enemy_damage/100)                  # 道具效果
  × (1 + danger_enemy_damage/100)                # 危险等级：D3=12%, D4=26%, D5=40%
  × (1 + (玩家数-1) × 0.08)                      # 联机修正
  × 辅助功能伤害系数
  × (1 + 无尽系数 × (1 + sqrt(平均诅咒)/25))     # 无尽
  × (1 + 百分比修正/100) )                       # 增益/精英修正
```

**HP**：
```
hp = round( 波数基础HP
  × (1 + Σ玩家enemy_health/100)
  × (1 + danger_enemy_health/100)                # D3=12%, D4=26%, D5=40%
  × (1 + (玩家数-1) × 0.3)
  × 辅助功能HP系数
  × (1 + 无尽系数 × 2.25 × (1 + sqrt(平均诅咒)/10))
  × (1 + 百分比修正/100) )
```

**速度**：
```
speed = round( (基础速度 + 随机量) × 效果系数 × 辅助功能速度系数
  × (1 + min(1.75, 无尽系数 / 13.33)) × (1 + 百分比修正/100) )
```

**无尽系数**：
```
endless_wave = max(0, wave - 20)
endless_mult = 2.0 + max(0, (wave - 35) × 0.2)
无尽系数 = (endless_wave × (endless_wave+1) / 2) / 100 × endless_mult
```

危险等级对敌方数值的设计：D1 解锁新敌人；D2 +精英（1 次）；D3 +12% HP/伤害；D4 +26%、更多精英（3 次）；D5 +40%、双 Boss。

### 4.2 死亡掉落

- 掉落开关：`can_drop_loot`（被超量清除的敌人为 false）。
- **材料掉落概率**：`wave<5 恒为 1.0；否则 max(0.5, 1 - wave×0.015)`；蜂群波 ×0.65；`always_drop_consumables` 怪恒为 1.0。
- 金币价值 = 属性表 `value`（邪恶土豆改写为 `吞金量×(进化级+1)`），再乘联机系数与 `gold_drops / enemy_gold_drops` 效果；散布半径 `clamp((value-1)×25, gold_spread, 200)`。
- **消耗品掉落**：`几率 = item_drop_chance × (1 + Σ幸运/100) / (1 + 本波已掉宝箱数)`，再乘 `(1 + crate_chance/100)`；`always_drop_consumables 且 item_drop_chance≥1` 且未进无尽时必掉。

## 5. 行为组件

### 5.1 移动行为

| 移动模式 | 逻辑 |
|---|---|
| 直线追击 | 直线追当前目标；可配置接近目标距离（30）后停下 |
| 全图随机游走 | 在全图随机选点，到达后再选 |
| 目标周边随机点 | 在目标周围 `range±随机量` 取随机点（可每 X 帧强制刷新） |
| 固定偏移跟随 | 始终朝"玩家位置+固定随机偏移"移动（偏移在出生时定死；不允许进入半径内时取圆周点） |
| 保持距离 | 保持与目标 `target_range±随机量` 的距离（沿径向移动） |
| 侦测切换 | 复合模式：目标进入侦测范围（300）前用模式 A，进入后切模式 B 并 `临时加速 +300`、广播"发现玩家"事件 |
| 绕场巡逻 | 沿地图中心半径 `0.8×半图高` 的圆周巡逻 |
| 沿边巡逻 | 沿地图边缘 `边距±随机量` 巡逻 |
| 奔向中心 | 直奔地图中心 |
| 模仿邻近实体 | 随机游走，但探测区内出现其他实体时复制其目标点 |
| 寻金移动 | 朝场上随机一枚金币移动，无金币时随机游走（邪恶土豆用） |
| 玩家输入移动 | 玩家操控（鼠标/摇杆/按键，联机分设备） |

巡逻型敌人：配合"侦测切换"模式使用，侦测到玩家时 `临时加速 +300`；当前没有敌人采用该配置（仅宠物 doc_moth 使用侦测系移动）。

### 5.2 攻击行为

**射击攻击行为**
主要参数：`弹体类型、弹速(±随机量、逐发增减)、冷却(±最大随机量)、初始冷却、每 X 发一次长冷却/长冷却时长、伤害、每波伤害成长、最小/最大射程、弹数、扇形散布(可均匀分布)、基础方向随机量、完全随机方向、在目标处生成弹体、生成散布、仅在圆周上生成、指定角度生成、至少一发命中目标点、朝移动方向射击、朝向/背向自身射击、死亡时删除弹体`。
触发条件：未在播放射击动画、冷却归零、与目标距离在 `[最小射程, 最大射程]` 内。弹速乘辅助功能速度系数。

**冲锋攻击行为**
参数：`冷却、每 X 次一次长冷却/长冷却时长、最小/最大触发距离、冲锋时长、冲锋速度、目标计算时机(起手/出手)、目标类型(玩家本体/玩家附近随机点/全图随机点)、随机目标范围、仅取目标方向上的点、冲锋时长随距离缩放`。
机制：起手锁定方向并禁止移动、质量 ×5（撞不动）；出手时 `临时加速 = 冲锋速度×辅助系数`、锁定移动方向持续"冲锋时长"秒；冲锋结束恢复质量与速度。"玩家附近随机点"模式会预判玩家移动方向（在其前进方向半圆锥内随机取点）。

**召唤攻击行为**
参数：`召唤的敌人类型、冷却、每次召唤数量、是否随机位置、是否在自身半径内、最大总召唤数`。Boss 的召唤量随玩家 `number_of_enemies` 效果放大。召唤请求经事件进入生成队列（继承召唤者的魅惑状态）。

### 5.3 目标行为（宠物用）

`最近玩家目标`（敌→最近玩家/自家玩家）、`最近敌人目标`（可排除精英/Boss、无目标时可退回玩家）、`最近未燃烧敌人目标`、`最近材料目标`、`寻金虫专用目标`、`主人目标`。

## 7. Boss 与精英

### 7.1 共用框架

- Boss 与精英共用同一框架，以 `is_elite` 区分精英（true）与 Boss（false）；不可被增益、不进对象池复活。
- **阶段状态机**：每个 Boss 配置若干阶段，每个阶段含 `hp_start`（HP 百分比阈值）、`timer_start`（秒）、独立的移动行为 + 攻击行为。状态检查每 1 秒执行一次：`HP ≤ 最大HP×hp_start` **或** 存活时间 `≥ timer_start` 即进入该阶段（按序单向推进），切换时播放吼叫音效并执行阶段切换回调。
- **登场 HP 修正**：
  - `wave ≤ 8`：HP ×0.5；
  - `(双Boss 且最终波) 或 wave ≤ 12`：HP ×0.75；
  - 最终波 且 难度≤0 且 非教学区域：HP ×0.9。
- 玩家 `damage_against_bosses` 效果对其增伤；HP 百分比类效果分母为 1000（效果 ×1/10）。
- 死亡分别计入本局精英击杀数 / Boss 击杀数；`stronger_elites_on_kill` 效果使后续精英按已杀数提升 HP（伤害提升量减半）。
- 全员基础数值：HP `1 + 750/波`（Monk 700/波；Invoker/Predator 基础 15000 + 750/波）、伤害 `1 + 1.5/波`、击退抗性 1.0、价值 10、必掉材料与消耗品。

### 7.2 出现条件

- **Boss**：最终波由波次数据的"Boss 组"触发，从当前区域的 Boss 池随机 1 只；危险 5 的 `double_boss` 效果取 2 只。区域 1 Boss 池：`boss_wizard`（Invoker）、`boss_crab`（Predator）。
- **精英/蜂群**：难度 <2 无；难度 2-3 共 1 次；难度 ≥4 共 3 次。出现波次：第 1 次 wave 11-12、第 2 次 wave 14-15、第 3 次 wave 17-18（第 3 次必为精英）。每次 40% 概率为蜂群（HORDE）否则精英（角色 Jack/Gangster 蜂群概率 0%、单人 Ogre 100%）。精英从区域精英池随机不重复。区域 1 精英池：butcher、colossus、croco(croc)、demon(gargoyle)、insect(mantis)、mom、monk、rhino 共 8 种。

## 8. 出生机制（出生预警）

- 所有非玩家实体（敌、中立、结构、宠物）出生前先生成"出生预警"标记。
- **预警时长**：60 帧 ≈ **1 秒**（无个体覆盖）。
- 闪烁节奏：每 `剩余时间/3.25` 帧透明度降到 0.25，持续 6 帧后恢复。
- 颜色：敌人=负面色（红），中立=正面色，结构/宠物=矢车菊蓝，被魅惑召唤=魅惑色。
- **防压脸**：预警检测区（半径 72，仅检测玩家）若计时结束时仍与玩家重叠且类型为敌人，则**传送到随机新位置并重新计时 1 秒**——这是敌人的"出生保护"，敌人实体在预警期间根本不存在，因此既不能被攻击也不会造成伤害（等效无敌帧）。
- 出生节流：生成队列每 3 帧处理一次，普通队列每次 1 只（积压超过 100 时升到最多 2 只）；树、Boss、召唤物、结构有独立队列。
- 数量上限：波次数据的 `max_enemies`（联机每人 +1/8）；超限时随机（优先"优先被清除"标记的敌人）令存量敌人无掉落死亡。结构上限 100。
- 敌人出生点与玩家最小距离 `300 + 个体附加最小距离`，找不到时每次放宽 5 直至 25。
- 联机敌人数量：`数量 += 数量 × 0.3 ×(玩家数-1)`，再乘 `(1 + number_of_enemies/100)`。

## 9. 玩家实体

- **受伤无敌帧**：
  ```
  iframes = clamp( (受伤占比/0.15) × MAX, MIN, MAX )，MIN=0.2s，MAX=0.4s
  无尽模式：MIN/MAX 都除以 max(1, 无尽系数)
  ```
  受击后禁用受击区并启动无敌计时（仅"可闪避"类型的伤害触发；"无视无敌"的伤害如每秒掉血不受无敌帧影响）。
- **伤害结算**：先闪避（`randf() < dodge`，闪避受上限属性约束）→ 再 `hit_protection`（格挡次数）→ 否则 `max(1, round(dmg × 护甲系数))`，玩家护甲为系数减伤，与敌人的平减不同。
- 闪避触发：反伤（dmg_on_dodge）、闪避回血、闪避临时属性。受伤触发：受击爆炸、低血爆炸、temp_stats_on_hit、吸引周围治疗果。
- **拾取范围**：吸引区半径 `max(30, 150 × (1 + pickup_range/100))`；实际拾取区半径 32。吸引逻辑：治疗品仅未满血时吸引、金币就近抢占。
- **碰撞设计**：玩家本体与玩家、墙体碰撞，碰撞半径 24；玩家受击区检测敌人本体与敌方弹幕，半径 21；敌人受击区接受玩家武器与爆炸的判定。
- HP 再生：按固定间隔计时（间隔由 hp_regen 属性换算），每跳 1 点（受 hp_regen_bonus 低血加成、torture 特性改写）。
- 武器挂点：1~6 把按预设位置排布，>6 把按半径 `60+(n-6)×5` 圆周均布。
- 被增益怪强化：临时 +max_hp%（即时回血）/+速度/+攻速，5 秒后移除。

## 11. 玩家结构物

### 11.1 归属与属性继承

- 结构物记录归属玩家编号；属性在生成时由武器数值规则注入——即**继承该玩家的工程学等缩放属性、结构攻速/伤害/范围效果**；宠物型结构走宠物属性初始化通道。
- 可被诅咒（粒子+描边）；超上限时延迟 3 秒死亡。

### 11.2 炮塔通用规则

- 索敌：索敌范围 = 属性表 `max_range`，每帧取范围内最近且 ≥`min_range` 的目标。
- 冷却：最大冷却 = 基础冷却经结构攻速效果修正，实际每次取 `rand(0.7×, 1.3×)`；支持"每 X 发一次大装填"的额外冷却。
- 射击：每次 `弹数` 发，方向加 `±(1-精度)` 误差与弹体散布；射击动画速度自动加速至打满射速（上限 3 倍）。
- 被增益怪强化：伤害/范围/攻速按百分比临时放大。

## 13. 敌人数据的组织

- 每种敌人 = `表现与行为配置 + 属性表 + 可选专属逻辑 + 图鉴条目`。
- 图鉴条目：含 `is_boss/is_elite` 标记、展示 HP/伤害/速度/击退抗性/掉落与护甲信息，按击杀数解锁。
