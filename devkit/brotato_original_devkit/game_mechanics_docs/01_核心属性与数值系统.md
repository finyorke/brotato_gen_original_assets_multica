## 1. 数据架构总览

玩家数值由 4 个常驻数据层 + 1 个求值模块协同管理：

| 模块 | 职责 |
|---|---|
| 局内数据中枢（永久属性层） | 本局永久数据：每位玩家的玩家数据容器（属性、效果、物品、武器、金币、经验、等级），波次状态，伤害/治疗/吸血事件总线 |
| 玩家数据容器 | 单个玩家的数据载体；初始属性表与初始效果表定义全部属性与效果键的初始值 |
| 临时属性层 | 波内临时属性（移动/静止增益、受击增益、衰减增益、Boost 等），波结束清零 |
| 联动属性层 | 基于临时属性层扩展，由 `stat_links` 效果驱动的"按 X 换算 Y"联动属性，整组重算式更新 |
| 属性求值器 | 最终属性求值 / 上限截断求值入口、属性缓存、随机判定、属性转换 |

关键事实：

- **所有属性都是整数**，存储在玩家效果字典中（属性与"效果"共用同一个字典，键为字符串哈希）。初始化时会把初始属性表的全部属性合并进效果字典。
- 最多支持 4 名玩家（合作模式），所有缓存均为长度 4 的数组；无主伤害源使用保留玩家编号 123。
- 属性求值带**逐玩家缓存**。任何永久属性增减、临时/联动属性层变更、物品增删都会使该玩家的属性缓存失效。
- 属性变化通过脏标记在每物理帧合并后统一广播一次"属性已更新"事件。
- 武器/建筑/宠物的属性重算由面板重算调度器按队列分帧执行：每个条目最多延迟 60 帧，每帧至少重算 `ceil(排队帧龄/8)` 个。

## 2. 属性清单与默认值

### 2.1 16 个主属性（primary stats）

本游戏定义 16 个主属性，每个主属性附带图标与本地化名称，并接入属性增益与上限体系。设计初始值如下。

| 属性键 | 中文名 | 初始值 | 精确作用（公式见后文章节） |
|---|---|---|---|
| `stat_max_hp` | 最大生命 | **10** | 最大生命 = `max(1, min(stat_max_hp, hp_cap))` |
| `stat_hp_regeneration` | 生命再生 | 0 | 决定再生计时器间隔，每次跳恢复 1 HP，见 §7.1 |
| `stat_lifesteal` | 生命窃取 | 0 | 每点 = 每次武器命中 +1% 概率回 1 HP，见 §7.2 |
| `stat_percent_damage` | %伤害 | 0 | 所有武器/爆炸/燃烧伤害乘 `(1 + 值/100)`，见 §5 |
| `stat_melee_damage` | 近战伤害 | 0 | 仅通过武器的 `scaling_stats` 换算成面板伤害 |
| `stat_ranged_damage` | 远程伤害 | 0 | 同上 |
| `stat_elemental_damage` | 元素伤害 | 0 | 同上（燃烧 DoT 通常以它缩放） |
| `stat_engineering` | 工程学 | 0 | 同上（建筑/工具类武器缩放）|
| `stat_attack_speed` | 攻击速度 | 0 | 缩短武器冷却：`cd / (1 + 攻速/100)`，负值变成乘 `(1+|攻速|/100)`，见 §5.4 |
| `stat_crit_chance` | 暴击几率 | 0 | 每点 = 武器暴击率 +1%（加在武器基础暴击率上），受 `crit_chance_cap` 截断 |
| `stat_range` | 射程 | 0 | 远程武器 +1 射程/点，近战武器 +0.5 射程/点 |
| `stat_armor` | 护甲 | 0 | 减伤系数 `10 / (10 + 护甲/1.5)`，负甲增伤，见 §6.4 |
| `stat_dodge` | 闪避 | 0 | 每点 = 1% 完全免伤概率，默认上限 60，见 §6.3 |
| `stat_speed` | 移速 | 0 | 实际移速 = `基础450 × (1 + min(速度, speed_cap)/100)`（玩家基础移速设计值 450） |
| `stat_luck` | 幸运 | 0 | 影响掉落/品质判定，见 §10 |
| `stat_harvesting` | 收获 | 0 | 每波结束 = 收获值的材料+经验，见 §8 |

> 扩展属性（如 `stat_curse` 诅咒）由 DLC 内容标记为扩展属性后动态注入初始属性表。

初始值除注明外均为 0，括号内为非 0 默认值：

|---|---|
| `xp_gain` | 经验获取 %（获得经验乘 `1 + 值/100`） |

| `consumable_heal` | 拾取食物额外回复（与食物基础回复相加） |
| `burning_cooldown_reduction` / `burning_cooldown_increase` / `burning_spread` | 燃烧跳伤间隔减少/增加、燃烧传染数 |
| `piercing` / `piercing_damage` | 穿透次数 / 穿透伤害 %（降低每次穿透的衰减） |

| `pickup_range` | 拾取范围 |

| `knockback` | 击退 |

| `map_size` | 地图尺寸 % |
| `gold_drops` / `enemy_gold_drops` / `neutral_gold_drops` | 材料掉落量 %（总/敌人/树木） |
| `enemy_health` / `enemy_damage` / `enemy_speed` / `boss_strength` | 敌人三围与 Boss 强度 %（负面/危险等级用） |
| `explosion_size` / `explosion_damage` | 爆炸范围 % / 爆炸伤害 % |
| `damage_against_bosses` | 对 Boss 伤害 % |
| `weapon_slot` | 武器槽数量（默认 **6**） |
| `items_price` / `weapons_price` / `reroll_price` | 物品价格 % / 武器价格 % / 刷新价格 % |
| `harvesting_growth` | 收获每波增长 %（默认 **5**） |
| `hit_protection` | 格挡次数（每波开始装填，免疫整次伤害） |
| `structure_attack_speed` / `structure_percent_damage` / `structure_range` | 建筑攻速/伤害%/射程 |
| `accuracy` | 精准度 %（加到武器 accuracy 上） |
| `projectiles` | 额外弹丸数（同时增加 0.1 弧度散布/枚） |

### 2.3 重要布尔/容器型效果键（节选）

`torture`（饱腹/无法直接治疗、改为每秒回固定值）、`no_heal`（禁止一切治疗）、`hp_shop`（用最大生命当商店货币）、`hp_start_wave`/`hp_start_next_wave`（开波时生命百分比，默认 100）、`pacifist`/`cryptid`（波末按存活敌人/树折算材料）、`free_rerolls`、`double_boss`、`stat_links`（联动属性，§12.2）、`temp_stats_while_moving` / `temp_stats_while_not_moving` / `temp_stats_on_hit` / `temp_stats_on_dodge` / `temp_stats_per_interval`（临时属性来源，§12.1）、`stats_end_of_wave` / `stats_next_wave` / `stats_on_level_up`（波次/升级永久属性）、`convert_stats_end_of_wave` / `convert_stats_half_wave`（属性转换，§13）、`max_melee_weapons`/`max_ranged_weapons`（默认 999）、`min_weapon_tier`/`max_weapon_tier`（默认 0/99）、`next_level_xp_needed`（升级所需经验 %）、`gain_stat_*`（属性增益，§3.2）等。

## 3. 属性聚合与计算管线

### 3.1 最终属性公式

```
最终属性(stat, p) =
      永久层(stat, p)      # 永久属性层
    + 临时层(stat, p)      # 临时属性层
    + 联动层(stat, p)      # 联动属性层
```
（带逐玩家、逐属性缓存）

其中每一层都各自乘以"属性增益"系数：

- `永久层(stat, p) = 效果字典[stat] × 属性增益系数(stat, p)`
- `临时层(stat, p) = 临时属性值[stat] × 属性增益系数(stat, p)`
- 联动层基于临时属性层扩展，同样乘增益。

### 3.2 属性增益 gain_stat_X

```
属性增益系数(stat, p) = 1 + effects["gain_" + stat名] / 100
```
如 `gain_stat_max_hp = -50` 表示所有最大生命来源减半。**注意增益作用在每一层的"存量总和"上，是乘法修正而非逐次修正**；且 `effects[stat]` 中保存的是未乘增益的原始值。16 个主属性 + `stat_curse` 各有对应 `gain_stat_*` 键。

### 3.3 上限截断求值

仅 4 个属性有上限：

```
上限截断求值(stat, p) = min(最终属性(stat, p), effects[cap_key])
cap_key: stat_max_hp→hp_cap, stat_speed→speed_cap,
         stat_dodge→dodge_cap(默认60), stat_crit_chance→crit_chance_cap
```

使用处：最大生命（且再套 `max(1, …)` —— **最大生命永远 ≥ 1**）、移速与闪避、所有暴击率加成。

### 3.4 永久属性的写入

- 物品/角色/升级的效果通过"效果施加"直接修改效果字典；卸下物品执行"效果撤销"反向操作。
- 运行时永久加点走统一的永久属性增减接口，**无下限/上限钳制**——属性可为负值，负值的后果由具体使用处决定（如负收获扣材料、负攻速增加冷却、负甲增伤）。
- 角色本身也是一个"物品"（开局以物品形式加入物品列表），角色属性加成与物品同管线。
- 物品被添加且某主属性增量 > 0 时，会触发 `gain_stat_for_equipped_item_with_stat` 效果（如雪球：每获得一件带某属性的物品额外得属性）。

### 3.5 玩家实体同步

玩家实体在每次属性更新后把聚合结果写进自身面板：

```
max_health = max(1, min(stat_max_hp, hp_cap))
speed      = 450 × (1 + min(stat_speed, speed_cap)/100)
armor      = 最终属性(stat_armor)            # 不截断
dodge      = min(stat_dodge, dodge_cap)/100  # 概率小数
```
最大生命提高时当前生命同步 +增加量；降低时当前生命不变（仅在重置时拉满）。

## 4. 效果存储模型

所有物品效果共用同一个效果条目基类，按 `storage_method` 决定写入方式：

| StorageMethod | 行为 |
|---|---|
| `SUM`（默认） | `effects[key] += value` / 卸下时 `-= value` |
| `KEY_VALUE` | 向 `effects[custom_key]` 数组中追加/合并 `[key_hash, value]` 对（同 key 累加，减到 0 移除） |
| `REPLACE` | 记录原值后整体替换；卸下时还原 `base_value`。⚠️ 多个 REPLACE 效果互相覆盖时按后进先出还原 |
| `APPEND_KEY` | 把 key 追加进数组（去重） |
| `APPEND_KEY_VALUE` | 追加 `[key, value]`（卸下时按 key+value 精确匹配移除一条） |

物品基础数据含 `tier`（品质枚举：COMMON=0, UNCOMMON=1, RARE=2, LEGENDARY=3）、`value`（基础价格）、`effects` 数组、`is_cursed/curse_factor`（DLC 诅咒强化系数）。

特殊全局效果组（武器层面）在每次增删武器后重算：`unique_weapon_effects`（每种不同武器给一组效果）、`additional_weapon_effects`（每把武器一组）、`tier_iv_weapon_effects`（每把红装武器）、`tier_i_weapon_effects`（每把白装武器）。武器套装 `sets`：同套 ≥2 把激活套装加成，加成档位 = `min(同套数-2, 档数-1)`；`all_weapons_count_for_sets` 效果使所有武器计入每个套装。

## 5. 武器/伤害数值构建公式

所有武器面板按"武器基础值 + 玩家属性"在武器面板构建流程中生成。武器基础参数设计默认值：cooldown=60 tick、damage=1、accuracy=1.0、crit_chance=0.03、crit_damage=1.5、max_range=150、lifesteal=0。游戏固定 60 tick/秒。

### 5.1 伤害

```
# ① 缩放属性求和
scaled = base_damage + Σ_i ( 最终属性(scaling_stat_i) × ratio_i )      # ≥ 1
#   scaling_stat 可为任意属性；stat_levels 特殊 = 玩家等级 × ratio

# ② 百分比放大
damage = max(1, round( scaled × (set_bonus_dmg + percent_bonus + exploding_bonus) ))
percent_bonus   = 1 + stat_percent_damage/100        # 建筑用 structure_percent_damage
exploding_bonus = explosion_damage/100（仅带爆炸效果的武器）
set_bonus_dmg   = Σ 武器类别加成中的 %伤害项 / 100
```

补充修正（均发生在 ① 前）：
- `weapon_type_bonus`：按近战/远程类型直接加武器字段。
- `weapon_class_bonus`：按套装类别加字段；`lifesteal`/`crit_damage` 项按 `value/100` 加。
- `weapon_scaling_stats` 效果：向 scaling_stats 注入/叠加 `值/100` 的额外缩放。
- 同名武器叠加效果：同名武器每多 1 把加 `value`。
- 宠物且有"驯兽师"效果时，主缩放比率被复制到全部 4 种伤害属性。

### 5.2 爆炸伤害

```
explosion_damage = max(1, round( (base + Σ scaling) × (1 + %伤害/100 + 爆炸伤害%/100) ))
```
物品触发型爆炸（受击爆炸、低血爆炸等）的暴击率 = `效果基础暴击 + min(暴击率, 上限)/100`。

### 5.3 燃烧

全局燃烧（`burn_chance` 效果，携带独立燃烧参数）与武器自带燃烧合并：武器无燃烧时直接继承全局参数；有燃烧时 `damage += 全局damage`；之后同样过 scaling_stats 和 %伤害放大，`spread += burning_spread`。

### 5.4 冷却（攻速）

```
atk_spd = (stat_attack_speed + 武器attack_speed_mod) / 100    # 建筑/宠物为 0
cooldown(tick) = max(2, base_cd / (1 + atk_spd))      # atk_spd ≥ 0
cooldown(tick) = max(2, base_cd × (1 + |atk_spd|))    # atk_spd < 0（负攻速惩罚）
```
冷却下限固定为 2 tick；基础冷却低于 2 tick 先抬到 2。正攻速同时等比缩短 recoil 与 recoil_duration。另有 `minimum_weapon_cooldowns` / `maximum_weapon_cooldowns` 效果对"总循环时长（含挥舞/弹道时间）×60"做下/上限修正。建筑冷却用 `structure_attack_speed + structures_cooldown_reduction(按属性缩放)` 同公式换算。

实际开火冷却带随机扰动：`rand(max(1, cd−r), cd+r)`，`r = min(武器数×cd/5, 武器数×5)`（武器数按 `min(实际武器数, 6)` 计）；波开始时首个冷却最长 180 tick。

### 5.5 射程、击退、弹丸

- 近战：`max_range += stat_range / 2`；远程/建筑（用 structure_range）：`max_range += stat_range`，射程下限 25。
- `increase_projectile_speed_with_range`：弹速 `clamp(base + base/300 × range, 50, 6000)`。
- 击退：`knockback = clamp(base + 玩家knockback, min, max)`，只允许负击退的武器把玩家击退取负；`negative_knockback` 效果反转符号。
- 弹丸数：`nb_projectiles += projectiles效果`，散布 `+0.1/枚`。
- 暴击率：`crit_chance += min(stat_crit_chance, crit_chance_cap)/100`（建筑需 `structures_can_crit`）。
- 精准度：`accuracy += accuracy效果/100`。
- 吸血：`lifesteal += stat_lifesteal/100`（建筑不享受）。

## 6. 命中、暴击、闪避、护甲判定

### 6.1 命中（accuracy 不是未命中率，而是弹道偏移）

游戏没有"miss 判定"。`accuracy ∈ [0,1]` 决定射击方向随机偏移角（弧度）：

```
spread_offset = rand(-1 + accuracy, 1 - accuracy)   # 弧度
```
（建筑炮塔与宠物同理）。accuracy = 1 时完全精准；0.8 时 ±0.2 弧度（≈±11.5°）随机偏移，可能打空。

### 6.2 暴击（对敌判定）

```
is_crit = rand() <= 该次命中携带的暴击率       # 通用概率判定
if is_crit: dmg = round(value × crit_damage)   # 默认 crit_damage = 1.5
```
补充：命中燃烧目标时 `crit_on_hitting_burning_target` 效果强制暴击；概率秒杀效果按概率直接秒杀（伤害抬至当前生命）。暴击击杀触发 `gold_on_crit_kill`（逐条概率 +1 材料）与 `heal_on_crit_kill`（概率回 1 HP）。

### 6.3 玩家受击判定顺序

```
1. 闪避: if 可闪避 and rand() < 闪避概率 → 伤害 0
         闪避概率 = min(stat_dodge, dodge_cap)/100，dodge_cap 默认 60
2. 格挡: elif 剩余格挡层数 > 0 → 伤害 0，消耗一层
3. 护甲: value = max(1, round(dmg × armor_coef))   # 标记为"无视护甲"的伤害跳过此步
```
- **任何穿透到第 3 步的伤害至少为 1**。
- 受击后获得无敌帧：`iframes = clamp(伤害占最大生命比例 × 0.4 / 0.15, 0.2, 0.4)` 秒；无尽层数会按 `max(1, endless_factor)` 等比缩短上下限。被闪避的攻击同样触发无敌帧。
- 闪避成功触发：`dmg_on_dodge`（按某属性百分比反伤，先 `max(1, pct×属性)` 再乘 %伤害）、`heal_on_dodge`、`temp_stats_on_dodge`。
- 受击成功（伤害>0）触发：受击爆炸、低血爆炸（每波每效果 1 次）、`temp_stats_on_hit`、清除 `reset_on_hit` 的累积临时属性、`decaying_stats_on_hit`。
- `lose_hp_per_second` 失血：不可闪避、无视护甲、无视无敌帧。

### 6.4 护甲减伤公式

```
armor_coef = 10 / (10 + |armor| / 1.5)        # armor ≥ 0: 受到伤害的百分比
armor < 0 : armor_coef = (1 - 10/(10+|armor|/1.5)) + 1   # 即 2 - 正甲公式，最大趋近 2 倍
```
示例：armor=15 → 受伤 50%；armor=30 → 33.3%；armor=−15 → 受伤 150%。该公式是渐近曲线，**护甲无上限但收益递减，负甲最多双倍受伤**。

### 6.5 敌人受击的"护甲"

敌人不用上述系数公式，而是**平减**：`value = max(1, dmg − enemy.armor值)`。此前还要叠加：对高/低血量目标加伤效果（按当前血量比例阈值）、对燃烧目标的非元素加伤、按目标当前生命百分比的加伤（巨人腰带暴击、希腊火燃烧等，且按无尽系数除以 `max(1, endless_factor × 0.2)` 削弱）。最终 `dmg × (1 + Σ加伤%/100)` 后再减甲。

## 7. 生命恢复与吸血

### 7.1 HP 再生（hp_regen）

再生由一个固定步进计时器驱动，间隔规则如下：

```
hp_regen ≤ 0 → 间隔 99 秒（实际不恢复）
间隔(秒) = 5 / (1 + (hp_regen - 1) / 2.25)
```
即 1 点再生 = 每 5 秒回 1 HP；10 点 = 1 HP/秒；100 点 = 9 HP/秒（间隔 5/45 ≈ 0.11 秒）。每次计时器触发**固定回 1 HP**（单跳基准恢复量为 1），但 `hp_regen_bonus` 效果（如医生帽）在当前生命低于阈值% 时按 `1 × (1 + Σ倍率)` 放大单跳回复。满血时计时器停止；受击后立即重启计时。

`torture`（饱腹类）效果：覆盖为固定 1 秒间隔、每跳回 `torture` 值；同时治疗入口拒绝其他一切治疗来源。`no_heal` 则完全禁止治疗。所有治疗都被钳到不超过缺失生命。

### 7.2 吸血（lifesteal）

```
每次武器/弹丸命中：if rand() < 武器面板lifesteal → 回复事件
武器面板lifesteal = 武器基础吸血 + stat_lifesteal/100
回复量 = 1 HP（有 stat_double_lifesteal_bonus 效果则 2 HP）
```
**节流**：玩家身上的吸血节流计时器间隔 0.1 秒，计时未结束时吸血事件被丢弃 → 吸血回复至多 10 次/秒。建筑武器不享受吸血属性。

### 7.3 其他治疗渠道

- 食物（consumable）：回复 `max(0, 食物基础值 + consumable_heal)`；`consumable_heal_over_time > 0` 时改为持续回复（总量不变，时长=该值秒，每跳 1 HP）。
- 满血吃食物：`consumable_stats_while_max`（永久属性，可带每波上限第 3 参数）与 `temp_consumable_stats_while_max`（临时属性）。
- 拾取材料概率回血 `heal_when_pickup_gold`、击杀回血 `heal_on_kill`、暴击击杀回血 `heal_on_crit_kill`。
- 治疗触发 `dmg_when_heal`（按属性反伤随机敌人）。

## 8. 收获 Harvesting

### 8.1 每波结算（波计时器归零后执行）

```
val = harvesting_stat            # 最终属性求值（含临时/联动/增益）
    + round(存活敌人总数 × pacifist/100)        # 和平主义（蜂群 Horde 波减半；总数含因性能上限被移除的敌人）
    + 存活树数 × cryptid                        # 隐生物
    + 精英击杀加成                              # 保留项，设计值恒为 0
    + 存活敌人数 × materials_per_living_enemy
    + Σ 被魅惑敌人的材料价值（按材料价值规则换算）

val ≥ 0 → 加材料(val) 且 加经验(val)            # 收获同时给材料和经验！
val < 0 → 扣材料(|val|)                         # 负收获只扣材料，不扣经验
```
经验部分仍乘 `(1 + xp_gain/100)`（见 §9.1）。

### 8.2 每波增长

结算后若 `harvesting_stat > 0`，启动 0.66 秒一次性计时器，触发时：

```
普通波(wave ≤ 20):  harvesting += ceil(harvesting × harvesting_growth/100)   # 默认 +5%
无尽波(wave > 20):  harvesting -= ceil(harvesting × 20/100)                  # 无尽收获衰减固定 20%
```
增长是**永久属性修改**，向上取整（所以任何 >0 的收获每波至少 +1）。皇冠（item_crown）只在此处记录追踪值 `ceil(harvesting × crown%/100)`——皇冠本体加成以效果数值形式直接计入 harvesting_growth。

## 9. 经验与升级

### 9.1 经验获取

```
加经验(value): current_xp += value × (1 + 最终属性(xp_gain)/100)   # value ≤ 0 直接忽略
```
来源：拾取材料（每 1 材料 = 1 金币 + 1 经验，合作模式逐个轮询分配给各玩家）、收获结算（§8.1）。

### 9.2 升级所需经验

```
升级所需经验(level) = (3 + level)²
下一级所需 = 升级所需经验(当前等级+1) × (1 + next_level_xp_needed/100)
```
即升到 1 级需 16，2 级 25，3 级 36……20 级 529。升级时 `current_xp -= 所需`（保留溢出，经验足够时连续升级）。

### 9.3 每级奖励

1. **+1 最大生命**（固定）。
2. `stats_on_level_up` 效果逐条永久加属性（如角色成长）。
3. 获得一次升级四选一（波末处理）：升级品质由升级品质规则按等级决定：
   - 等级 5 → 固定 Tier II；等级 10/15/20 → 固定 Tier III；其余 5 的倍数（25,30,…）→ 固定 Tier IV；
   - 其他等级用品质概率公式（**用等级而非波数代入**，受幸运影响，见 §10.2）。
   - `level_upgrades_modifications` 效果对升级数值乘 `(1 + 值/100)`。
4. 武器槽位角色（`weapon_slot_upgrades`）在槽未满前升级选项被替换为"+1 武器槽"。

等级本身还可作为武器缩放属性 `stat_levels`（每级 × 比率计入武器伤害）。

## 10. 幸运 Luck

幸运参与且仅参与以下判定：

### 10.1 击杀掉落

合作模式下取**所有玩家幸运之和**：

```
luck = Σ_players 最终属性(stat_luck)/100

① 食物掉落概率:
   chance = min(1, 敌人base_drop_chance × (1 + luck))
   无尽波再除以 (1 + endless_factor)

② 宝箱概率:
   item_chance = 敌人item_drop_chance × (1 + luck) / (1 + 本波已掉宝箱数)
   item_chance += item_chance × Σ crate_chance%/100
   判定成功时食物升格为宝箱（Boss 在 20 波内掉传奇宝箱）
```

### 10.2 商店/升级品质

对 Tier IV→II 从高到低依次判定（单玩家幸运）：

```
wave_base_chance = max(0, (wave-1 - MIN_WAVE) × WAVE_BONUS_CHANCE)
luck ≥ 0: wave_chance = wave_base_chance × (1 + luck)
luck < 0: wave_chance = wave_base_chance / (1 + |luck|)
命中条件: rand ≤ min(BASE_CHANCE + wave_chance, MAX_CHANCE)
```

品质参数表：

| Tier | MIN_WAVE | BASE_CHANCE | WAVE_BONUS_CHANCE | MAX_CHANCE |
|---|---|---|---|---|
| I 普通 | 0 | 1.0 | 0 | 1.0（兜底） |
| II 罕见 | 0 | 0 | 0.06 | 0.60 |
| III 稀有 | 2 | 0 | 0.02 | 0.25 |
| IV 传奇 | 6 | 0 | 0.0023 | 0.08 |

该判定同时用于商店物品/武器与升级选项（升级以等级代入 wave）。`increase_tier_on_reroll` 等效果可在结果上 `clamp(tier+n, 0, 3)`。

此外幸运与其他属性一样可作为武器 scaling_stat、可被联动/转换效果引用。**幸运不影响**暴击、闪避、吸血等纯随机判定。

### 10.3 通用概率判定

通用概率判定规则：`chance == 0` 必失败，否则 `rand() <= chance`。诅咒类按渐近公式换算概率：`诅咒系数(value, max) = max × (1 − 1/(1 + value/50))`。

## 11. 材料（金币）经济数值

与属性系统强耦合的经济公式：

- **掉落量**：`value = base × (1+coop系数) × (1 + (gold_drops + 对应类型drops)/100)`，下限 `0.5 × base`。距离缩放效果 `scale_materials_with_distance` 再乘 `(1+pct/100)`。小数部分按概率进位成 1 颗。
- **掉落衰减**：第 5 波起敌人有 `max(0.5, 1 − wave×0.015)` 概率掉材料，精英群波再 ×0.65。场上材料超过 50 颗时不再生成新实体，而是随机给已有材料 +1 价值并放大贴图。
- **拾取**：`value ×= (1 + increase_material_value/100)`；`chance_double_gold`% 概率 ×2；然后逐 1 轮询分给各玩家并各给 1 XP。
- **HP 商店**（`hp_shop`）：货币显示/扣除改为最大生命属性本身。
- **波外材料（bonus_gold）**：波间掉的材料进入公共池，下一波掉落时每颗最多翻倍消耗池子；`convert_bonus_gold` 效果（工匠）把每 N 池材料转为属性。
- **商店标价**：`price = max(1, (v + wave + v×wave×0.1) × (1 + (items_price + 特定物品price)/100) × (1 + endless_factor/5))`，武器另乘 `(1 + weapons_price/100)`。刷新价：`delta = int(max(1, 0.4×wave×√(1+endless_factor)))`，`价格 = int(wave×0.75) + delta×(1+已付费刷新次数)`，再乘 `max(0.1, 1+reroll_price/100)` 后向上取整（免费刷新不计入次数）。回收价 = 商店价 × `clamp(0.25 + recycling_gains/100, 0.01, 1)`。

## 12. 临时属性与联动属性机制细节

### 12.1 临时属性层

- 结构：每玩家一张 `{属性键: 整数}` 表，初始全 0。
- **清零时机**：波结束、玩家死亡、新一局开始。
- 写入来源：
  - `temp_stats_while_moving` / `while_not_moving`：开始/停止移动即加/移除；其中 `percent_materials` 项不进临时属性层，而是按计时器每跳直接加/扣 `max(1, |pct|×当前材料)` 材料（可带上限参数）。
  - `temp_stats_on_hit`（受击时叠加）、`temp_stats_on_dodge`（闪避时叠加）。
  - `temp_stats_per_interval`：每 `interval` 秒 +value，`reset_on_hit=true` 的累积量受击全部清除。
  - `decaying_stats_on_consumable` / `on_hit`：+value 持续 duration 秒后移除；同名同时长效果刷新时间而非叠加。
  - `stats_next_wave`：上波购买的"下波临时属性"开波注入。
  - Boost（如蜗牛魔王激素）：`max_hp` 按当前最大生命百分比临时增加并直接加当前生命，结束时若超过上限把当前生命压回。
  - 半波属性转换的产物（`permanent=false`，见 §13）。

### 12.2 联动属性层

`stat_links` 效果格式：`[stat_to_tweak, nb_to_tweak, stat_scaled, nb_stat_scaled, perm_stats_only]`，语义"每 `nb_stat_scaled` 个 X 给 `nb_to_tweak` 点 Y"：

```
amount = int( nb_to_tweak × actual_scaled / nb_stat_scaled )   # 向零取整
```

- **整组重算**：每次重算先清空整个联动临时表，再按全部链接重新累加。任何物品增删都会触发重算。
- 缩放源 `stat_scaled` 支持：材料数、建筑数、宠物数、燃烧敌人数、存活敌人数、存活树数、玩家缺失生命百分比（`floor(缺失% / 1)`，经缺失生命换算规则）、不同物品数 / 普通品质物品数 / 传奇物品数 / 重复物品数、指定物品数量（`item_*`）、空武器槽数，以及**任意其他属性**（此时 `perm_stats_only` 决定是否计入对方的临时属性层；联动层之间互相不可见，防循环）。
- **动态源每 0.5 秒刷新**：若链接引用了材料/敌人/缺血等实时量，该玩家被打上半秒刷新标记，主流程每半秒整组重算一次。
- 同公式的一次性版本是"一次性存量换算"（武器"按存量给一次加成"效果用）。

## 13. 属性转换与波次属性事件

### 13.1 属性转换规则

`convert_stats_end_of_wave`（永久，波结束触发）与 `convert_stats_half_wave`（临时，波进行到一半触发）共用同一套换算：

```
chunks = max(0, floor(源属性值 × pct/100 / value))     # 源为 materials 时取材料数
基础移除 = chunks × value ; 基础获得 = chunks × to_value
实际移除 = round(基础移除 / 源属性增益) ; 实际获得 = round(基础获得 / 目标属性增益)
```
**除以增益是为了让"显示数值"与转换承诺一致**（之后读取时会再乘回增益）——这是属性增益与转换叠加时的关键隐藏规则。半波版本写入临时属性层（波末随临时属性消失）。

### 13.2 波末/开波属性事件时序

1. 波计时器归零：`stats_end_of_wave` 永久属性逐条入账 → `convert_stats_end_of_wave` → 收获结算（§8）→ 清场 → 临时属性层清零。
2. 开波：按 `hp_start_wave`/`hp_start_next_wave` 设定开局生命（`max(1, 最大生命×pct/100)`）→ `gain_pct_gold_start_wave` 按当前材料百分比加/扣材料（无尽波负值变 −100%）→ 注入 `stats_next_wave` 临时属性 → 检查半血以下属性效果。

## 14. 无尽模式系数与其他隐藏机制

### 14.1 无尽系数

```
endless_wave  = max(0, wave - 20)
endless_mult  = 2 + max(0, (wave - 35) × 0.2)
endless_factor = (endless_wave × (endless_wave+1) / 2) / 100 × endless_mult
```
用于：敌人按生命百分比的加伤削弱（÷ `max(1, factor×0.2)`）、无敌帧缩短、商店价格（×(1+factor/5)）、刷新价、食物掉率衰减。第 20 波后每 10 波额外 +1 精英。

### 14.2 其他隐藏/易漏机制汇总

- **属性可为负且基本不钳底**：负移速会得到负的面板移速，但实际移动取 `max(0, …)`；负收获扣材料；负攻速按独立公式放大冷却；负幸运按除法削减品质/掉率；负甲按 §6.4 增伤。
- **最大生命下限 1**、**穿甲后伤害下限 1**、**武器伤害下限 1**。
- **闪避上限默认 60%**，是字典里的效果值（`dodge_cap`），物品可修改（如骑士盾类效果可改写该键）；暴击率上限默认无穷，同理可被物品改写。
- **吸血是概率制 + 0.1 秒节流**，不是按伤害百分比回血。
- **accuracy 是散布角不是未命中率**。
- **收获给等量经验**（被很多攻略忽略，xp_gain 同样放大收获经验）。
- **`stats_end_of_wave`、收获增长等永久成长全部走永久属性增减接口，会立刻反映到联动属性与下波面板**。
- 角色/难度修改也是效果条目（难度选项的效果在重置/重开时施加）。
- 合作模式提供效果聚合规则（武器效果存在性检查、全队效果求和、全队效果拼接）：掉落类判定常用**全队属性和**（幸运、crate_chance、gold_drops 的求和/平均差异见 §10.1、§11）。
- 合作模式材料平分采用全局轮询指针，保证跨拾取均匀。
- 全队叠乘聚合：`total += stat_i × (1 + total/100)`（用于全队百分比类效果）。
- 商店锁定/禁用（ban，禁购令牌上限 8）、物品回收追踪等不改属性，只动经济。
- 玩家死亡即清空该玩家临时属性层并停用全部计时器，合作模式下其余玩家继续。
- 升级/物品的属性数值本身按品质分档配置（如近战伤害 Tier I = +2、Tier IV = +8），由 §9.3 的品质判定抽取。

```
enemy_damage (敌人面板)
 → 玩家受击结算
   ├─ 无敌帧检查（0.2~0.4s，被打/被闪避都触发）
   ├─ 闪避: rand() < min(dodge, dodge_cap)/100 → 0 伤害 + 闪避反伤/回血/临时属性
   ├─ 格挡: hit_protection > 0 → 0 伤害
   ├─ 护甲: dmg = max(1, round(value × 10/(10+armor/1.5)))   # 负甲→最多×2
   ├─ 扣血、受击爆炸/低血爆炸/受击临时属性/重启回血计时器
   └─ 生命 ≤ 0 → 死亡（清临时属性）
```

## 附：玩家武器对敌一次攻击的完整数值流程

```
基础伤害 + Σ(属性×缩放比)            # ≥1
 × (套装%伤 + 1 + %伤害/100 + 爆炸%/100)  # ≥1, round
 → 暴击 roll (武器暴击 + min(暴击率,上限)/100): ×crit_damage
```
