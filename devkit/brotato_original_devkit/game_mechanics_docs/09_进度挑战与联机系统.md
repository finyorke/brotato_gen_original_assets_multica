# 09 进度、挑战与联机系统

> 本章覆盖：解锁/进度系统、挑战（成就）系统、存档系统、本地合作（Co-op）模式、宠物系统、胜利/无尽条件，以及影响玩法的全局设置。

## 1. 解锁与进度系统

进度存档系统为全局常驻服务，维护以下解锁清单：

| 字段 | 含义 |
|---|---|
| `zones_unlocked` | 已解锁地图区域（zone） |
| `characters_unlocked` | 已解锁角色（哈希 ID） |
| `weapons_unlocked` / `items_unlocked` | 已解锁武器 / 道具 |
| `upgrades_unlocked` / `consumables_unlocked` | 已解锁升级 / 消耗品 |
| `challenges_completed` | 已完成挑战 |
| `systems_unlocked` | 已解锁系统（目前仅"禁用系统 Ban System"） |
| `difficulties_unlocked` | 每角色×每区域的危险等级记录（角色难度信息数组） |
| `killed_enemies` / `killed_by_enemies` / `items_bought` | 图鉴统计：击杀过的敌人、被谁杀死过、买/获得过的物品次数 |
| `data` | 全局累计计数器（见 1.3） |

游戏启动时把所有标记为"默认解锁"的内容自动加入解锁清单。基础升级（每个属性 4 个等级，如 `upgrade_armor_1..4` 均默认解锁）默认全解锁，升级本身不是挑战奖励。

### 1.1 危险等级（Danger）解锁规则

- 每个角色、每个区域各保存一条区域难度信息，字段包括：
  - `difficulty_selected_value`：该角色上次选择的危险等级（单人模式下记忆，合作模式不写入）；
  - `max_selectable_difficulty`：可选上限（0~5，最大危险等级为 5）；
  - `max_difficulty_beaten` / `max_endless_wave_beaten`：最佳通关 / 最佳无尽记录（难度成绩记录）。
- **解锁条件**：胜利结算时若 `当前危险+1 ≤ 5`，则把**所有角色、所有区域**的 `max_selectable_difficulty` 提升到 `clamp(当前危险+1, 原值, 5)`——即危险等级解锁是**全账号共享**的，不按角色单独解锁。
- 读档时还会做一次"全局拉平"：取所有记录中的最大可选难度，并同步给所有角色/区域。
- 难度选择界面：当 `难度值 ≤ max_selectable_difficulty` 或该难度默认解锁时可选。
- 难度成绩记录包含通关时的：危险值、波数、敌人血/伤/速三项辅助缩放系数、重试次数（retry）、使用的禁用（ban）次数、是否合作模式。新纪录的"更优"判定：危险或辅助系数更高 > 同等条件下重试更少 / ban 更少 / 从合作改为单人。无尽纪录按设置 `endless_score_storing`（最高波数优先或最高难度优先）比较。
- 角色选择界面按该角色已通关的最高危险给角色头像染色分级。

### 1.2 角色/武器/道具解锁

全部通过挑战系统发放，见第 2 节挑战清单。胜利时还会按角色完成 `chal_<角色名>` 挑战（解锁对应物品/武器奖励）；Steam/GOG 上 DLC 区域（zone 1，"abyss"）的角色通关有独立平台成就 `chal_xxx_abyss`。

### 1.3 全局计数器

`fruit_eaten_full_hp`（满血吃水果数）、`chal_hourglass_quit_wave`（沙漏挑战标记）、`is_unlock_all_save`（是否一键全解锁存档）、`run_won`、`run_started`、`enemies_killed`、`materials_collected`、`trees_killed`、`steps_taken`（每波取所有玩家中最大步数累计）、`enemies_killed_far_away`（远距离击杀，DLC 挑战用）、`evil_mob_killed` / `evil_mob_killed_by`（吞吞怪 Gobbler 击杀/被杀）。

## 2. 挑战 / 成就系统

挑战系统为全局常驻服务，本体注册 **113 个挑战**。每条挑战数据包含：`value`（达成数值）、`stat`（自动监测的属性）、`reward_type`（ITEM/WEAPON/ZONE/STARTING_WEAPON/CONSUMABLE/UPGRADE/CHARACTER/DIFFICULTY/SYSTEM）、`reward`（奖励内容）、`additional_args`。

判定机制：
- **属性型**（`stat != ""`）：属性变更时自动检查；`value > 0` 时达到即成，`value < 0` 时低于即成；
- **累计型**：每波结束统一检查击杀/采集/砍树/远距击杀/吞吞怪计数；
- **事件型**：在对应玩法逻辑处直接触发完成；
- 完成时立即解锁奖励、写入存档并同步平台成就；一键全解锁存档（`is_unlock_all_save`）不再上报平台成就。

## 3. 存档系统

### 3.1 存档文件结构（V3，当前格式）

- **3 个存档槽位**，每槽两个 JSON 文件：
  - `save_v3_<id>.json`：进度（解锁、挑战、难度记录、图鉴计数、全局统计、版本号）；
  - `run_v3_<id>.json`：当前进行中对局的完整状态。
- 另有 `settings.json` 保存全部设置 + 当前槽位 id。
- **写入流程**：先写 `.tmp` 临时文件，再拷贝为正式文件（防写坏）；Windows 上额外滚动备份 `save_v3_<id>_00.bak ~ _04.bak`，最多 5 份。读档损坏时依次尝试备份。
- 对局状态（run state）内容：所有玩家的对局数据序列化（角色、血量、等级、经验、金钱、武器/道具及其效果、禁用列表、ban 令牌等）+ 全局状态（波数、危险、区域、奖励金、精英/头目刷新表、本局完成的挑战、商店锁定/商店当前货品、重掷计数、无尽标记、合作标记、启用的 DLC 等）。

### 3.2 版本迁移

读档顺序：V3 → 失败且为 0 号槽时尝试 V2（成功后再合并旧 V1，取最大值合并）→ V1（多份历史存档文件名，主机平台路径不同）→ 都没有则尝试从回退目录拷贝，最后新建。V1/V2 中设置与进度混存，迁移时一并合并；V3 起物品 ID 全部转为整数哈希存储。

槽位操作：重置（写空白默认解锁）、一键全解锁（标记 `is_unlock_all_save=1`，结算界面显示标记图标且不上报平台成就）、槽间复制。

### 3.3 DLC 管理

启动时加载 DLC《Abyssal Terrors》内容包并检测平台所有权；可在设置中停用 DLC（`deactivated_dlcs`）。继续存档时校验存档用到的 DLC 是否可用（DLC 区域或 DLC 角色缺失则禁用继续）。

## 4. 本地合作模式（Co-op）

本地最多 **4 人**（Switch 平台限 2 人）；同一设备长按"确认"键 **0.7 秒**加入。

### 4.1 敌人与材料随人数缩放

每多 1 名玩家：
- **敌人数量 +30%**（应用于每波刷怪数量）；
- **敌人生命 +30%**（乘法系数 `1 + 0.3×(人数-1)`）；
- **敌人伤害 +8%**。

材料补偿：每个材料价值乘 `1 + 补偿系数×(人数-1)`，其中补偿系数 `= ((人数/敌人倍率) - 1)/(人数-1) × 0.8`（材料补偿因子 0.8）。即材料总量不足以让每人都拿满单人量，约为单人量的折中。

### 4.2 金钱与经验：完全均分

- 任何玩家捡起材料后，其价值按 1 点 1 点**轮转分配给所有玩家**（含金钱与等量经验；用轮转指针记录下一个受益者）——**没有独立拾取收益，钱和经验全队均分**；但每人各自的 `xp_gain`、增益效果按各自属性结算。
- 每名玩家**钱包独立**、等级/升级独立。
- 波结束未捡的材料吸入袋子变成全队共享的"奖励金 bonus_gold"（下波材料增值）。
- 幸运对箱子掉率取**全队相加**；`gold_drops` 类效果按全队总和÷人数（平均）生效；敌人血量/伤害/数量等全局效果取全队总和。

### 4.3 商店：同屏并行、各自独立

- 合作使用专用合作商店界面，**不是轮流制**：4 名玩家各有一个面板，**各自独立的 4 件货品、独立重掷价格/次数、独立锁定、独立购买**（所有商店状态均为按玩家索引的四槽数组）。
- 所有玩家都按下 GO 才进入下一波。
- 任意玩家可随时暂停（游戏内每个玩家的暂停键均有效）。
- 合作下商店物品视觉上对所有人解锁显示。
- 波末升级/箱子结算用合作升级界面统一处理。

### 4.4 共享战利品（share_coop_loot）

设置项 `share_coop_loot`（默认开）：波内捡到的**道具箱**在波末分配给"待处理箱子最少"的玩家（平局随机），而不是捡到的人；关闭则归拾取者。治疗类消耗品（水果等）始终作用于**拾取者本人**。

### 4.5 死亡与复活

- 玩家死亡后**本波内不会复活，没有复活倒计时或复活代价**：血条隐藏、高亮关闭；只要还有任一存活玩家，本波继续。
- 击杀掉落类效果（死亡爆炸、死亡弹幕等）只在**存活玩家**身上结算。
- **全员阵亡才算失败**；下一波开始时所有玩家（含上波死亡者）重新生成并按 `hp_start_wave` 比例回满血（每波都重建玩家实体）。
- 开启无障碍选项 `retry_wave` 时，第 2 波起团灭可原地重打本波（恢复波开始状态，`retries+1` 记入成绩）。

### 4.6 其他差异

- 共享一个动态摄像机；选项 `lock_coop_camera` 可锁定；合作下屏幕受伤红晕（vignette）禁用。
- 出生点：多人时在地图中心按 2×2 网格错开 100 像素。
- 地图尺寸效果取全队总和。
- 每名玩家独立选角色、独立选起始武器；难度由 1P 确认且不记忆 `difficulty_selected_value`。
- 禁用（Ban）系统在合作下每人独立 8 个令牌（令牌上限 8），有专用按键提示。
- 胜利时**每名玩家的角色分别**获得角色通关挑战与危险记录，且成绩记录标记为合作（之后用单人以同条件通关可刷新为"更优"记录）。
- 续档：合作存档继续时进入专门的合作续档界面，需逐个重新连接手柄/键盘。
- 玩家颜色固定 4 色，UI、宠物高亮按玩家着色。
- 无尽模式开关在合作商店中仅 1P 面板显示。

## 5. 宠物系统

宠物为 1.1 版（Paws 'n' Claws）加入的本体内容（解锁条件见 2.5），DLC 可附加"诅咒"变体。

- **获得方式**：道具携带宠物效果：把效果对象加入该玩家的 `stat_pets` 列表；标记为构筑物的宠物同时计为构筑物。波开始时由实体生成器按玩家队列生成宠物实体。
- **归属与继承**：每只宠物绑定主人玩家索引；
  - 移动速度按主人速度属性放大：`speed × max(1 + 玩家速度%/100, 1)`；
  - 战斗数值通过武器数值系统用**主人属性**初始化（如 Bonk 狗近战按近战宠物属性初始化，其爆炸大招按基础属性初始化，即继承主人近战/元素/暴击等加成）；
  - 合作模式下宠物高亮使用主人玩家颜色。
- **目标管理**：全局宠物目标表仅维护"每只宠物已锁定目标"（防止多宠物抢同一目标），每波重置。
- **数量统计**：宠物总数 = `stat_pets` + 果冻护盾计数 + 标记为宠物的构筑物，供"按宠物数缩放"的效果使用。
- **个体行为**：Bonk 狗（突进近战+周期爆炸）、猫特林机枪（远程射击）、布雷机器人（布雷）、焰蜥蜴（火焰）、医蛾（治疗向）、果冻护盾（格挡投射物，计入 `stat_jellyshield_count`）、搜刮虫虫（吸取材料：瞬时吸金触发时材料优先飞向虫虫）、替罪羊（可被敌人锁定，嘲讽型）、鼠斯拉 Ratzilla。
- 波结束时宠物若在主人 110 像素内且主人存活，播放"被摸摸"动画。
- 无障碍选项：`pet_opacity`（透明度）、`pet_highlighting`（高亮）。

## 6. 胜利条件与无尽模式

- 标准局共 **20 波**。**胜利** = 完成第 20 波（或在第 20 波杀光全部头目立即结算：杀死最后一只头目/精英时波计时直接压到 0.1 秒；全部末波头目阵亡直接判胜）。第 20 波头目从当前区域头目池随机 1 只，`double_boss` 效果则 2 只。第 20 波后的隐藏头目双结局属于 DLC 区域内容，不在本体范围。
- **失败** = 全员阵亡且未到第 20 波（或第 20 波全灭且头目未清）。
- **无尽模式**：角色选择界面开关（记忆于 `endless_mode_toggled`）或第 19 波商店一次性按钮。第 20 波杀完头目后继续补刷怪群；无尽中**全员阵亡即视为"获胜"结算**（波数 > 20 时）。
- 无尽强度：`endless_factor = ((w-20)(w-21)/2)/100 × (2 + max(0,(w-35)×0.2))`；敌人血量额外 ×2.25 系数、诅咒进一步放大；每 10 波追加精英；20 波后每次收获结算收获值衰减 20%；存钱罐/皇冠在无尽中被禁入商店池，负的开波金钱百分比在 20 波后变为 -100%。
- 无尽纪录按角色×区域保存 `max_endless_wave_beaten`，比较口径由设置 `endless_score_storing` 决定（最高波 / 最高难度）。

## 7. 其他全局规则与影响玩法的设置

- **禁用系统（Ban System）**：危险 ≥1 通关解锁；总开关 `ban_mode_toggled`（默认开）；每名玩家每局 8 个令牌；在商店按禁用键（或波末箱子界面的 Ban 按钮）可"禁用"非武器物品：当场获得等同回收价的材料并将该物品**从该玩家本局商店池中永久剔除**；使用过的 ban 次数计入通关成绩。
- **影响玩法的设置项**：
  - `enemy_scaling`（敌人血/伤/速 3 个无障碍系数，进局时快照并记入成绩）；
  - `retry_wave`（团灭重试本波）、`share_coop_loot`、`lock_coop_camera`、`keep_lock`（商店锁定跨波保留）；
  - `endless_mode_toggled`、`ban_mode_toggled`、`zone_selected`、`play_mode`；
  - `optimize_end_waves`（波末材料不做吸入动画、直接折算为奖励金）；
  - 手动瞄准（`manual_aim`、`manual_aim_on_mouse_press`）、`mouse_only`；
  - `endless_score_storing`（无尽纪录口径）、`no_item_appearance`（不显示道具外观）。
- **统计/图鉴**：`killed_enemies`、`items_bought` 达到各内容设定的图鉴阈值后解锁图鉴描述；标题页进度百分比由挑战（排除难度类）/物品/武器/敌人四项计算。
- **平台成就映射**：PS4/PS5/Xbox 平台用编号映射表同步成就，Switch 不上报；Steam 角色成就区分本体/深渊区域两套。
