## 1. 材料（Materials）机制

材料（绿色小方块掉落物）是本作唯一货币，同时兼任**金钱**与**经验**两种资源。

### 1.1 掉落

- 敌人死亡时进行掉落结算：
  - **掉落概率随波数衰减**：第 1~4 波必掉；第 5 波起概率为 `max(0.5, 1 - wave * 0.015)`，即每波 -1.5%，下限 50%。
  - 蜂群波（horde wave）额外乘 `0.65`。
  - 标记为"必掉消耗品"的单位（如树）强制 100% 掉落。
- **掉落价值**计算：
  - 基础值 = 单位数值面板的 `value`；
  - 联机时乘 `(1 + 联机系数)`；
  - 受 `gold_drops`（全体材料掉落 %）、`enemy_gold_drops`（敌人材料掉落 %）、`neutral_gold_drops`（树木材料掉落 %）效果加成，多人时这些效果按人数平摊；
  - 下限保护：`value = max(value, 0.5 * base_value)`（下限系数 0.5）；
  - 部分效果还按"击杀距离"缩放（`scale_materials_with_distance`）。
- **小数部分按概率取整**：生成数量 = `floor(value)`，余数作为额外掉 1 个的概率。

### 1.2 地面材料上限（溢出合并）

- 场上最多同时存在 **50** 个材料实体。
- 超出时不再生成新实体，而是**随机挑一个已有材料 +1 价值并放大其贴图**，尺寸上限 2.0 倍。即视觉上 1 个大材料可能值多点。

### 1.3 拾取：金钱与经验双重收益

材料拾取结算：

- 每 1 点材料价值 = **+1 金钱且 +1 经验**；联机时按"轮转指针"在玩家间逐点均分。
- 拾取时依次结算的效果：
  - `increase_material_value`：拾取价值百分比加成；
  - `chance_double_gold`：概率双倍（金属探测器）；
  - `heal_when_pickup_gold`：概率回 1 HP（可爱猴子）；
  - `dmg_when_pickup_gold`、`reload_when_pickup_gold`（拾取触发伤害/武器装填）。
- 经验入账：实际经验 = `value * (1 + xp_gain / 100)`。

### 1.4 袋子（Bag）与波次结束回收（bonus_gold）

波次结束清场时：

- **地面上未拾取的材料不会消失**：全部被吸到屏幕角落的"材料袋"。
- 进袋的材料不归属任何玩家 → 计入全局补偿池 `bonus_gold`，**不直接给钱也不给经验**。
- **下一波偿还机制**：之后每生成 1 个材料时，若 `bonus_gold > 0`，该材料价值翻倍（`gold.value += min(gold.value, bonus_gold)`）并从 bonus_gold 池中扣除，外观放大 1.25 倍。即袋内材料以"下波材料双倍"的形式逐步返还。
- 设置项 `optimize_end_waves` 开启时跳过吸附动画，直接求和入袋，此时仍逐个结算 `chance_double_gold`。
- 工程师（Builder）角色特例：`convert_bonus_gold` 效果会把 bonus_gold 按比例转化为炮塔属性而非材料。
- 波次结束时地面**消耗品**（无伤害效果的）也会自动吸到存活玩家身上。

### 1.5 收获（Harvesting）属性结算

波次结束时进行收获结算：

- 每名玩家获得 `收获值` 的**金钱 + 等量经验**（负收获只扣钱不扣经验）。
- 总收获 = `harvesting + pacifist加成 + cryptid加成 + 精英击杀加成 + 存活敌人加成 + 魅惑敌人价值`；蜂群波 pacifist 减半。
- **收获成长**：结算后启动计时器，每波收获属性自增 `harvesting_growth%`（默认 5%，向上取整）。皇冠（Crown）道具会提高该成长值。
- **无尽模式衰减**：超过第 20 波后，每波收获属性反而**减少 20%**（无尽收获衰减率 = 20）。

### 1.6 波次开始的"利息"

波次开始时：`gain_pct_gold_start_wave` 效果（如存钱罐 Piggy Bank，tier2、max_nb=1）使玩家获得 `当前金钱 × pct%`。负值（诅咒类）照常扣除；无尽模式下正值利息失效，负值会变成 -100%（清空金钱）。无尽模式商店池也会直接禁售存钱罐与皇冠。

## 2. 商店机制

每波结束（处理完升级和宝箱后）进入商店界面。

### 2.1 商店物品数量与池构成

- 每个玩家的商店固定 **4 个栏位**。
- 物品池在开局按"已解锁 + 可被商店刷出（can_be_looted）+ max_nb≠0"分 tier 装入分级奖池，分为道具 / 武器 / 消耗品 / 升级项四类池。

**武器 vs 道具比例**：

| 条件 | 规则 |
|---|---|
| 当前波 ≤ 2 | 保底 2 把武器 |
| 当前波 ≤ 5 | 保底 1 把武器 |
| 其余栏位 | 每格 35% 武器 / 65% 道具（武器出现率 = 0.35） |

- 已锁定的武器计入保底数。
- `minimum_weapons_in_shop` 效果可抬高保底；玩家没有武器栏位（武器栏=0 角色）时全部刷道具。
- `guaranteed_shop_items` 效果（角色/道具给的"必刷某道具"）优先占用道具栏位。

**武器倾向性**（同一次刷新只掷一次倾向骰）：

- 概率 `0.2 + max(0,(6-wave)) * 0.01` → 只保留玩家已持有的同名武器（基础同名倾向率 = 0.2，前 5 波额外摊入 0.05）；
- 否则概率 `0.35 + max(0,(6-wave)) * 0.04` → 只保留与玩家武器同系（套装）的武器（基础同系倾向率 = 0.35，前 5 波额外摊入 0.2）。

**道具倾向性**：5%（标签倾向率；被增强效果提升时 10%）概率只保留带角色 `wanted_tags` 的道具。

**池过滤**：角色禁用道具/道具组（`banned_item_groups`，按预设分组表匹配）、玩家手动 ban 的道具、`no_melee_weapons / no_ranged_weapons / no_duplicate_weapons / remove_shop_items`(按 tag，如 structure) 等效果、达到 `max_nb` 上限的道具（持有+商店中+锁定的合计）都会被移除。过滤后池空则用备用池（仅做硬性过滤），再空则直接从全 tier 池抽。

**特例**：美西螈（Axolotl）被抽中时有 50% 重抽一次（降低出现率一半）。

### 2.2 Tier(1-4) 出现概率公式

各品质档位配置（最低波数 / 基础概率 / 每波加成 / 概率上限）：

| Tier | MIN_WAVE | BASE_CHANCE | WAVE_BONUS_CHANCE | MAX_CHANCE |
|---|---|---|---|---|
| 1 普通 (COMMON) | 0 | 1.00 | 0 | 1.00 |
| 2 罕见 (UNCOMMON) | 0 | 0 | 0.06 | 0.60 |
| 3 稀有 (RARE) | 2 | 0 | 0.02 | 0.25 |
| 4 传说 (LEGENDARY) | 6 | 0 | 0.0023 | 0.08 |

品质抽取公式——roll 一个 `rand ∈ [0,1]`，**从 tier4 往 tier1 依次判定**，命中即停：

```
wave_base = max(0, (wave - 1 - MIN_WAVE) * WAVE_BONUS_CHANCE)
wave_chance = wave_base * (1 + luck)        # luck = 幸运/100，luck<0 时改为 ÷ (1+|luck|)
命中条件: rand <= min(BASE_CHANCE + wave_chance, MAX_CHANCE)
```

即（幸运为 0 时）：tier4 从第 8 波开始有 `(wave-7)*0.23%` 概率，封顶 8%；tier3 从第 4 波开始 `(wave-3)*2%`，封顶 25%；tier2 从第 2 波 `(wave-1)*6%`，封顶 60%；否则 tier1。幸运对"随波数增长部分"乘算，但**无法突破 MAX_CHANCE 上限**。

- 武器 tier 还会被 `min_weapon_tier / max_weapon_tier` 效果钳制。
- `increase_tier` 参数（金鱼等"重随提升 tier"道具）在结果上 `clamp(tier + increase_tier, 0, 3)`，触发后消耗来源道具。

### 2.3 价格公式

商店标价规则：

```
V = 物品基础价 value                       # 武器再乘 (1 + weapons_price/100)
价格 = max(1, (V + wave + V*wave*0.1)
            * (1 + (items_price + 指定道具price)/100)   # 道具价格效果
            * (1 + endless_factor(wave)/5))             # 无尽通胀
```

- `wave` 用的是**该商品被刷出时的波数**（`wave_value`）。
- `items_price`（全体物品价格 %）、`specific_items_price`（指定物品价格 %，如优惠券 Coupon 按 id 匹配）、`weapons_price`（武器价格 %）皆为对局内效果键。
- 无尽系数 `endless_factor`：`((max(0,wave-20) * (wave-19)) / 2) / 100 * (2 + max(0,(wave-35))*0.2)`，20 波后价格快速膨胀。
- **HP 商店**角色（`hp_shop` 效果）：价格改为 `ceil(原价/20)` 点最大生命，货币即最大 HP。

### 2.4 重随（Reroll）价格公式

重随价格规则：

```
delta   = int(max(1, 0.4 * wave * sqrt(1 + endless_factor(wave))))
原价    = int(wave * 0.75) + delta + delta * 已付费重随次数
实付    = ceil(原价 * max(0.1, 1 + reroll_price/100))    # reroll_price 为折扣效果(望远镜等)，下限 10%
返回 [实付, 原价-实付]
```

- 普通局（无尽前）即：首次重随 ≈ `wave*0.75 + 0.4*wave`，**每次付费重随递增 `0.4*wave`**（取整、至少 +1）。
- 只统计付费重随次数，**免费重随不增加后续价格**。
- 免费重随来源：
  - `free_rerolls` 效果（危险兔子等），每个商店开始时重置为效果值；
  - **买空商店奖励**：商店 4 格全部买完（或 ban 完）时，下一次重随免费。
- 重随触发 `gain_stats_on_reroll`（骨骰等概率加/减属性）。
- 购买降低重随价的道具后，当前重随价会即时重算。

### 2.5 锁定（Lock）机制

- 每个商店栏位有锁定按钮；锁定即把 `[物品数据, 刷出波数]` 存入该玩家的锁定列表。
- 下一个商店开启时，锁定物**优先填入栏位**，剩余 `4 - 锁定数` 格再随机；锁满 4 格则无法重随。
- **锁定冻结价格**：商店价格按 `wave_value`（刷出时波数）计算，锁定的物品跨波后仍按旧波数计价，比新刷的便宜。
- 重随**不会刷新锁定格**，只重随未锁定格。
- 设置项 `keep_lock` 决定锁定是否跨商店保留；关闭则进店后清空锁定列表（仅本店有效）。
- 锁定的道具计入持有数，参与 `max_nb` 上限与宝箱开箱的去重。
- `disable_item_locking` 效果禁用锁定；个别物品 `is_lockable = false` 不可锁定。
- DLC 诅咒机制：带 `curse_locked_items` 效果时，离开商店时每个锁定物有概率被诅咒，未命中会累计保底（pity）`+= 未诅咒数 * (效果值/4)`。

### 2.6 同一商店去重规则

- 一次填充中，已抽出的物品与"刷新前的旧商品/进店时的锁定品"一起进入排除列表，从池中剔除，因此：
  - **同一商店 4 格不会出现两个完全相同的物品**；
  - **重随后不会立刻刷回上一轮的同名商品**。
- 注意 tier 不同的同名武器 id 不同，可同屏出现。
- 限购道具（`max_nb`）：持有数 + 本商店已出现数 + 锁定数 ≥ 上限时不再刷出。
- 去重在池为空时失效（回退备用池/全池）。

### 2.7 购买、偷窃、Ban 与回收

- **购买**：扣货币 → 道具直接入包（`duplicate_item` 效果可复制购买）；武器在无空位且可升级时自动与同名同级武器**合成升级**。
- **偷窃**（小偷手套 `item_steals` 效果）：免费拿走商品，但有"被抓"概率召来下一波精英：`min(0.95, base + wave*0.1 * (价格/300) * ((tier+1)*0.25))`。
- **Ban**（完成对应挑战后解锁）：消耗 ban 代币把道具从本局奖池永久移除。
- **回收/出售**（武器在商店界面；升级界面道具同理），回收价规则：

```
回收价 = max(1, 商店现价 * clamp(0.25 + recycling_gains/100, 0.01, 1.0))
       且不超过商店现价；基础价为 1 的物品固定回收 1
```

## 3. 升级（Level-up）机制

### 3.1 经验与升级

- 升 `level+1` 级所需经验：`(3 + (level+1))^2`，可被 `next_level_xp_needed` 效果加成。
- 经验获取乘 `(1 + xp_gain/100)`。
- 每升 1 级固定 **+1 最大生命**，并触发 `stats_on_level_up` 效果。
- 升级不打断战斗，升级选项排队到**波次结束后**处理。

### 3.2 升级选项

- 每次升级展示 **4 个选项**。
- 4 个选项**互相去重**，且与同一次结算里已展示过的选项去重，最多重抽 50 次。
- 升级界面也可**付费重随**，价格公式与商店重随完全相同（独立计数）。
- 若有 `weapon_slot_upgrades` 效果且武器栏未到上限，则强制只给"武器栏 +1"选项。

### 3.3 升级选项 Tier 概率

升级选项品质规则：

- 普通等级：调用与商店相同的品质抽取公式——**用等级数代替波数**代入 §2.2 公式（同样吃幸运）。
- 固定特殊等级覆盖：
  - **5 级 → 必为 tier2**；
  - **10 / 15 / 20 级 → 必为 tier3**；
  - **其余 5 的倍数（25、30…）→ 必为 tier4**。
- `level_upgrades_modifications` 效果按百分比缩放升级数值。

### 3.4 各 Tier 升级数值表

各属性升级项按品质的数值规格（效果的 `key`/`value`）：

| 属性 | T1 | T2 | T3 | T4 |

## 4. 消耗品（Consumables）

### 4.1 种类与数据

消耗品数据在道具数据基础上扩展，关键字段 `to_be_processed_at_end_of_wave`（是否在波末结算）：

| 消耗品 | tier | 效果 | 波末处理 |
|---|---|---|---|
| 水果 `consumable_fruit` | 0 | 拾取立即治疗（value=3） | 否 |
| 宝箱 `consumable_item_box` | 1 | 拾取治疗 3（复用水果效果）+ 波末开箱 | 是 |
| 传说宝箱 `consumable_legendary_item_box` | 3 | 拾取治疗 **100** + 波末开传说箱 | 是 |

### 4.2 水果治疗量公式

消耗品治疗效果结算：

```
治疗量 = max(0, 3 + consumable_heal)      # consumable_heal 为道具效果(肉干等)
```

若有 `consumable_heal_over_time` 效果则改为持续治疗。满血拾取时可触发 `consumable_stats_while_max`（额外胃等：满血吃水果得属性，带每波上限）与临时属性效果。

### 4.3 掉落规则

两层判定：

1. **掉不掉**：
   - `掉落率 = min(1, 单位.base_drop_chance * (1 + 全体玩家幸运之和/100))`；
   - 无尽模式再除以 `(1 + endless_factor)`；
   - tier 在单位的最低/最高消耗品品质之间随机（普通敌人为水果）。
2. **是否升格成宝箱**：
   - `宝箱率 = 单位.item_drop_chance * (1 + 全体幸运/100) / (1 + 本波已掉宝箱数)`，再乘 `(1 + crate_chance效果/100)`；
   - 命中则掉宝箱；**正常波次的 Boss 命中时掉传说宝箱**；
   - 注意：当第 1 层"掉不掉"判定**失败**时，还有 `enemy_fruit_drops/100` 概率补掉一个水果（水果篮），与宝箱判定无关。
- 树等中立单位 `always_drop_consumables` 必掉。
- 部分建筑（如园艺）会周期性在身边 100~150 像素随机点生成水果。
- 掉落位置：单位位置向外推 `随机(50, 100+gold_spread)` 的随机点。

### 4.4 宝箱开箱（波末结算）

- 拾取宝箱只入队列（联机可按 `share_coop_loot` 均分），波次结束在升级界面逐个打开。
- 开箱内容：按当前**波数**用 §2.2 的 tier 公式抽一个**道具**（不会开出武器）；传说宝箱固定为传说品质；已持有/已锁定的道具参与 `max_nb` 去重。
- 玩家可选择 **拿取** 或 **回收成材料**（回收价 = §2.7 公式），也可消耗 ban 代币直接 ban 掉。
- 道具"袋子 Bag"的 `item_box_gold` 效果：每拾取一个宝箱额外 +N 材料。

## 5. 道具效果数据结构

### 5.1 物品数据层级

- **物品基础数据**：`my_id`、`name`、`tier`、`value`（基础价）、`effects[]`、`can_be_looted`（能否商店刷出）、`is_lockable`、`is_cursed/curse_factor`（DLC 诅咒）、`unlocked_by_default`、`tracking_text`（物品面板统计文本）。
- **道具数据**（扩展物品基础数据）：`max_nb`（限购数，-1 不限）、`tags`（角色偏好/移除过滤用）、`item_appearances`（角色外观贴片）、`replaced_by`。
- **武器元数据**：另带 `weapon_id`（同名武器跨 tier 共用）、`upgrades_into`（合成链）、`sets`（武器系别）。
- **升级数据**（扩展道具数据）：加 `upgrade_id`，名称自动拼 tier 罗马数字（II/III/IV）。
- **消耗品数据**（扩展道具数据）：加 `to_be_processed_at_end_of_wave`、`pickup_sounds`。

### 5.2 属性效果条目（基础结构）

- 字段：`key`（属性/效果键）、`value`、`custom_key`、`storage_method`（存储方式）、`effect_sign`（文本着色：正/负/中性/随值）、`custom_args`（文本参数格式化）。
- 生效/失效时直接对玩家的效果字典做加减/插入，五种存储方式：
  - `SUM`：`effects[key] += value`（普通属性加成）；
  - `KEY_VALUE`：往 `effects[custom_key]` 列表插 `[key, value]` 并可叠加（如"每波获得X属性"类）；
  - `REPLACE`：覆盖并记住旧值（上限类）；
  - `APPEND_KEY` / `APPEND_KEY_VALUE`：追加键/键值对（不合并）。

### 5.3 特殊效果类型（节选）

- **结构效果**：生成的建筑表现资源、`spawn_cooldown`、`stats`、生成位置（场地中央/玩家周围）、`can_be_grouped`、`is_pet`；生效时把自身加入 `structures` 效果列表。
- **宠物效果**。
- 道具向：炮塔效果、受击发射弹效果、全局燃烧概率效果、道具爆炸效果（含低血量触发变体）、属性转换效果、属性联动效果（每X点A给B）、属性上限效果、属性获取增减%效果、最大最小属性互换效果（美西螈）、周期临时属性效果、武器职业加成效果、价值翻倍效果、武器冷却下限效果等。
- 武器向：燃烧效果、爆炸效果、命中附属弹道效果、击杀成长效果等。
- 消耗品向：消耗品治疗效果、消耗品伤害效果。
- **效果行为绑定**：部分效果绑定一个运行时行为对象，承载需要逐帧驱动的逻辑。

## 6. 经济相关公式汇总

| 机制 | 公式 |
|---|---|
| 物品价格 | `max(1, (V + w + 0.1*V*w) * (1+价格效果%) * (1+endless/5))` |
| 重随价 | `int(0.75w) + delta*(1+已付费次数)`，`delta=int(max(1, 0.4w*sqrt(1+endless)))` |
| 回收价 | `min(现价, max(1, 现价*(clamp(0.25+回收加成%, 0.01, 1))))` |
| 偷窃被抓率 | `min(0.95, base + 0.1w * 价格/300 * (tier+1)*0.25)` |
| 升级所需经验 | `(3+level)^2 * (1+next_level_xp_needed%)` |
| 经验获取 | `value * (1 + xp_gain%)` |
| 材料拾取 | 1 价值 = 1 金钱 + 1 经验（联机轮转均分） |
| 敌人掉落率 | `wave<5: 100%；否则 max(0.5, 1-0.015w)`；蜂群×0.65 |
| 收获结算 | 波末 `+harvesting`（金钱+经验），每波成长 `+5%`(默认)，无尽每波 `-20%` |
| 波始利息 | `当前金钱 × gain_pct_gold_start_wave%`（无尽正值失效） |
| 无尽系数 | `(max(0,w-20)*(w-19)/2)/100 * (2+max(0,w-35)*0.2)` |

注：`w` = 波数；"endless" = 无尽系数（随波数计算）。

## 7. 物品 Tier 颜色 / 稀有度体系

品质档位定义：`COMMON=0, UNCOMMON=1, RARE=2, LEGENDARY=3`，另有 `DANGER_4/5/0` 仅用于危险度（难度）显示。

默认颜色规格，可在设置中自定义：

| Tier | 显示名 | 默认颜色 (RGB) |
|---|---|---|
| 1 COMMON | TIER_COMMON / 无角标 | 白/灰（边框用黑色回退） |
| 2 UNCOMMON / "II" | TIER_UNCOMMON | 蓝 (90,190,255) |
| 3 RARE / "III" | TIER_RARE | 紫 (173,90,255) |
| 4 LEGENDARY / "IV" | TIER_LEGENDARY | 红 (255,59,59) |
| 危险度4 | — | 橙 (255,119,59) |
| 危险度5 | — | 黄 (208,193,66) |

- 商店卡片、背包格子、升级卡的边框/底色统一按品质颜色染色；锁定时边框改为白色。
- 同名武器各 tier 是独立数据资源（`weapon_id` 相同、`my_id` 不同），靠 `upgrades_into` 链升级；商店里两把同名同级武器可合成高一级（边买边合成）。
