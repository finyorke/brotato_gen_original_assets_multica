# 07 波次、难度与生成系统

> 本篇规定：危险等级（难度）系统、波次结构与计时、敌人组成（波次数据）、精英/Boss/蜂群波、生成位置与上限、地图机制、波次结束流程、无尽模式数值放大、主循环胜负判定。

---

## 1. 难度系统（危险等级 Danger 0–5）

### 1.1 数据结构

每个危险等级是一份难度数据，`value` 字段即危险等级数值，开局选定后写入对局数据的"当前难度"。

难度 6–10 是**纯展示用占位**（效果列表为空，仅有图标），用于无尽模式记录界面显示 6–10 级图标，**没有任何游戏效果**。

### 1.2 每级精确修正

选择难度后，其效果列表逐一施加（存档恢复时同样重新施加）。各级效果如下：

| 危险等级 | 效果配置 | 实际数值修正 |
|---|---|---|
| 0 | 空效果 | 无任何修正 |
| 1 | "新敌人"（文案提示效果） | 解锁最低难度 1 的敌人生成组（见 §3） |
| 2 | "新敌人" + "精英出现"（均为文案提示） | 新敌人组（含最低难度 2）+ 出现精英/蜂群波（见 §4） |
| 3 | 上述 + `danger_enemy_health = 12` + `danger_enemy_damage = 12` | 敌人 HP +12%、敌人伤害 +12% |
| 4 | 上述 + "更多精英"，HP/伤害效果值为 26 | 敌人 HP +26%、伤害 +26%；精英事件 1 次 → 3 次（见 §4） |
| 5 | 上述 + `double_boss = 1`，HP/伤害效果值为 40 | 敌人 HP +40%、伤害 +40%；3 次精英事件；最终波 **2 个不同 Boss** |

> 注："新敌人"/"精英出现"/"更多精英"本身是无数值的文案提示效果，真正的机制由波次数据中的"最低难度"字段（§3.2）和精英排期规则的难度分支（§4.1）实现。

### 1.3 敌人 HP / 伤害 / 速度最终倍率公式

（结果按波缓存）：

```
敌人伤害 = round(基础值 × danger × accessibility × coop × effects × endless × boost)
  danger  = max(0.01, 1 + danger_enemy_damage/100)            # 难度 3/4/5 = 1.12/1.26/1.40
  coop    = 1 + (玩家数-1) × 0.08                              # 联机修正
  effects = 1 + Σ玩家 enemy_damage 词条/100
  endless = 1 + 无尽系数 × (1 + sqrt(平均诅咒)/25)

敌人HP   = round(基础值 × danger × accessibility × coop × effects × endless × boost)
  coop    = 1 + (玩家数-1) × 0.30
  endless = 1 + 无尽系数 × 2.25 × (1 + sqrt(平均诅咒)/10)

敌人速度 = round(基础值 × effects × accessibility × (1 + min(1.75, 无尽系数/13.33)))
```

`accessibility` 为玩家在选项中的"敌方强度"无障碍设置。`无尽系数` 见 §8.1，前 20 波为 0。

### 1.4 解锁条件

- 危险 0 默认解锁，1–5 默认锁定。
- 最高危险等级为 5。
- **胜利一局（危险 D）→ 解锁危险 D+1**：把当前角色当前区域的"可选最高难度"升至 `D+1`；并且把**所有角色所有区域**的"可选最高难度"同步抬到 `clamp(D+1, 原值, 5)` —— 即难度解锁是全局共享的，一次跨级胜利可同时解锁多级。
- 每角色每区域同时记录"已通关最高难度"（含无障碍设置、重试次数、是否联机等信息）。

## 2. 波次结构与计时

### 2.1 总波数与时长

- 一局固定 20 波。
- 每波时长由该波波次数据的"波时长"字段决定，区域 1 实际数据：

```
波 1–8:  20,25,30,35,40,45,50,55 秒（即 15 + 5×波数）
波 9–19: 60 秒
波 20:   90 秒（Boss 波）
```

- 每波敌人上限 `max_enemies = 100`（全部 20 波相同）。
- 倒计时驱动：剩余 < 6 秒开始播放滴答音效并使计时 UI 变红；同时存在一个半场计时器（在波时长一半时触发一次，用于 `convert_stats_half_wave` 类效果）。
- 计时结束 → 进入波次结算（§7）。

### 2.2 波内生成节奏（波次管理器）

每过去 1 整秒检查一次所有生成组：

- 触发条件：`当前难度 ≥ 组的最低难度` 且 `已过去时间 ≥ 组的首刷时间(spawn_timing)`；**Boss 组无视首刷时间，开波即出**；持有 `trees_start_wave` 效果时中立组（树）也在开波前 5 秒内立即生成。
- 触发时按组的生成概率掷骰决定是否真的发出生成请求。
- **重复机制**：`repeating` 为重复次数（999≈无限，-1/0 = 只出一次），首次触发后每隔"重复间隔"秒重复一次；每次重复后间隔减少"间隔递减量"，下限为"最小重复间隔"。即大多数组的生成频率随波内时间推进而加快。

### 2.3 单次生成的数量公式

```
number = randi(单位条目.min_number, 单位条目.max_number)
若为敌人(非宝藏组):
  number += number × 0.3 × (玩家数-1)            # 联机修正
  number = max(1, number × (1 + Σnumber_of_enemies效果/100))
若为中立(树): number += Σtrees 效果
最终数量 = floor(number × 单位生成率)，小数部分按概率补 1
```

### 2.4 出生预警与生成队列

- 敌人不会瞬间出现：先放置"出生预警"标记，闪烁约 1 秒（60 帧）后实体化；若玩家站在敌人出生标记上，标记会**传送到随机新位置并重新计时**。
- 生成请求进入队列，每 3 个物理帧（即 1/20 秒）处理一次：普通敌人队列每次取 1 个；若积压超过 100，每次取 `clamp((积压-100)/10, 1, 2)` 个；树/Boss/召唤/建筑/宠物队列各自每次取 1 个。

### 2.5 同屏敌人上限与超限处理

每次生成组触发时检查：

```
上限 = max_enemies + (玩家数-1) × max_enemies/8     # 单人 100，4 人 137
```

超限时随机挑选现存敌人（优先"优先被清除"名单中的低价值敌人，如召唤物）直接死亡，**不掉落任何材料**，计入"性能清除数"（该计数会算进"和平主义者"收割加成）。建筑上限 100。

## 4. 精英波、蜂群（Horde）波与 Boss 波

### 4.1 精英事件排期

开局（及选完难度后）执行精英排期，默认参数 `基准波=10, 蜂群概率=0.4`：

- **危险 < 2：没有精英/蜂群事件**；危险 2–3：1 次；危险 4–5：3 次。
- 排期波数：第 1 次 = `randi(11,12)`；第 2 次 = `randi(14,15)`；第 3 次 = `randi(17,18)`，**第 3 次强制为精英**（非蜂群）。
- 每次事件 40% 概率为 **HORDE（蜂群）**，否则为 **ELITE（精英）**。
- 角色特例：Jack / Gangster → 蜂群概率 0%；单人 Ogre → 100% 蜂群（第 3 次仍为精英）。
- 精英个体从本区域 8 种精英中**不重复**随机：`elite_butcher`、`elite_colossus`、`elite_croco`、`elite_insect`、`elite_monk`、`elite_rhino`、`elite_demon`、`elite_mom`。
- 排期结果 `[波数, 类型, 精英id]` 存入精英排期表，HUD 时间轴与商店"下一精英波"面板据此显示。

### 4.2 精英/蜂群波的执行

当前波命中排期时：

- **ELITE**：复制精英组模板（带 is_boss 标记，边缘生成，开波即出）并填入该精英。
- **HORDE**：把区域的蜂群组（满足 `最低波 ≤ 当前波 ≤ 最高波`）全部加入本波：

| 蜂群组 | 内容 | 节奏 | 波数限制 |
|---|---|---|---|
| 蜂群组 1 | baby_alien×4 + chaser×4 | t1，每5s→2s，边缘 | 无 |
| 蜂群组 2 | baby_alien×4 | t8，每4s→2s，聚团200 | 无 |
| 蜂群组 3 | chaser×4 | t15，每3s→1s，聚团200 | 无 |
| 蜂群组 3b | baby_alien×4 + chaser×4 | t30，每3s→1s，聚团300 | ≥10 波 |
| 蜂群组 4 | helmet_alien×8-9 | t25，每2s→1s，聚团400 | ≥14 波 |
| 蜂群组 5 | fin_alien×6-7 + baby_alien×4 + chaser×4 | t30，每2s→1s，聚团400 | ≥15 波 |

- **蜂群波减益**：敌人材料掉落概率 ×0.65；"和平主义者"收割加成减半。

### 4.3 Boss 波（第 20 波）

- 开局执行 Boss 排期：从本区域 Boss 池（`boss_crab` 捕食者、`boss_wizard` 召唤师）随机抽取；**危险 5（double_boss 效果）抽 2 个不同 Boss**，否则 1 个。
- 第 20 波数据含一个 is_boss 空组（边缘、开波即出），在波初始化时填入 Boss 单位；同时该波还有固定的小怪滚动生成表（见 §3.2 波 20 行）。
- **Boss HP 修正**（对精英同样生效，因为精英共用 Boss 框架）：波 ≤ 8 时 ×0.5；（危险 5 双 Boss 最终波）或波 ≤ 12 时 ×0.75；最终波且危险 ≤ 0 且非教学区域时 ×0.9。
- **提前击杀全部 Boss**：非无尽时把剩余波时间改为 0.1 秒，立即结束；无尽模式则标记"最终波 Boss 已清空"并追加 `int(wave/10×3)` 个随机补怪组填满剩余时间。
- 精英/Boss 死亡分别记入本局精英/Boss 击杀数；`stronger_elites_on_kill` 效果会按已击杀数提高后续精英 HP（+值）与伤害（+值/2）。

## 5. 生成位置规则

### 5.1 坐标体系与常量

- 地格尺寸 = 64 像素；边缘生成距离 = 64 像素。
- 地图矩形 = `(0,0)` 到 `(宽格数×64, 高格数×64)`。
- 随机取点规则 `get_rand_pos(edge)`：在四边各内缩 `edge` 像素的矩形内取点；`edge` 不得超过较短边的 20%。

### 5.2 组位置与单位位置

1. **组中心**：不限定区域 → 全图随机；区域=N → 随机点（内缩 N/2+离边距离）。组中心与最近玩家距离需 ≥ `300 + 区域/2`（上限为地图长边一半）；不满足则重选，每次要求 -5px，降到 25px 后放弃（保证不会死循环）。
2. **每只单位**：
   - 边缘生成 → 随机选一条边，在距该边 64px 的线上随机取点；
   - 不限定区域 → 全图随机（按"离边距离"内缩）；
   - 否则 → 组中心周围 `区域×区域` 方形内随机。
   - 单位与最近玩家距离需 ≥ `300 + 单位附加最小距离`（初始最小距离 300），同样按 -5px/次衰减到 25px。
3. 出生标记被玩家踩住 → 随机传送重计时（§2.4）。

## 6. 地图机制

- 区域 1 基础尺寸 **32×24 格 = 2048×1536 像素**。
- **地图大小修改**：`尺寸 = max(12, 基础格数 × (1 + Σmap_size效果/100))`，最小 12 格。该效果来自角色/道具（如 Demon 等带 map_size 词条者），单人/联机所有玩家效果求和。
- 摄像机活动边界 = 地图矩形向外扩 96 像素（顶部 ×2）。
- 背景：每区域 6 套随机背景数据（含地砖贴图、描边色、渐变色），开局随机决定；区域还定义专属水果/宝箱贴图。

## 7. 波次结束流程

波计时结束 → 清场 → 结算计时结束，依次执行：

1. **波末效果结算**：`stats_end_of_wave`（如义警戒指）、`convert_stats_end_of_wave`、挑战检查。
2. **收割（Harvesting）发放**：`val = 收割属性 + 和平主义者加成 + 树加成(cryptid) + 存活敌人加成 + 魅惑敌人价值`，作为材料+经验直接发放；随后收割属性按 `harvesting_growth`% 增长（无尽模式改为**每波衰减 20%**）。
3. **判定运行状态**（§9.2），停表、清场：所有存活敌人/Boss/树/建筑/宠物死亡（不掉落）。
4. **未拾取材料回收**：地上所有材料飞向"材料袋"；以**全额价值**进入 `bonus_gold` 池（开启"波末优化"设置时直接求和入池）。**没有折损比例**，但它不直接变成钱：下一波生成材料时每颗材料 `value += min(value, bonus_gold)`（即翻倍）并扣减池子，直至耗尽。Builder 角色的 `convert_bonus_gold` 效果可把池子按比例转化为建筑属性。
5. 地上未拾取的**消耗品（水果/宝箱）自动飞向存活玩家**被吸收。
6. 精英排期补充：`波数 % 10 == 0 且 ≥ 10` 时为下一个十波段生成精英排期（基准波+10，蜂群概率 0；供无尽模式使用）。
7. 结算等待（失败 2.5s / 胜利 4s / 重试 1.5s）后：依次弹出**升级选择**（每级一次四选一）与**宝箱开箱**（拿取 / 回收。回收价值 = `min(商店价, max(1, 商店价 × clamp(0.25 + recycling_gains/100, 0.01, 1)))`）→ 切换到**商店界面**。
8. **治疗**：没有波末治疗；下一波开始时 HP 直接设为 `max_hp × hp_start_wave%`，默认 100（满血开波）；`hp_start_next_wave` 效果可覆盖一次。
9. **波失败重试**：若所有玩家死亡且开启无障碍选项"重试本波"且波数 > 1，显示重试界面而非结束（重试次数计入记录）。

## 8. 无尽模式

### 8.1 进入与波数据生成

- 第 19 波结束后的商店出现"无尽模式"按钮（仅当前波 = 19 时显示）；点击后本局标记为无尽局。
- 第 21 波起波次数据走无尽生成分支：

```
wave_index = (index-1) % 20；若 < 10 则 +10        # 循环使用第 11~20 波的数据
max_enemies ×= 1.25 + wave_index × 0.01            # wave_index ∈ 10–19 → 同屏上限 ~135-144
波时长 = 60 秒
追加 nb = int(index/10 × 2) 个随机无尽组（铺满 60 秒，附加精英数 = nb/5）
```

- 随机无尽组：以"无尽敌人组模板"生成，生成时间均匀铺满 60 秒、50% 边缘/50% 聚团(200-300)、每 6-8 秒重复、敌人从区域无尽池随机：tentacle、lamprey、slasher_egg、fly、horned_bruiser、horned_charger、pursuer、buffer、spawner。
- **每逢 ×10 波（30、40…）映射到第 20 波数据 → Boss 波**：波 > 20 时按"无尽 Boss 规则"生成（恒 2 个 Boss）+ 一个精英组；且所有精英组额外追加 `ceil((波-20)/10)` 个随机精英。
- 普通精英排期也在每个十波段补一次（§7 第 6 条），蜂群概率 0。

### 8.2 数值放大

```
endless_wave  = max(0, 波数 - 20)
endless_mult  = 2.0 + max(0, (波数 - 35) × 0.2)
无尽系数 = (endless_wave × (endless_wave+1) / 2) / 100 × endless_mult
```

代入 §1.3：HP 倍率额外 ×`(1 + 无尽系数 × 2.25 × (1+诅咒项))`，伤害 ×`(1 + 无尽系数 × (1+诅咒项))`，速度 ×`(1 + min(1.75, 无尽系数/13.33))`。其他规则：收割每波 -20%；`gain_pct_gold` 负值效果在 20 波后强制 -100%；每波 ≥ 20 结束都会更新"无尽最高波"记录。

## 9. 主循环：暂停与胜负判定

### 9.1 暂停

- 每帧轮询暂停输入（联机为各玩家设备各自的暂停键；流式合家欢模式仅 P1），触发暂停菜单并记录发起玩家。
- 窗口失焦自动暂停（重试界面显示时除外）。
- 调试模式下按 1/2/3 可调全局时间倍率 0.5/1/2。

### 9.2 胜负判定

| 条件 | 结果 |
|---|---|
| 波 < 20 且全员死亡 | 失败（可按设置重试本波） |
| 波 = 20，非无尽：存活到结束（或提前杀完 Boss） | **胜利**（解锁下一危险等级，§1.4） |
| 波 = 20，非无尽：全员死亡 | 失败 |
| 波 = 20，无尽：全员死亡且已杀完 Boss | 胜利 |
| 波 = 20，无尽：全员死亡未杀完 Boss | 失败 |
| 波 > 20（无尽）：全员死亡 | **判定为胜利**（结束无尽，保留记录） |

胜利时执行"对局获胜"结算：发角色胜利挑战/成就、更新角色难度记录、全角色解锁危险 D+1、存档。失败时重置并保存新的对局状态。死亡瞬间还会记录击杀来源敌人统计。胜利/失败画面结束后切到对局结算界面。
