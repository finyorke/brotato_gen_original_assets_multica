## 12.0 工程级设定

| 项 | 值 |
|---|---|
| 启动场景 | 引导 Splash 状态机（见 12.10.5） |
| 分辨率 | 1920×1080，2D 等比拉伸，保持宽高比，支持高 DPI |
| 物理 | 2D 重力为 0（俯视角），线性/角阻尼 0 |
| 自定义鼠标指针 | 自定义光标贴图，热点 (3,3) |
| 逻辑帧率 | 60 物理帧/秒（每秒逻辑刻数 = 60） |

游戏由一组常驻全局服务驱动（进度存档、物品、区域、对局状态、音频、文案、挑战、输入、武器数值、实体、宠物、合作、UI 等），按依赖顺序在启动时初始化。

音频总线布局：3 条总线——`Master`、`Sound`（→Master）、`Music`（→Master），默认 0 dB。设置菜单的三个音量滑条以线性值存入 `settings.volume.{master,sound,music}`，应用时转换为 dB 写入对应总线。
设置 `mute_on_focus_lost` 开启时，窗口失焦把 Master 调到静音，重获焦点时恢复。

## 12.1 音乐系统

**结构**：全局单例，单个音乐播放器（走 `Music` 总线）+ 单个渐变缓动，**暂停时继续播放**。

**没有"菜单曲/战斗曲/Boss 曲"之分**——整个游戏共用一个随机播放列表（shuffle playlist），状态切换只通过**音量渐变**实现：

### 12.1.1 曲目池

| 池 | 开关设置 | 曲目 |
|---|---|---|
| 旧音轨（legacy，6 首） | `settings.legacy_tracks`（默认 false） | after-midnight、density-wave、extreme-chaos、power-punch、time-jumper、wasteland-survivors（授权曲） |
| 新音轨（streamer mode，5 首） | `settings.streamer_mode_tracks`（默认 true） | Adonai - Hyperspace / Liquid Glass / Phase Break / Turbine Kick / Wubs of the Deep |
| DLC 曲目 | 每个已激活 DLC 自带的音轨列表 | 由 DLC 内容注册提供 |

歌单重建时把启用的池合并后整体洗牌。曲目默认不循环（播完换下一首）。

### 12.1.2 播放规则

- `play(目标音量)`：列表空则重建（无可用池则停止）；取下一首；若与当前曲相同则再取一首（避免连播同曲）；起播音量固定 `-20 dB`，再在 **1 秒**内线性渐变到目标音量（默认 0 dB）。
- 曲目播完 → 自动播下一首。
- 渐变进行中再次调用会等待上一个渐变完成。

### 12.1.3 音量状态机（调用点）

| 时机 | 调用 | 含义 |
|---|---|---|
| 标题画面进入 | `play(0)` | 起播，渐变到 0 dB |
| 主菜单进入运行准备 | 渐变至 -5 dB | 压低 |
| 波次开始 | 渐变至 0 dB | 恢复全音量 |
| 波次失败（全员死亡） | 渐变至 -20 dB | 大幅压低 |
| 波次结束进入升级/消耗品结算 | 渐变至 -8 dB | 中度压低（商店期间维持 -8） |
| 结算/胜负画面 | `play(0)` 换曲并恢复，离开时渐变至 -5 dB | |
| 重开运行 | `play(0)` | |
| 设置中切换曲目池/DLC 音乐 | 重建歌单 + `play()` | 立即换曲 |

> 设计要点：商店、Boss 波、菜单**不更换曲目**，仅靠 -5/-8/-20/0 dB 四档渐变区分场景。

## 12.2 音效系统

### 12.2.1 播放器池与限流

音效系统分两套：全局无定位版与 2D 定位版，共享同一套基础逻辑：

- **固定 12 个播放器**，走 `Sound` 总线，**暂停时继续处理**。
- 待播队列上限 **16**；播放请求到来时若"无空闲播放器或队列已满"则**丢弃**该音效；除非 `always_play=true`——此时挤掉队头强行入队。这就是同帧限流：每帧只出队 1 个音效播放。
- 出队时设置：音量修正 dB、`音调 = 1.0 + rand(-pitch_rand, +pitch_rand)`（**随机变调**）；2D 版本额外设置世界坐标。
- 波次结束时清空两套队列的残留。

### 12.2.2 同名音效计数限流

按字符串 id 统计"当前正在播放的实例数"，超过上限直接忽略；播放后等待音频时长再减计数。实际使用者均为宠物叫声：医蛾（上限 2）、Bonk 狗攻击（上限 1）、猫特林机枪狂暴（上限 2）。

## 12.3 视觉反馈完整清单

### 12.3.1 受击闪白（闪白着色器）

- 着色器逻辑：`color.rgb = mix(原色, flash_color.rgb, flash_modifier)`，参数 `flash_color`（默认白）与 `flash_modifier`（0~1，默认 1.0）。
- 单位默认配置：闪白色 (1,1,1,1)、强度 1.0 → **纯白剪影**。
- 流程：受击实际伤害 >0 → 闪白：把精灵材质换成闪白材质并启动闪白计时器（**0.1 s**）；超时还原原材质。玩家受击（实际伤害>0）同样闪白。已在闪白时再次受击只重置计时器。

### 12.3.2 轮廓与变色着色器

- 描边着色器：邻域采样描边，参数 `width`（默认 3 px）、`alpha`、`desaturation`（灰度化）、最多 **4 层**轮廓色。实体通过"加描边/去描边"接口管理颜色栈并动态新建材质。用途：增益单位红色描边 `#a30000`、魅惑 `#9bc6dd`、角色/武器高亮 `#72fff2`（设置 `character_highlighting`/`weapon_highlighting`）；另有投射物与武器的预制描边材质。
- 色相偏移着色器：整体色相旋转（参数 hue 0~1）+ alpha + 去饱和，用于变体配色（如诅咒敌人/特殊外观）。
- 噪声扰动着色器：用于果冻护盾、鼠斯拉等宠物的果冻/波动效果（参数含波动速度等）。

### 12.3.3 粒子总管

局内统一的粒子调度器。全局上限 **100 个并存视效**，超限直接不播；实例走全局对象池。设置 `visual_effects=false` 时跳过受击粒子与拾取粒子。CPU 粒子实例的数量乘以 `effect_scale`（最小 1）。

| 视效 | 触发 | 关键参数 |
|---|---|---|
| 受击血浆粒子 | 单位受击且伤害>0 且单位会出血（机械系敌人无血）；位置 = 单位位置 + 击退方向 × `贴图宽×0.25` | 暗红 `(0.62,0.15,0.15)`，lifetime 0.5，单发，球形发射半径 15，spread 180°，初速 600±50%，scale 0.5±25% 衰减曲线 |
| 受击斩击特效 | 同上（无论是否有血都播）；**播放概率 = effect_scale**（燃烧 DoT 的基础 effect_scale=0.1 → 10% 概率小特效） | 帧动画，3 帧 @20 fps，随机旋转 |
| 暴击粒子 | 单位暴击受击事件 | 金黄色加法混合，amount 1，lifetime 0.5±49%，初速 177±31%，负加速曲线 |
| 一击必杀粒子 | 一击必杀效果触发（斩首之剑型） | 专属粒子 |
| 治疗粒子 | 敌人被治疗事件 | amount 1，lifetime 0.5，半径 45，初速 300±50%，scale 0.75 |
| 增益粒子 | 敌人被 buff 事件 | amount 3，其余同上 |
| 减速解除粒子 | 减速生效事件 | amount 3，lifetime 0.25，半径 25 |
| 拾取材料粒子 | 材料被拾取（清场阶段抑制） | amount 6，lifetime 0.3，初速 600±50%，scale 0.4 |

粒子完成回收：池化粒子用完成计时器在重播后计时发"完成"事件 → 归还对象池。

### 12.3.4 燃烧火焰

每个单位自带燃烧粒子组件（火把武器有独立变体）：
- 点燃时开始发射，按燃烧来源的第一个缩放属性选色：元素伤害 → 红色渐变，工程学 → 蓝色渐变。
- 主粒子 + 次级粒子双层；熄灭后再拖尾 `lifetime × 1.2` 秒才隐藏。
- 自带**传播区域**：燃烧带 spread 值时启用碰撞，把火传给接触的未燃烧单位（每次传播 spread-1）。

### 12.3.5 屏幕震动

设置 `screenshake`（默认 true）关闭后全部失效。

| 触发 | 强度 intensity | 时长 |
|---|---|---|
| 玩家受到 >0 伤害 | **5** | **0.1 s** |
| 任意敌人/中立单位受击 | **min(伤害/3, 3)** | 0.1 s |

新震动只有同时强于当前强度且更久才覆盖（不叠加）。每帧实现：`相机偏移 = (randf(), randf()) × intensity`（只向右下随机偏移），时长递减归零后偏移复位。

### 12.3.6 Hit-stop / 时间缩放

**没有 hit-stop 机制**，正式玩法不修改全局时间缩放。死亡敌人通过正常速率的"death"死亡动画表现，死亡击退最小 15。

### 12.3.7 浮动文字

- 单条浮动文字（Label）：两段缓动——先按 `direction.rotated(±spread/2)` 弹性缓出位移 `duration` 秒，再 `duration` 秒弹性缩放到 0 + 线性淡出，然后回池。
- 默认参数：`direction = (0,-80)`，`duration = 0.5`（带图标的属性文本 ×2），`spread = π/4`，**同屏上限 100 条**（超限丢弃，除非 always_display 如玩家受伤/ONE_SHOT）。
- 伤害数字着色（可被 `settings.damage_display` 关闭）：玩家受伤=红色 `color_negative` 前缀 `-`；闪避=`DODGE`、无效化=`NULLIFIED`（翻译键）；敌人暴击=黄色；护甲减伤生效=灰色；one-shot=`ONE_SHOT`；特殊命中（暴击得金/诅咒击杀得金）附图标。
- 其他浮动文字：升级 `LEVEL_UP`、敌人变异 `MUTATION`、治疗 `+N`（绿色）、收获结算带镰刀图标（多条沿 (0,30) 偏移向上堆叠）、拾取翻倍 `xN`、商店内属性变化。属性图标文本统一金色 `#76ff76`。

### 12.3.8 材料/物品掉落与拾取动画

- 掉落：落点确定后每帧向目的地指数趋近（速度系数 5/秒，距离² >400 时持续）；到位后恢复可探测并空闲 10 帧才可被吸引。掉落目的地 = 距离 `rand(50, 100+spread)` 的随机点；材料随机旋转 0~2π。
- 吸引：被吸附者（玩家/搜刮虫虫/波末金袋）吸附，初速 **500 px/s**，每帧加速 **+20**。
- 材料外观：11 张随机贴图；被奖励金增值时 scale 1.25；超过 50 个上限时旧材料合并增值，scale 每 +1 价值增 0.05，上限 **2.0**。
- 拾取闪光粒子 + 浮动文字见 12.3.3/12.3.7。波末回收：`optimize_end_waves=true` 时不播放飞行动画直接结算。

### 12.3.9 其它视效

- **伤害暗角/低血变暗**：全屏着色矩形，`multiplier = clamp(HP/MaxHP, 0.3, 0.8)`；设置 `darken_screen=false` 时固定 0.8。仅单人模式 P1 生效。
- **实体出生标记**：落地前的旋转标记，出生前摇 60 物理帧（1 秒），闪烁周期 = 剩余时间/3.25，闪烁透明度 0.25；颜色：敌人=设置负色提亮 0.22、树=正色提亮、结构/宠物=矢车菊蓝、魅惑=魅惑色；敌人与结构各有专用贴图；玩家踩在上面会延迟出生（检测玩家层）。
- **通用闪烁器**：UI/精灵闪烁——0.05 s 间隔切换透明度，共 12 次，可选结束后隐藏（用于合作模式血条死亡闪烁等）。
- **奔跑烟尘**：玩家移动时发射，颜色取当前背景描边色；水中角色替换为游泳气泡。
- **落地动画**：第 1 波开局所有玩家播放落地动画（水中为游泳落地）。
- **胜利动画**：存活玩家延迟 0.25 s 播胜利动画，1 s 后实例化飞船光柱并 0.1 s 淡入（含专属音效），随后进入胜利结算；另有"跳舞"动画用于结算画面。
- **拾取者感叹号**：拾取者型敌人吃到材料时头顶弹出 `!`（缩放弹跳动画）。
- **诅咒粒子**：物品版（amount 3, lifetime 1.2±25%）与敌人版（amount 3, lifetime 0.6, 初速 113）——诅咒标识，紫色渐变（诅咒紫 `#ca61ff`）。
- **爆炸烟雾**：amount 40，半径 128，lifetime 0.75，配合爆炸投射物。
- **幽灵权杖粒子**：武器特效。
- **波次结算大字** + 全屏渐暗层。
- **手动瞄准准星**：单人+手柄手动瞄准时显示准星精灵，位于玩家 (y-32) 沿右摇杆方向 200 px；鼠标手动瞄准时把鼠标指针换成准星图（热点 35,35）。普通指针/手动指针切换由光标管理服务处理。
- **Y 排序**：所有实体挂在统一容器下按 y 深度排序绘制。

| 宠物攻击区（Bonk 狗等） | 1024 | 探测区掩码 4 = enemies |
| 材料 | 64 | 0 |
| 消耗品 | 32 | 0 |
| 金袋 | 0 | 256 = bonus_gold |
| 出生标记 | 0 | 2 = player（检测玩家阻挡出生） |
| 拾取者/吞吞怪吞噬区 | 0 | 96 = items+gold（敌人吃掉落物） |

敌人射击行为可用自定义碰撞层覆盖投射物层（如魅惑敌人发射玩家方子弹）；投射物回收时恢复原始碰撞层。

### 12.4.3 攻击区 / 受击区判定流程

两者都是触发区域 + 单个碰撞形状，自身层/掩码在基础模板中均为 0，由宿主按上表配置。

- **受击区**：纯被动，只有启用/禁用（禁用走延迟设置）。
- **攻击区**：携带完整攻击参数——伤害、暴击率、暴击倍率、精准、击退（方向/量/穿透）、effect_scale、燃烧参数、是否治疗、命中派生弹、忽略对象表、附加效果、减速百分比、来源（武器/单位）、伤害统计键、缩放属性、激活标志。攻击参数包支持从武器属性/爆炸参数一次性拷贝。
- **注册方式**：受击区的"区域进入"事件连到宿主单位，延迟一帧处理命中。
- **判定顺序**：① 过滤（无宿主 / 攻击区未激活 / 在忽略表 / 已待死亡）→ ② 触发双方效果行为的"受击时"回调 → ③ 爆炸转化效果概率成立则改走爆炸（不直接吃伤害）→ ④ 燃烧判定（燃烧概率且非治疗）→ ⑤ 伤害结算：暴击 roll、一击必杀 roll、护甲/加成计算、闪白、音效、击退（含基于与攻击者距离的拉拽式负击退衰减）→ ⑥ 命中派生弹、减速施加 → ⑦ 回调攻击区"命中了目标"（伤害统计）。击杀时回调"杀死了目标"、暴击回调"暴击了目标"（供武器吸血/连锁等使用）。
- 投射物穿透通过忽略对象表累加已命中目标实现。

### 12.4.4 地图边界

边界系统在四边动态创建 4 个矩形碰撞墙，厚度 = `4×地砖尺寸 = 256 px`，位于 obstacles 层（128）。同时单位每物理帧将自身位置钳制在当前区域矩形内（双保险）。

## 12.5 相机行为

局内相机**暂停时仍处理**，吸附过渡时长 5.0 秒。

- **边界**：相机活动范围 = 地图矩形向外扩边缘量 96（上方 ×2）。`最大缩放 = max(地图宽/1920, 地图高/1080)`。
- **单人**：开局即吸附——相机每帧直接贴在玩家位置（夹紧到边界，地图小于屏幕的轴居中），缩放恒为 1.0。
- **合作（动态相机）**：每物理帧平滑跟随所有存活玩家的平均位置：
  - 位置插值因子 0.98（按 `1-pow(1-f, dt)` 做帧率无关化）；
  - **缩放**：目标包围盒（带 `边缘量/缩放` 边距）超出当前视野 → 立即放大到容纳（缩放上限为最大缩放）；包围盒小于屏幕 0.4 比例 → 以因子 0.6 拉近；否则以因子 0.99 缓慢回到容纳所有玩家 + 500 px 边距的级别；
  - 仅剩 1 名存活玩家 → 用 5 s（缓动指数 0.2）把位置与缩放同步过渡到该玩家并永久吸附；
  - 设置 `lock_coop_camera`（禁用动态相机）→ 相机固定地图中心、缩放固定为最大缩放显示全图。
- 相机最大边界还用于波次中敌人投射物等计算"屏内"范围（合作模式按最大缩放扩张）。
- 波次结束后将三个速度因子归零冻结相机。

## 12.6 性能机制

### 12.6.1 通用对象池

全局对象池为 `池 ID → 实例数组` 的字典，池 ID = 对象模板标识哈希或显式指定。取用时弹出并挂到目标容器；归还时从父级摘除存回（不销毁）。带"当前池 ID"单项缓存加速同 ID 连续取用。场景退出时统一销毁。

入池对象：材料、消耗品、浮动文字、视效粒子/斩击特效、出生标记、实体本身（死亡动画完毕回池）、玩家与敌人投射物。各类对象有专用容器：材料/消耗品/出生标记/特效/爆炸/浮动文字/玩家投射物/敌人投射物。

### 12.6.2 数量上限与裁撤

| 项 | 上限 | 行为 |
|---|---|---|
| 同屏视效 | 100 | 超限不播 |
| 浮动文字 | 100 | 超限丢弃，关键文本除外 |
| 地面材料 | 50 | 超限改为随机增值现有材料并放大 |
| 同屏敌人 | 每波数据的敌人上限，合作每多 1 人 +1/8 | 超限随机让旧敌"无掉落死亡"，计入"性能裁撤数"（和平主义者效果会补偿该计数） |
| 结构 | 100 | 超限优先销毁"优先移除"清单内的结构 |
| 音效 | 12 播放器 + 16 队列 | 见 12.2.1 |

### 12.6.3 分帧（time-slicing）

- **生成节流**：实体生成器每 3 物理帧才执行一轮出队生成，普通敌人队列每轮 1 个；队列积压超过 100 时每轮提升到最多 2 个。
- **属性重算分帧**：属性重算系统把玩家/武器/结构/宠物的属性重算放入队列：玩家每帧全量；武器/结构/宠物按"队列年龄"分摊——条目超过 60 帧必算，否则每帧限额 `ceil(age/8)`；非诅咒结构/宠物按同类共享同帧缓存避免重复计算。
- **链接属性半秒刷新**：每名需要的玩家一个 0.5 s 计时器，启动时间按 `0.5/人数` 错开避免同帧。
- **波末优化**：`optimize_end_waves` 跳过材料吸附动画；清场时停掉计时器并令各管理模块进入"清场中"模式。
- **缓存**：每玩家属性字典缓存、手动瞄准判定的帧内缓存（帧末重置）、敌人列表脏标记。
- **轻量计时器**：纯数据的手动推进计时器（推进/循环接口），用于玩家 HP 再生等高频逻辑，避免完整计时器对象的开销。

## 12.7 RNG 体系

- **全局随机数发生器**：启动时随机化种子。封装统一的取整数/区间整数接口。
- **提供设种子接口但正式游戏从不调用**（仅供测试/模组场景）；**没有运行种子机制**，每局完全随机。
- 大量逻辑直接用语言层全局随机函数（与全局发生器不同源）。
- 关键工具：`概率成功(chance)` = `chance==0 ? false : randf() <= chance`；洗牌序列；随机取元素；带小数期望的掉落取整（整数部分 + 余数概率 +1）；地图随机方位/位置系列函数。
- 多人公平性：拾取材料的金币/XP 用轮转指针分配，存活玩家列表常用"洗牌后的存活玩家"随机化顺序。

## 12.8 本地化与文本格式化

### 12.8.1 语言

13 种语言：`de, en, es, fr, it, ja, ko, pl, pt, ru, tr, zh(简体), zh_TW(繁体)`，由翻译表编译生成。语言键 `LANGUAGE_<大写代码>`。首次启动语言可由平台层提供。DLC 与服务器公告可在运行时注入额外翻译。

### 12.8.2 文案格式化系统

`text(key, args, arg_signs)` 是全游戏统一的文案格式化入口：

1. 取翻译，按 `{0}{1}...` 占位替换。
2. **运算符表**：列出"哪个 key 的第几个参数要补 `+` 号"（负数自带 `-`）。若 key 需要参数 0 带符号但翻译串里没有 `{0}` 占位，自动把 `{0}` 拼到句首——**日语例外**：拼到句尾。
3. **百分号表**：对应参数追加 `%`。
4. **着色**：参数符号取"正面/负面/覆写/中性" → 用富文本包裹 `settings.color_positive`（默认绿）/`color_negative`（默认红）/诅咒紫 `#ca61ff`。
5. 千分位格式化：千位逗号分隔。

DLC 可向两张表合并自己的键。浮动文字按需决定是否走翻译。

### 12.8.3 字符串哈希机制

所有 id/键在运行时经字符串哈希（DJB2 变体）转为整数，并维护"哈希 → 字符串"反查表；空串恒为 `5381`。效果数据加载时批量转换。实现时必须保持同样的哈希一致性（或全部改回字符串键）。

### 12.8.4 字体与主题

- 原始字体：Anybody-Medium（拉丁/西里尔）+ Noto Sans JP/KR/SC/TC（CJK 回退）。
- 实际使用动态字号字体资源集：22/26/32/35/40/60/100 号及描边/加粗变体、按钮/菜单/最小字号字体。设置 `font_size`（手柄默认档 1.2，键鼠 1.0）缩放最小字号字体 = 基准 × font_size。
- 主题：一个根主题，另有公告、商店物品栏、小标签、滚动条隐藏、面板/分隔线/按钮样式等专用主题。UI 配色常量：金色、诅咒紫、高亮青、次要字体黄 `#eae2b0` 等。

## 12.9 皮肤/外观系统

纹理级替换系统（不是角色装扮）：

- 启动时（非主机平台）：① 尝试从游戏安装目录加载外置皮肤资源包（如联名皮肤包）；② 扫描皮肤目录下每个皮肤集定义（含集合 id + 皮肤列表）注册为可用皮肤集。
- 单条皮肤定义：`original`（原纹理）→ `skin`（替换纹理），可带外观子项递归映射。
- 激活/停用按集合整体进行，停用列表持久化在 `settings.deactivated_skin_sets`。
- 应用：所有取纹理处经统一的皮肤查询接口——以原纹理标识为键查替换表，命中则换图。地图地砖也走此通道。
- 主机平台全部短路禁用。

## 12.10 地图视觉、平台层与全局服务补充

### 12.10.1 地图视觉

- 地砖：地砖尺寸 64，自动拼接地砖集，对 zone 宽×高逐格填充，子砖按**优先级权重随机**抽取。6 套地砖贴图对应 6 个背景（BG_DIRT/FOREST/VOLCANO/DREAMY_LANDS/BONEYARD/DARKLANDS），波次开始时把背景的地砖贴图设进地砖集并把边框（九宫格描边贴图）调制成背景描边色。
- 画布层背景：屏幕外渐变贴图，第二色从背景渐变色池随机抽取。
- 地图尺寸：`zone.width/height × (1 + map_size效果%/100)`，下限 12 格。
- 边界墙见 12.4.4；另有边界专用地砖集。

### 12.10.2 平台层（与玩法无关，通用工程建议）

建议提供一个平台抽象层，统一封装：成就同步（把游戏内挑战映射为平台成就）、DLC 所有权校验、系统语言获取、商店页跳转。各发行平台（Steam/Epic/GOG/主机）各自实现同一接口，本地版作为空实现回退；主机平台可附加定制行为（如切场景时的高性能模式、串流合作等），均不影响玩法数值。

### 12.10.3 全局辅助服务

- **全局确认弹窗**：最顶层画布、暂停时仍处理，用于平台/系统级提示，确认后恢复之前的焦点。
- **光标管理**：普通指针（热点 3,3）与手动瞄准指针（热点 35,35）切换。
- **合作焦点信号中转**：记录"哪个玩家触发了控件操作"，供共享控件识别操作者。
- **网络公告服务**：从公告服务器拉取按平台过滤的公告（多语言 + 图片下载缓存），已读 id 持久化；主机平台禁用。

### 12.10.4 输入层补充（表现相关）

键鼠/手柄自动检测并切换鼠标可见性（合作多人强制隐藏鼠标）；手柄方向重复输入（echo）延迟 0.3s、间隔 0.09s；摇杆死区 0.5；失焦静音见 12.0；"失去焦点"事件驱动自动暂停。

### 12.10.5 启动流程

启动场景即引导屏状态机 READY→BOOT_SPLASH（首帧绘制触发）→ 等进度存档系统就绪 → HIDE_SCENE（应用音量设置 + 播放启动动画）→ CHANGE_SCENE 切到标题画面。Splash 背景色 `#1c1c1c`。

### 12.10.6 其他支撑模块

| 模块 | 用途 |
|---|---|
| 行为基类 | 敌人移动/攻击行为的共同祖先（持有宿主引用） |

## 12.11 设计检查清单（本章关键数值速查）

- 音乐：起播 -20 dB → 1 s 渐至 0；波败 -20、结算 -8、菜单 -5。
- 音效：12 播放器 / 16 队列 / 每帧出队 1 个；命中音变调 ±0.2，UI 音 ±0.1。
- 闪白：纯白 0.1 s；屏震：玩家受击 (5, 0.1 s)，敌人受击 (min(dmg/3,3), 0.1 s)；无 hit-stop。
- 受击特效位移 = 贴图宽 ×0.25 沿击退方向；斩击特效播放概率 = effect_scale（燃烧 0.1）。
- 碰撞 12 层位值表见 12.4.1/12.4.2；边界墙厚 256 px 在 obstacles 层。
- 相机：合作动态缩放 0.4 比例 / 500 px 边距、独存玩家 5 s 吸附；锁定相机=全图静止。
- 上限：视效 100、浮动字 100、材料 50、结构 100、敌人按波次上限（合作每人 +1/8）。
- RNG 无种子；材料/敌人数量的小数部分按概率进位。
- 13 语言；文案 `+`/`%`/着色由两张键表驱动；日语参数后置。
