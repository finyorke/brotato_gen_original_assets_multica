# 资产对应_04_道具与图标（Brotato v3.7 原版资产 → 游戏元素对应表）

> 本文档覆盖：**208 个道具图标、消耗品、材料、17 张属性图标、升级图标、全局图标**，以及它们在商店/物品栏/结算界面的引用方式与品质（tier）颜色规则。

## 0. 总览与数量自查

| 类别 | 路径 | 数量 | 说明 |
|------|------|------|------|
| 道具图标 | `item_icons/<道具名>/` | **208 个目录 / 209 个道具ID（hourglass 目录含 2 个道具）** | 每个道具一个子目录；商店/物品栏图标通常名为 `<道具名>_icon.png`（例外见表注） |
| 消耗品 | `consumables/` | 3 种（水果、宝箱、传说宝箱） | 含贴图与拾取音效 |
| 材料 | `materials/` | 11 张地面材料贴图 + 3 张 UI 贴图 + 1 张收获图标 + 10 个拾取音效 | 地面绿色小石子即货币 |
| 属性图标 | `stat_icons/` | **17 张** | UI 内属性小图标（含 1 张占位 `empty.png`） |
| 升级图标 | `upgrade_icons/` | 20 张属性升级大图标 + 1 张通用升级图标 | 升级选择界面使用 |
| 全局图标 | `item_icons/` | 6 张 | 禁用/诅咒边框/锁定/随机/信息 |
| 挑战图标 | `ui/icons/` | 2 张 | 禁用系统、驯兽师挑战 |

**画布尺寸规律**：208 个道具图标中 203 个为 96×96；例外：`candy_bag_icon.png`、`hourglass_broken_icon.png`、`will_o_the_wisp_icon.png` 为 150×150，`healBosster_flower_icon.png`（doc_moth）为 250×250，`scapegoat_balofhay_icon.png` 为 95×95。属性图标与升级图标均为 96×96（`upgrade_icon.png` 为 80×80），材料地面贴图为 72×72，`material_ui.png`/`material_bag.png` 为 64×64。UI 中图标统一放入 96×96 的图标面板（开启自动拉伸缩放），因此尺寸差异不影响显示。

## 1. 全部道具的图标对应表（208 个目录 / 209 个道具ID）

说明：
- **道具ID** 即机制文档 05 章使用的 `my_id`。
- **图标路径** 为该道具数据实际引用的贴图（非按文件名猜测）。
- **同目录其他贴图** 为角色身上的外观贴图（`item_appearances`）、弹体、动画帧等，例如 `acid.png` 是穿戴后显示在角色身上的小贴图；重制时同样需要。
- 图标文件名与目录名不一致的特例：`blazemander→blazemander_egg_icon.png`、`bonk_dog→meleedog_bowl_icon.png`、`bot_o_mine→bot_o_mine_remote_icon.png`、`catling_gun→catling_gun_box_icon.png`、`doc_moth→healBosster_flower_icon.png`、`eyes_surgery→eye_surgery_icon.png`、`fresh_meat→icon.png`、`jellyshield→jellyshield_egg_icon.png`、`lootworm→egg_lootworm_icon.png`、`scapegoat→scapegoat_balofhay_icon.png`。

## 2. 消耗品（consumables/）

消耗品是敌人死亡后掉落在地面的可拾取物，对象结构为"触发区域 + 精灵图层 + 16×16 碰撞矩形"。按消耗品生成规则掉落时，直接把该消耗品的图标设为地面贴图——**地面显示贴图与商店图标是同一张 png，原始大小直接显示（scale 1.0），无染色**。

| 消耗品ID | tier | 贴图（兼地面外观与图标） | 拾取音效 | 备注 |
|---|---|---|---|---|
| `consumable_fruit` | 0 | `consumables/fruit/fruit.png`（80×80，橄榄绿色带斑点的水果） | `fruit/comedy_bite_chew_01~03.wav`（3 选 1 随机） | 拾取立即回血 |
| `consumable_item_box` | 1 | `consumables/item_box/item_box.png`（100×100，木箱正面镶绿宝石） | `item_box/item_box_pickup.wav` | `to_be_processed_at_end_of_wave = true`：波次结束结算（开箱送道具） |
| `consumable_legendary_item_box` | 3 | `consumables/legendary_item_box/legendary_item_box.png`（96×96，白色木箱镶红宝石） | 复用 `item_box/item_box_pickup.wav` | 波次结束开出传说道具 |

掉落物的地面行为：从敌人位置弹出后向随机点平滑推移弹开，静置后可被拾取；被玩家吸引时以 500 px/s 起步、每帧 +20 加速度飞向玩家。消耗品与材料共用此逻辑。

## 3. 材料（materials/）

### 3.1 地面材料（货币）

- 对象：地面材料（货币）拾取物。
- 贴图：`materials/material_0000.png ~ material_0010.png` 共 **11 张 72×72** 灰白色小石子/碎矿石图形（形状各异：立方体、长条、月牙、椭圆、棱块等），**贴图本身自带烘焙的白色柔光光晕**。材料生成规则：每次生成时从 11 张中等概率随机挑一张。
- **绿色不是画在贴图里的**：材料对象整体染色 (0.462745, 1, 0.462745, 1)（≈ `#76FF76`），把灰白石子整体染成标志性的亮绿色。重制时材料贴图请画成灰白色，统一染绿。
- 普通材料初始 value = 1，scale = 1.0。

### 3.2 增值材料（boosted）与“闪烁/放大”规则

材料生成规则与拾取结算规则：

1. **奖励金增值**（角色有 `bonus_gold` 时，如商店折返奖励）：新掉落的材料 value 翻倍、`boosted = 2`，并放大 `scale = 1.25`——地面上比普通材料明显大一圈，颜色不变（仍是绿色 modulate）。
2. **场上材料数量达到设计上限时**：不再生成新材料，而是随机挑一颗已有材料 +1 value，并按每点 value +0.05 放大 scale，缩放上限 2.0——这就是后期地面出现"超大绿石子"的原因。
3. **拾取时**：若 `boosted > 1`，`floating_text_manager` 在拾取位置弹出绿色 "x2"（颜色取设置项 `color_positive`）浮动文字。
4. 材料本身没有闪烁动画/shader；视觉上的"发光"完全来自贴图内烘焙的白色光晕 + 绿色 modulate。波次结束若开启 `optimize_end_waves`，剩余材料直接隐藏并折算成奖励金。

## 7. 图标在商店/物品栏/结算界面的引用方式与 tier 边框颜色规则

### 7.1 结论：品质颜色 100% 来自运行时染色，图标 png 本身不含品质颜色

所有道具/升级图标 png 都是**透明背景、无品质色边框**的纯插画。商店卡片边框、物品栏底色、名称文字颜色全部在运行时按 `tier` 上色：

- 颜色常量（默认值，玩家可在设置中改）：`progress_data`
  | tier | 含义 | 默认主色 RGB | 默认深色（背景用） |
  |---|---|---|---|
  | 0 COMMON | 白/普通 | (230,230,230) `#E6E6E6` | (0,0,0) |
  | 1 UNCOMMON | 蓝/罕见 | (90,190,255) `#5ABEFF` | (15,32,40) |
  | 2 RARE | 紫/稀有 | (173,90,255) `#AD5AFF` | (16,10,27) |
  | 3 LEGENDARY | 红/传说 | (255,59,59) `#FF3B3B` | (36,9,9) |
  | 4 DANGER_4 | 橙（危险 4 难度） | (255,119,59) `#FF773B` | (36,17,8) |
  | 5 DANGER_5 | 金（危险 5 难度） | (208,193,66) `#D0C142` | (26,24,8) |
  （是同一套默认值，运行时实际读取玩家存档设置中的 `tier_X_color`。）

- 取色规则：按品质查色表（可选深色版本），色值见下表。
- **商店卡片/弹窗/道具面板**：面板品质染色规则——把面板边框色设为品质主色、底色设为对应深色（保留原透明度）；品质 0 时边框用黑色（弹窗用 0.3 灰）。应用于：商店卡片、图标格、悬浮弹窗、波间道具面板、升级四选一、角色选择等。锁定商店物品时边框改为纯白。
- **物品栏/结算界面的小格子**（商店底部物品栏、结算/死亡界面物品列表、图鉴均用它）：`change_inventory_element_stylebox_from_tier(stylebox, tier, 0.25)`——直接把格子 `bg_color` 填成品质主色、alpha 0.25（tier 0 用黑色 alpha 0.39），**没有边框**；图标原图叠在底色上。
- **名称文字**：`item_description`：`_name.modulate = get_color_from_tier(tier)`，道具名按品质染色。
- **图标本体永不染色**（除图鉴未解锁时整体涂黑剪影 `silouhette_if_hidden_in_codex`），诅咒道具在图标上层叠 `curse_border_light.png`。
- 武器（非本章范围）另有描边 shader 用同一套 `get_color_from_tier` 上色，规则一致。
