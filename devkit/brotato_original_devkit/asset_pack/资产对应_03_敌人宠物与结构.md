## 0. 通用规则（所有单位共享，必读）

### 0.1 对象分层结构模板

```
实体（物理刚体, 自定义运动积分）
├─ 动画容器层 (基准位置 y=-24)              ← 所有程序动画作用在这一层
│   └─ 主贴图精灵图层 (单张静态贴图, scale=1) ← 子对象仅替换贴图 / 位置
├─ 碰撞形状                                ← 物理体（推挤/墙）
├─ 受击区（触发区域, 默认位置 y=-24/-16）    ← 受击判定
├─ 伤害区（触发区域）                       ← 接触伤害判定（仅敌人）
├─ 动画轨道 (自动播放 "idle")
├─ 闪白计时 (0.1s) / 燃烧计时 (0.5s)
└─ 移动行为 / 攻击行为 / 效果行为
```

要点：
- **每个敌人只有 1 张静态 PNG，没有逐帧序列**。走路/攻击/死亡全部是动画轨道对动画容器层做的程序化压扁拉伸（squash & stretch），重制时必须保留这些动画曲线（见 0.2）。
- 主贴图图层 `scale = (1,1)`，贴图按原始像素 1:1 显示；朝向翻转由单位的 `mirror_sprite_with_movement=true` 控制（向左移动时水平镜像）。`corrupted_tree`、`slasher_egg` 关闭了翻转。
- 敌人碰撞层：本体 `layer=4, mask=132`；受击区 `mask=1032`（玩家武器+爆炸）；伤害区 `layer=4`。
- 受击白闪：`shaders/flash.gdshader`，闪白时长 0.1s。
- 描边/诅咒材质：`shaders/outline.gdshader`。

### 0.2 共用动画资源（程序动画，无贴图帧）

| 动画 | 资源 | 关键参数 |
|---|---|---|
| idle（默认循环） | entity 内嵌 `idle` | 1.0s 循环：动画容器层 y -24→-19→-24；scale x 1→1.15→1、y 1→0.85→1（贝塞尔） |
| death | entity 内嵌 `death` | 1.05s：贴图 0.5s 内 scale→0、旋转 360°、染色→灰(0.349)；0.25s 触发停止燃烧、1.0s 触发死亡动画结束事件 |

| 资产 | 尺寸 | 用法 |

显示参数：
- 标记贴图 `scale=0.33`（屏显约 84×84px），出生时 `rotation_degrees = rand(0,360)`。

- 防压脸检测区域：半径 72、mask=2（玩家层）；倒计时结束若与玩家重叠且类型为敌人→传送随机点重计 1s。

## 4. 宠物（9 种，pets/）

### 4.1 共用

- 宠物基类：动画容器层 y≈-24、本体 r40（多数子对象覆盖）、`can_be_targeted_by_enemies=false`（默认）。
- **组装式骨骼渲染**：宠物不是单图——各部件精灵图层挂在一个离屏视口（渲染结果垂直翻转）下，由动画轨道旋转/位移部件，最终把视口渲染结果作为贴图贴回主显示图层。重制时可以用同结构，或直接烘焙等效逐帧动画。
- **摸摸手**：units/pet/hand_pet.png（120×120 白手套），波末抚摸演出共用。
- 每个宠物目录均有 `<id>_icon.png`（95/96×95/96 道具图标）、`<id>_screen.png`（549×236 说明图）、`<id>_item`、`<id>_stats`。
- `<id>.png`（如 blazemander.png 256×160）是**部件图集预览**，运行时不直接使用；运行时引用的是拆分部件 PNG。
