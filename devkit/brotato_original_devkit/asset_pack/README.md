# Brotato 资产包

> 本包为游戏全套美术、音频资产与使用参数对应表，配合 `../game_mechanics_docs/`（13 章机制文档）使用。开发者依据本包资产与各对应表中的设计规格参数，即可完整实现游戏的画面与音频表现。

## 文件夹结构

资产按用途分类组织（1535 个文件）：

```
asset_pack/
├── assets/
│   ├── characters/<角色id>/      # 49 个角色的图标与外观部位件
│   ├── player/                   # 玩家本体贴图（身体/腿/影子/高亮/降落伞）
│   ├── weapons/<武器id>/         # 61 把武器的贴图、图标与专属音效
│   ├── projectiles/<弹道名>/     # 弹道贴图与序列帧
│   ├── enemies/<敌人id>/         # 敌人贴图、图鉴图标与专属音效
│   ├── pets/<宠物id>/            # 宠物贴图与音效
│   ├── structures/<结构id>/      # 炮塔/地雷等结构物贴图
│   ├── neutral/                  # 中立物（树/石头）
│   ├── item_icons/<道具id>/      # 道具图标与外观件
│   ├── stat_icons/               # 属性图标
│   ├── upgrade_icons/<属性>/     # 升级图标

│   ├── consumables/              # 消耗品（水果/宝箱）贴图与音效
│   ├── ui/                       # UI 贴图（hud/icons/menus/popups/splash 分组）
│   ├── vfx/                      # 特效与粒子贴图（含出生标记、命中特效）
│   ├── tiles/                    # 地砖
│   ├── shaders/                  # 3 个着色器（受击闪白/色相偏移/描边）
│   ├── fonts/                    # 字体文件

│   ├── audio/music/              # 音乐
```

## 使用方法

1. 每份对应表的"游戏元素 ID"与机制文档章节一一对应（资产对应_01 ↔ 机制文档 02 章角色 ID，资产对应_02 ↔ 03 章武器 ID，以此类推）。
2. 表中"资产路径"均为 `assets/` 下的相对路径。
3. **使用参数列必须照搬**：精灵图层的缩放/偏移/锚点、动画帧数与时长、碰撞形状半径、九宫格切片边距、音量 dB 与 pitch 随机范围——这些设计规格参数与机制文档中的玩法数值共同决定还原度。
4. 程序化表现注意（详见各对应表）：
   - 玩家/敌人的 idle/move 动画是**变换动画**（位移+缩放插值），不是帧动画；
   - 武器 Tier 1–4 **共用同一贴图**，品质区分只靠 UI 边框颜色（代码取色）；
   - 道具品质颜色、地面材料绿色均为**代码染色**，图标本体不含品质色；
   - 地面为 12 子砖按权重随机填充，六套主题每局随机抽取；
   - 音乐为 11 首单列表洗牌轮播，场景区分仅靠音量分档（见机制文档 12 章）。
5. 预留/未使用资产：约 20 个未挂接到任何游戏元素的图片/音频（各对应表已标注"预留/未使用"），实现时可忽略。
