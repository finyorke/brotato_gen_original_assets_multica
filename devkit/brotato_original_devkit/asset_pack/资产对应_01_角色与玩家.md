## 1. 玩家实体通用视觉（所有角色共用）

玩家对象 `player`，继承通用单位 `unit`。
对象分层结构（与动画相关部分）：

```
玩家（物理刚体）
├─ 动画容器层                     position=(0,-24)
│  ├─ 影子精灵图层
│  ├─ 选中高亮精灵图层(默认隐藏)
│  ├─ 身体精灵图层                土豆身体
│  ├─ 左腿、右腿图层              双腿(显示在身体后面)
│  └─ (运行时追加) 角色外观精灵图层
├─ 武器容器层                     position=(0,-24)，武器挂点 One..Six
└─ 动画轨道
```

### 1.2 玩家动画（重要结论：全部是“变换动画”，不是帧动画）

动画轨道注册的动画与文件：

| 动画名 | 动画资源 | 时长 | 循环 | 内容（关键轨道） |
|---|---|---|---|---|
| `idle` | `player_idle` | 1.0s | 是 | **纯变换**：动画容器层贝塞尔轨道，position.y −24→−19→−24，scale (1,1)→(1.15,0.85)→(1,1)，关键帧 t=0/0.5/1.0。即"呼吸式压扁"——压低时同时变宽变矮。腿复位到 (15,18)/(−16,18)、旋转 0 |
| `move` | `player_move` | 0.6s | 是 | **纯变换**：动画容器层 position.y 在 −24→−14→−29→−24 间弹跳（t=0/0.1/0.2/0.3，0.6s 内跳两轮），scale (1,1)→(1.2,0.8)→(0.9,1.1)→(1,1) 同步挤压拉伸；左右腿各自有 position+rotation_degrees 7 帧轨道模拟交替迈步（rotation −29.1°~109.1°）；右腿 `scale=(-1,1)` 水平镜像。**事件轨道在 t=0.1 与 t=0.4 各触发一次脚步声** |
| `death` | player 内嵌动画 | 1.05s | 否 | 身体图层染色变暗至 0.349 灰、scale 缩到 1e-05、rotation 0→360°；t=1.0 触发死亡动画结束事件 |
| `landing` | 内嵌动画 | 1.2s | 否 | 开局空降：身体/腿/武器从 (−265,−725) 等高空插值落到原位，落地瞬间 t=1.05/1.1 squash (0.776,2.02)→(1.481,0.762)→(1,1)；影子在 t=1.0→1.1 淡入 |
| `landing_swim` | 内嵌动画 | 1.2s | 否 | 水面版空降（翻滚入水） |
| `idle_swim` | 内嵌动画 | 1.0s | 是 | 游泳待机：同 idle 弹性 + 身体图层 position.y −8.556 上浮、腿摆角 32.9°/−23° |
| `move_swim` | 内嵌动画 | 0.6s | 是 | 游泳移动；事件轨道 t=0.1/0.4 触发脚步声（此时音源已换为划水声组） |
| `dance` | 内嵌动画 | 2.0s | 是 | 胜利舞：身体左右摆 ±14.6px、旋转 ±12°，腿与武器随动 |
| `won` | 内嵌动画 | 7.0s | 否 | 胜利起跳后飞出屏幕（y −823）；t=1.15 触发进入胜利结算画面 |
| `RESET` | 内嵌动画 | 0.001s | — | 复位 染色/scale/rotation |

**朝向翻转规则**：不靠贴图水平翻转开关。单位每帧更新动画时——水平移动分量 > 0 时身体图层横向缩放取正值（朝右），< 0 时取负（水平镜像）。`mirror_sprite_with_movement = true`。角色外观图层与身体图层互为兄弟层、但镜像作用于各图层自身横向缩放，外观件挂到动画容器层下，与该层全部子图层一起翻转处理。

### 1.3 材质 / Shader

| 用途 | Shader 文件 | 挂载方式 |
|---|---|---|
| 受击闪白 | `shaders/flash.gdshader`（纯色混合：`color.rgb = mix(color.rgb, flash_color.rgb, flash_modifier)`） | 受击时将单位贴图材质替换为闪白材质（flash_color=白、flash_modifier=1.0），0.1 秒后还原原材质。**即受击=整张贴图变纯白 0.1 秒** |
| 描边/选中轮廓 | `shaders/outline.gdshader` | 角色描边材质（width=3.0，4 个描边色默认全透明）；用于角色高亮描边 |

### 1.4 角色外观件的运行时挂载规则（贴图如何上身）

- 每个角色的 `*_data` 里 `item_appearances` 数组引用若干 `*_appearance`。
- 每个 appearance 含：`sprite`（150×150 PNG，画布与 potato.png 对齐，**无需额外偏移**）、`position`（部位枚举，仅作冲突/分类元数据）、`display_priority`、`depth`（z 排序）。
- 道具效果生效时：按 `depth` 升序排序后，为每个 appearance 新建一个精灵图层加到动画容器层（位置 (0,0)，与身体同画布对齐）；`depth < -1` 的件显示在身体后面并移到该层最前（如恶魔尾巴、流浪汉行囊）。
- position 枚举值：0=OTHER 1=ABOVE_HEAD 2=TOP_LEFT 3=TOP_RIGHT 4=HAT 5=FOREHEAD 6=EYES 7=NOSE 8=MOUTH 9=BACK 10=NECK 11=TORSO 14=SKIN 15=TAIL。

## 3. 角色 icon 在 UI 中的引用位置

| UI 场景 | 引用方式 |
|---|---|
| 角色选择页 | `character_selection` + `character_selection_inventory`；每个格子是物品格控件 `inventory_element`，格内图标取角色数据的 `icon` |
| HUD 波次时间线 | `ui_timeline` 控件：当前位置图标取角色 `icon`（角色头像随波次进度在时间条上移动） |
| 商店角色描述面板 | 商店角色描述控件以"名称 + icon"填充 |
| 游戏内角色面板（图鉴/暂停预览） | `character_panel_ui` 界面：用各 appearance 的贴图实时拼出土豆+部位件的立绘（非 icon） |

## 4. 玩家相关音频

播放规则：玩家与通用单位共用，随机规则统一为"从候选数组中等概率随机抽一个"。

### 4.1 脚步声（陆地）
- 文件：
  `assets/audio/sfx/player_steps/Step 1.mp3` ~ `Step 6.mp3`（6 个，各约 0.65s）
- 触发：`move` / `move_swim` 动画的事件轨道在 **t=0.1s 和 t=0.4s** 各触发一次脚步声，即每 0.6s 循环响 2 次。
- 播放参数：全局音频播放随机文件——音量 −6dB，音调随机抖动 ±0.1。

### 4.2 游泳划水声
- 文件：`assets/audio/sfx/player_steps/Swim 1.mp3` ~ `Swim 4.mp3`（0.13~0.29s）。
- 触发：进入水中后玩家的脚步声组整体替换为划水声组，之后同 4.1 的脚步触发点播放。

### 4.3 玩家受击声
- 文件（覆盖 unit 默认值）：

- 触发：玩家受伤时随机抽一个播放；同时触发单位通用的 0.1s 受击闪白。
